任务

Javascript

如果我们检查页面，我们可以看到支持该功能的脚本。这是第一个骗局出现的地方。如果您查看页面源代码而不是检查元素，您可能不会注意到正在设置第一个变量。下图显示第一个变量被设置在最右边。不过，可能太小了，看不清。

但是，如果我们检查该脚本，您会看到在脚本的最开始，我们可以看到值'moo'被分配给变量'RawrRawr'。接下来，我们可以看到一个名为“x”的函数被创建，其中“hack_this_site”的值被添加到变量“+RawrRawr+”中。话又说回来，也许不是。如果我们看看'moo'是如何分配给'【T6]'的，我们只会看到一个等号。在javascript中， double equals 符号充当比较两个值的运算符，所以这里发生的只是将“hack_this_site”与“RawrRawr”进行比较。

解决方案

'RawrRawr'的值还是moo，接下来看看脚本怎么做吧。该脚本有一个if语句，将'x'与两个语音标记之间的空值加上'RawrRawr'的值进行比较。所以从本质上来说，这个脚本是在检查'x'的值，而不是检查'moo'的值，因为'moo'是更早设置的。所以完成任务的密码应该是moo。在框中输入，然后单击提交进行评分。
