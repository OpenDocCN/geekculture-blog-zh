## 什么是数据科学？

数据科学似乎是一个令人费解的术语，它的流行主要与机器学习框架的进步有关。一些人认为*数据科学*是高薪统计学家的术语，另一些人认为他们是从*软件工程*改编而来的角色，还有一些人认为他们是拥有技术技能的*商人*。我见过激烈的论坛讨论这个术语的精神特质。我发现菲尔德·卡迪对数据科学的定义恰到好处，可以解决这种紧张关系。

> 数据科学意味着做分析工作，出于这样或那样的原因，需要大量的软件工程技能。

这种实践的核心是一套使人成为数据科学家的技能。

# 什么是数据科学家？

数据科学家的伪装有三顶帽子。

统计学家被训练去做所有的分析方法，而数据科学家扩展他们的知识去应用统计技术来建立必要的工具来建模一个计算系统。数据科学家确保他们涵盖了尚未发现任何已知工具有用的边缘情况。他们运用他们在计算机编程方面的熟练程度，而这是传统统计学家所不具备的。数据科学家不需要统计工具来完成工作。

伪装成工程师。数据科学家必须精通技术。他们必须知道如何在不过度依赖现成软件的情况下，将数据转换成有意义的东西。覆盖边缘案例意味着探索未知领域会产生不规则性。数据科学家必须处理不规则现象。适应的一个方法是在手工制作工具的过程中发展软件技能，以覆盖边缘情况。请注意，将您的数据转化为最终对您的模型有用的东西是数据科学(和机器学习)的[最关键和最乏味的过程](https://www.elderresearch.com/blog/what-is-data-wrangling-and-why-does-it-take-so-long/)之一。

> 这也是激励任何人磨练软件技能的一个好点。不仅如此，你的成长超越了软件中提供的现成服务，而且你还获得了去任何你想去的地方的自由。没被拴住。

商务人士以产品的价值为导向。他们的感觉是遵循对他们的商业问题至关重要的东西。Field Cady (2017)指出，优秀的数据科学专业人员与平庸的专业人员之间的区别不在于他们的数学或工程技能，而在于制定正确的问题。我们要解决的问题是粗略估计我们产品的价值。此外，商务人士也善于与他人沟通。同样，数据科学家必须以故事的形式告诉我们他们最有趣的见解，这不会限制在一屋子了解数据科学的人之间的讨论。

> 再多的技术能力或统计严谨性也无法弥补解决了一个无用的问题。

# 数据浏览器的路线图

既然我们已经阐明了这些术语，那么让我们像野外数据探索者一样在工作流中布置我们的地图。这是我从弗朗索瓦·乔莱的《深度学习入门》和菲尔德·卡迪的《数据科学家手册》中得到的。

1.  **定义问题**。列出你想问的问题。这将是您理解所需数据类型的指南。这通常是分组进行的。这一步很关键，因为它设定了你的目标，你需要非常清楚这一点。在您的团队中进行这种讨论可以让每个人都在同一页上为同一组目标工作。
2.  **收集相关数据集**。检查数据集上是否有空白。连接两个数据集意味着什么？他们同意一件事吗？这也是至关重要的，因为它影响你的输出质量:你输入数据，你得到信息。如果你给你的模型输入一堆垃圾(有病态的模式，太多的异常值，等等。).在微软研究人员[米歇尔·班克和埃里克·比尔(2001)](https://dl.acm.org/doi/10.3115/1073012.1073017) 以及计算机科学家[诺维格*等人* (2009)](https://bityl.co/6bZq) 的一篇著名论文中，数据对算法更重要。
3.  **黑分析**。这就是它变得如此明亮和美丽的地方！你所做的就是玩一堆旋钮，并将它们随机扭在一起，看看一个如何影响另一个——这就是著名的神经网络架构:深度学习人工智能的基础。另一方面，是一组来自统计学和计算机科学的更严格的算法，称为聚类、分类、我们的传统回归和降维:这包括[浅层 ML 架构](https://bityl.co/6bdU)。
4.  **传达您的结果**。总之:我们应该在一个连贯的故事中裁剪我们的结果。让它引人注目！这包括我们的意图，以及以任何人都可以理解的格式汇总的分析结果。我们不再和房间里的同事说话了。我们为人民说话。让它可视化。
