**休息→**

很久很久以前，在一个叫做 developer 的社区中，他们用来创建 web 应用程序，也就是所谓的“单片”，因为他们过去常常在一个大文件中告诉应用程序要做什么..(想象一下调试这样的系统有多恐怖)。然后出现了一些聪明的人，他们认为我们为什么需要这样做，我们可以将它分成可以在不同端点提供的小服务，因此从灰烬中诞生了微服务的概念。

总之，REST APIs 帮助开发人员创建一个更易于管理的代码库，您可以在其中为特定请求提供应用程序数据。让我详细说明一下..

让我们以一个基本的电子商务 web 应用程序为例。现在我们想让一个用户登录。现在，我们可以将表单数据(form = = log in form)发送到一个端点，在那里我们将函数(我喜欢称之为 handler，因为它处理东西)映射到该端点(或链接),它提取表单数据并查询数据库以检查用户的存在和有效性。如果用户在场，它将告诉应用程序登录成功，可以继续了。如果失败，它可以向应用程序发送一条错误消息，甚至是一个带有消息的代码，可以用来告诉用户输入的信息是错误的。现在，由于这是通过将一个功能映射到一个端点来完成的，我们可以将这些与用户有关的功能分组到一个地方，这样就更容易管理了。

我稍后会展示它的样子。REST APIs 有一个叫做动词的东西，它告诉我们预期的请求类型。Get 和 Post 是最常见的，您可以认为 get-requests 的目的是从后端获取一些信息(如电子商务 web 应用程序的产品列表)。当您希望将数据发送到后端时，会发送一个 post 请求(例如，在前面的示例中，登录/登录将是一个 post 请求，因为我们希望将表单数据发送回后端，以检查用户的有效性，并且不会泄露用户在表单中输入的信息)。其他有趣的动词还有 **put，delete，patch …** 等。如果好奇的话，你可以阅读更多关于类型以及有哪些动词的信息，但 get，post，delete，patch，put 是我所经历的最常见的(大多数时候是按这个顺序)。

在这篇文章中，我会选择 SQL 作为数据库，因为 mongoDB 教程太常见了，有时关系数据库比非关系数据库更有效。现在，选择数据库是一个非常重要的决定，因为您需要最有效地存储和检索数据。我可以链接一个关于后端之战的很好的视频，作为何时使用哪个数据库的参考。你也可以同时使用这两种类型。(看看这个→【https://www.youtube.com/watch?v=rRoy6I4gKWU )🤔也许将来我也会写另一篇关于数据库的博客。

现在我们对 REST 有了一些了解(你总是可以得到)，让我们来谈谈编码。现在有很多选择来编写后端代码，但是最流行也是我最喜欢的是 javaScript 和 python。

Python 实际上有自己完整的框架来创建 web 应用程序，称为 django，但是如果你习惯于节点应用程序，flask 是你的好朋友。对于这个博客，我将坚持使用 javaScript。在 javascript 中，您需要 nodeJS 来创建应用程序。

这里有一个下载和安装节点的链接。也将它添加到 path 中，以便在任何地方从 cmd/terminal 使用它。

链接到节点→[https://nodejs.org/en/download/](https://nodejs.org/en/download/)

检查安装是否成功..运行`node — — version`如果它显示一个版本，祝贺节点已经安装，现在我们可以开始开发。

我们将使用 node package manager 安装模块来开发后端，我将解释并展示如何组织和模块化它。

**开始开发流程→**

我们将使用 npm 来安装用于创建后端服务器的软件包..快速 JS 模块。

为了方便和简洁，我们将使用 VSCode，但是您可以选择您最喜欢的编辑器继续。因此，为项目创建一个文件夹，并在编辑器中打开它。要开始开发，运行`npm init`并回答询问的信息(或者只需按 enter 一串时间来保留所有默认值)。这将创建一系列文件，为节点项目做好准备。

`npm install -g express --save`安装模块。这是安装包的标准方式。我们将在整个博客中安装很多包，所以你会很需要它。一旦安装了包，并将其上传到 git，(您需要将 node_modules 文件夹添加到 gitignore，因为上传到 git 会很大)。当你在一台新机器上再次拉这个项目时，你可以运行`npm install`来安装所有的包。

现在我们已经准备好了一个项目，我们需要创建一个 **server.js** 文件来为我们的服务器编写逻辑。启动服务器时，我们将运行这个文件在指定的端口启动服务器(将在代码解释部分解释)
