这个博客回答了我从那些不使用或不想过渡到云技术的人那里得到的最常见的问题。

> “为什么是云？”

我的回答相当一致:更好的安全性，更低的成本，更高的资源管理灵活性，没有基础设施维护开销。将所有这些以及更多因素组合在一起，您就可以得到 *AWS 云价值主张*。

概括地说，云的价值主张是:

> 安全可靠地访问高度可用、可靠且耐用的 IT 资源，如计算能力、存储和数据库，按需购买，按需定价

AWS 云价值主张有五个原则。请记住，这些不同于[的架构良好的框架](https://aws.amazon.com/architecture/well-architected/?wa-lens-whitepapers.sort-by=item.additionalFields.sortDate&wa-lens-whitepapers.sort-order=desc)。

*   **安全**

> AWS 为各种应用提供多种安全和合规服务。除了共享责任模型，还有配置额外安全措施的选项。

一些安全措施包括身份和访问管理、密钥管理服务、AWS 警戒和 AWS 屏蔽。[合规性](https://aws.amazon.com/compliance/programs/)特性包括 PCI-DSS、HIPAA/HITECH、GDPR 等。

*   **敏捷性**

> 可以在几分钟内跨多个地理位置开发和部署应用程序。与内部开发的应用程序相比，试验、开发和部署的速度成倍提高。

[AWS 全球基础设施](https://aws.amazon.com/about-aws/global-infrastructure/)提供了一个高度健壮和可靠的云平台，用于在区域、可用性区域和边缘位置开发和部署应用程序。

*   **灵活性**

> AWS 为各种应用提供多种服务。有 200 多种产品支持运行各种工作负载的服务。

AWS 为从网站托管、微服务到 AI-ML 和大数据的应用程序提供开发服务。 [AWS 开发人员工具](https://aws.amazon.com/products/developer-tools/)为持续完整性/持续交付(CI/CD)、监控和日志记录以及代码建模基础设施分发服务。

*   **弹性**

> 应用程序可以根据需求扩展平稳运行所需的资源—应用程序的容量足以处理传入的请求，同时只需为它们所使用的资源付费。

[自动扩展](https://aws.amazon.com/autoscaling/)在传入需求波动超过阈值时提供资源。例如，如果一个新实例的 CPU 利用率超过 70%,则提供该实例来平衡负载。当 CPU 利用率低于 40%时，新实例将被终止。

*   **成本**

> AWS 可以根据需求轻松扩展和缩小规模，并且消除了对内部数据中心的需求，从而降低了成本。通过使用现收现付模式，可以监控和降低发生的成本。

[成本优化工具](https://docs.aws.amazon.com/whitepapers/latest/cost-optimization-laying-the-foundation/reporting-cost-optimization-tools.html)帮助跟踪在不同地区和 az 推出的不同服务的资源分布情况。监控、记录和预测成本有助于降低向最终用户提供应用程序时产生的成本。
