# 什么是颤振

Flutter 的特性和它所释放的跨平台能力使它很容易与 React Native 和 Xamarin 等移动应用程序框架进行比较。然而，从技术上讲，这并不完全准确。Flutter 不仅仅是一个框架。

Flutter 是一个软件开发工具包(SDK ),其中包括构建跨平台应用程序所需的完整工具和资源套件。一体化解决方案。虽然开发人员瞄准 Flutter 应用程序的方式与他们瞄准任何其他框架的方式一样，但正是底层技术让 Flutter 在人群中脱颖而出。

有了 Flutter 的工作知识、该平台的副本和适当配备的 IDE，您就拥有了开始为 Android、iOS、Windows、macOS 和 web 构建应用程序的一切。更好的是，您可以从一个单一的代码库面向所有六个平台，而无需引入任何额外的工具或支持。

Flutter 是一项值得关注的令人兴奋的技术，因为它的方法与当今最流行的框架不同。这些令人兴奋的新变化的代价是适应创建平台基础的语言和架构差异。

虽然开发人员仍在跟上 Flutter 的速度，其社区仍在增长，但随着 Flutter 越来越受欢迎，得到越来越多的支持，势头越来越猛，要适应 Flutter 的构建方式将会有一个飞跃。

然而，与 Flutter 应用程序开发带来的优势相比，这种飞跃的代价是微不足道的。理解 Flutter 的架构以及使其成为移动应用程序开发最强大的工具之一的组件是构建强大的新应用程序的踏脚石。

# 什么是谷歌飞镖

颤振开发的最大技术和心理障碍之一是采用 Dart 编程语言。

Dart 由谷歌构建，是该公司内部开发的语言。Dart 创建于 2011 年，在最近的创新之前，它的发展相对缓慢。它在 Flutter 和该公司自己的移动操作系统 Fuchsia 中的使用已经吸引了越来越多的开发者。

学习 Dart 的初始投资比许多人想象的要少。这种语言实际上是为多平台开发量身定制的。具有 web 或 JavaScript 开发背景的开发人员会很快熟悉它。

Google 提供了丰富的可用资源，以及一个惊人丰富的社区，这有助于向该语言的过渡相对顺利。

作为学习 Dart 的交换，一系列丰富的现代工具可以将快速构建和测试带到应用程序开发的前沿。在这些工具中，Dart 灵活的类型和丰富的静态分析特性使其成为未来开发项目中有价值的伙伴。

# 颤振建筑

正是 Flutter 的架构和设计使其成为如此强大的应用开发平台。

Flutter 被设计成位于底层目标系统之上的一系列可互换的层。这些层从应用程序运行的平台(如 Android 或 iOS)中抽象出应用程序设计，从而允许开发人员以一个通用代码库为目标。这个特性集和 Flutter 实现它的方式，使开发人员更加接近真正的“一次构建，随处运行”的理念。

![](img/c1cd80e1c8a5a6cd2e38735f3bf6537f.png)

在 Flutter 的基础层是平台嵌入器——最底层的组件，也是唯一一个为每个目标平台独立设计的组件。嵌入器的作用是提供与目标系统直接交互的基本类。嵌入器为 Flutter 的平台无关代码提供了一个入口点，以便与底层的设备和操作系统进行通信。简而言之，嵌入器为特定于系统的特性提供了特定于颤振的抽象。

颤振体系的下一级是负责平台许多突出特性的发动机。其核心是为 Flutter 的 UI 和小部件提供动力的渲染引擎——这值得再看一眼。

进入 Flutter 引擎的相关组件包括网络、插件支持以及 Dart 语言的运行时文件和编译器。这里包括了基本的系统和库，这些系统和库允许在通过嵌入器与底层系统接口之前执行 Flutter 代码。

与引擎交互，Flutter 架构的最顶层是框架本身。层 [Flutter 开发人员](https://www.pangea.ai/services/flutter-development/)几乎把他们所有的时间都花在了编写输入上，这一层也为开发人员之间提供了一个方便的速记来整体引用 Flutter SDK。

在这个框架层中有 UI 小部件、应用程序组件和接口，它们用于构建多平台应用程序。不可避免地，Flutter 框架包含了 Google 的材料设计标准和 Apple 的 Cupertino 设计库的不同 UI 组件。

Flutter 的分层组件风格架构允许应用程序在性能上保持相对集中和精简。它允许只包含运行底层应用程序所必需的应用程序组件，排除他们不需要和不会使用的部分。

开发人员可能希望依赖的其他功能，如设备的摄像头或传感器、webview、应用内支付和身份验证，可以根据项目的需要添加到代码模块中。

# Flutter 的独特优势:渲染

Flutter 的突出特性和平台的独特卖点主要来自于它的渲染引擎。在多平台应用程序框架中，Flutter 是相对独特的，它为屏幕呈现自己的 UI 组件，这是该架构的核心设计理念之一。

通过在屏幕上绘制每个像素，Flutter 提供了很少框架能够比拟的细节和控制水平。

传统的做法是移动应用程序框架创建一个抽象层来调用系统的原生 UI 库。这简化了框架设计，并允许目标系统代替框架做“繁重的工作”。缺点是在与特定于系统的库交互之前，解释另一种语言(通常是 JavaScript)会产生额外的开销。

相比之下，Flutter 通过实现自己的一套跨平台使用的小部件来最小化这种开销。Flutter 应用程序完全绕过系统的 UI 库，将自己的视觉效果渲染到本机代码中，并提供高达每秒 120 帧的渲染性能。

从用户的角度来看，flutter 的方法的缺点来自于 Flutter 应用程序和非 Flutter 应用程序之间的按钮和 UI 功能的微小变化。由 Flutter 渲染的组件与本机类似，但来自系统本身以外的库。

好处是界面可靠、一致，可以跨多个平台和各种屏幕尺寸运行。

# 结合颤振和原生

因为 Flutter 背离了使用框架和创建跨平台应用程序的传统方式；对于将 Flutter 代码合并到本机应用程序中，或者相反，有一些自然的顾虑。

结合两个世界不仅仅是对 Flutter 能力的技术展示。当对现有工作或任何框架都无法满足的独特需求进行大量投资时，这种应用程序是必不可少的。在许多情况下，一个广泛的代码库的逐渐变化是比突然中断或从头重新设计更好的解决方案。

Flutter 解决了这个问题，并创造了将特定于平台的视图包含到 Flutter 应用程序中以及将 Flutter 组件导入到本机应用程序中的方法。

平台视图小部件使您能够将本地视图导入到 Flutter 应用程序中。解决一个棘手的多平台问题，并允许小部件充当视图和应用程序之间的中介。

相反的情况是通过一个 Flutter 模块实现的，该模块允许将一个 Flutter 应用程序嵌入到父本地应用程序中。该模块包括引擎、共享库和 Flutter 应用程序运行所需的工具。

将 Flutter 应用程序需要的一切都包含在应用程序中有一个自然的权衡，为了兼容性牺牲了完全的性能。这远远不是一个完美的解决方案，它是一个合理的中间步骤，当只有一个本地应用程序可以使用时，可以访问 Flutter 功能。

# 实践中的颤动

新技术本质上是令人兴奋和有吸引力的。全新的解决方案让团队想在第一时间跳上船，利用新趋势和尖端工具。然而，很少有事情能如此轻松地完成。

要投资一项新技术或平台，你必须证明这种方向上的改变将为未来提供真实的、可量化的价值和解决方案。很多企业跳过了这一步，只遇到了技术的缺点和不足。

有了 Flutter，平台的价值和优势很容易在它的架构和设计中体现出来。

与传统架构的巨大差异证明了类似本机的性能、快速跨平台应用程序开发和富于表现力的 ui 的说法是正确的。Flutter 支持一种更充分利用技术和平台的方法，它的额外特性值得认真考虑。

当然，也有缺点。其中最重要的是 Flutter 对实现布局和部署小部件所做的改变。

Flutter 工作方式的改变需要一些时间来让开发者跟上速度。还不熟悉 Dart 的开发人员也需要时间来学习谷歌语言的最深层次的特性和深度——这个过程被证明是惊人的短和简单。

该平台的一体化方法，即决定包含应用程序内部运行所需的一切，也增加了最终应用程序的大小。对于需要较小空间的应用程序，更轻量级的原生解决方案可能仍然是更好的方法。

在相对较新的平台上投资的回报是 UI 定制的新水平，上市时间的缩短，以及竞争平台之间丰富的代码共享。
