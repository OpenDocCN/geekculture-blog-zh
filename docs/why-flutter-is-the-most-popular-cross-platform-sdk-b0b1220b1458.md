# 什么是颤振？概述

Flutter 是一个 UI 工具包，用于从单个代码库开发跨移动、web 和桌面的类似本机的应用程序。它使用编程语言 Dart，包括材料设计和 Cupertino 部件。

Flutter 应用程序开发公司用看起来和感觉上都很自然的 Flutter 框架设计了令人惊叹的 UI。它可以在任何平台上自然运行，即使你使用的是单一的代码库。

Flutter 是一个具有移动 SDK 的跨平台框架，它提供了一种响应风格，而没有采用 JavaScript 桥，实现了与其直接竞争对手 React Native 相媲美的性能水平。

它可以与多个平台有效集成，如 Android、IOS 和 Linux、MAC、Windows 和 Google Fuchsia 应用程序。

# 颤振是用来做什么的？

Flutter 是为 iOS 和 Android 开发应用程序的领先解决方案，在这两个平台上使用相同的代码库。

这些应用程序的智能手机版本在苹果和安卓设备上作为真正的原生应用程序运行，并在发布前针对各自的平台进行组装。

它们不需要运行时模块或浏览器。使用单一代码库，您还可以为浏览器创建 web 应用程序，为 Windows、macOS 和 Linux 创建本地程序。

谷歌还将 Flutter 用于谷歌助手的多个模块和谷歌 Home hub 用户界面。一些领先的电子商务品牌，如阿里巴巴集团、易贝和 Groupon，都使用 Flutter 来提供无缝和直观的移动和网络应用体验。‍

# App 开发者为什么选择 Flutter 框架？

Flutter 的流行加速有几个原因。它为开发人员提供了许多优势，保证了更快的开发速度，并有助于交付更加用户友好的移动应用程序。

Flutter 3.0 使您能够为多个平台开发应用程序，如 Android、iOS、Linux、Windows、Mac、web 和 Google Fuchsia。

单一代码库为开发人员提供了无与伦比的生产力，同时使企业能够实现成本和时间效率。

*   **Flutter 加速了应用开发。**

Flutter 提供了许多高级特性，使得[跨平台移动应用开发](https://stackoverflow.com/questions/4127708/cross-platform-mobile-development)更加便捷。使用单个代码库和 UI 引擎进行跨平台开发，使得为 iOS 和 Android 平台生产应用程序版本成为可能。

Flutter 的各种小部件库也为开发人员提供了对预建功能的访问，从而降低了整体编码时间。最终，减少编码时间使您能够实现开发成本效率。

*   **可扩展性**

借助 flutter 框架，开发人员可以无缝地为应用程序添加功能或扩大用户群，而不会造成任何损害。Flutter 在后端支持 Google 的 Firebase 应用程序，具有可扩展性。

Firebase 提供实时数据库更新，保证客户端之间的数据同步更快。Firebase 还优化并减少了同步所需的更新次数，这意味着增加用户功能不会对性能产生负面影响。

*   **代码可重用性**

这是颤振的另一大特点。它允许开发人员创建具有跨平台友好 UI 组件的有吸引力、功能丰富的应用程序，这些组件可以使用大约 90%以上的代码来节省应用程序开发的时间、精力和成本。

*   **Dart 代码的使用**

Flutter 使用面向对象和严格类型的 Dart 代码，作为框架的技术催化剂。以下是 Dart 的一些基本特征，使其成为颤振的必备条件。

*   即时编译
*   JavaScript 桥
*   脚手架
*   消除 XML 文件
*   **MVP 的理想选择**

MVP 代表最小可行产品，指的是为吸引目标受众了解成品需求而开发的测试产品。

对于希望为产品开发筹集资金或希望更快地将产品推向市场的企业来说，考虑到构建应用程序所需的更短时间、精力和效率，Flutter 是开发 MVP 的理想选择。

*   **社区支持:**

Flutter 得到了有影响力的社区支持。25k GitHub 凸显了 Flutter 的人气上升。除此之外，社区支持谷歌本身也可以窥视开发者使用 Flutter 检测到的 bug，这使得它成为一个非常方便的混合应用开发平台。
