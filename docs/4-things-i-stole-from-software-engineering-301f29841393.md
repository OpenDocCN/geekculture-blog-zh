# 添加注释。

如果你在开发软件项目时询问开发人员社区的任何人关于评论的问题。评论总是被认为是一个好习惯。是你给未来自己的一种潜意识奖励。

![](img/ae6f4c54b29be2e96488c5a69077bb76.png)

Photo by [Luca Onniboni](https://unsplash.com/@lucaonniboni?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

在编码时，我们很清楚我们正在构建的逻辑。此外，我们试图编写尽可能模块化和优化的代码。但是当我们在一年后试图回忆我们所写的逻辑时。我们不确定是否能回忆起所有的事情，当然我们可能会错过一些用例。

这些评论就像一个救星。因此，通过适当的注释，我们能够解释特定代码集执行的功能。对于所有那些非软件开发人员，不要把评论误解为评论某人的人格。但基本上是我们写的一些帮助你/他人理解内容的台词

> 注意:评论总是好的。但是软件开发者总是被告知无用注释的缺点。写一个非常明显的注释被认为是反模式。像水这样的东西对你的健康有好处:-)

所以我试着把这个策略偷用到我的日常生活中。每当我创建任何谷歌表，文件等，我总是添加评论，只要有可能。现在，即使我在一年后打开那个文件，仅仅通过阅读你的评论，我就能回忆起我在创建那个文件时的想法。

是不是很厉害？

甚至很多时候，当我创建一个文件夹或下载任何软件，有一些独特的步骤来安装它们。我创建了一个单独的文档，就像 README.md 文件一样。普通的文本文件包含了所有那些我在重新安装时可能会忘记的步骤。

# 遵循一个结构。

在进行一个项目时，我们总是遵循一个适当的结构来保存我们的文件。这样做是为了在项目增长时实现可伸缩性和可搜索性。

![](img/f359d490ac2131b13c929b7b6c2d13c7.png)

Photo by [Dan Freeman](https://unsplash.com/@danfreemanphoto?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

一个类似的东西在我体内自动开发，也就是说，每当我存储任何新文件、文档时，它都遵循一个适当的结构。歌曲等等，因为我讨厌在下载文件夹里乱放东西。

假设有一个税务文件需要保存到我的系统中。所以我创建了一个子类别，即年份。然后，我将那一年的所有相关文档存储到那个特定的文件夹中。

这有助于我在需要任何特定文件时直接进入正确的文件夹。这在潜意识里给了我额外的力量，让我很容易记住我拥有的文件。

但是在最坏的情况下，即使我忘记了一些文件，我也可以简单地进入一个专用的文件夹并在那里搜索。这种方法不仅可以通过存储文件来实现，甚至可以在创建某种 excel 表格时实现。只要为适当的数据创建适当的表，你就能创造奇迹。

# 使用云存储。

在每个人都应该意识到云存储及其有用性的时代。为了给出一个基本的见解，让我更深入地解释一下我所指的云。

![](img/b4b9cf8fe425d70ddacd0e60c0fb1374.png)

Photo by [Denny Müller](https://unsplash.com/@redaquamedia?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

所有的 Google drive，dropbox 和其他在线存储，我们可以把我们的文档，文件，歌曲放入其中，这些都被称为云存储。甚至像 google.com、medium.com 这样的网站也托管在一些服务器上(云)。

通俗地说，云存储可以说是存在于互联网某处的硬盘。任何订阅了云服务并能访问数据的人都可以从任何地方访问数据，只要他们有活跃的互联网连接。

因此，简单地说，我们可以将数据存储在一些云存储中，而不是存储在很有可能丢失的普通系统中。这种通过云存储数据的方式保证了我们的数据可以从任何地方被访问。我们不需要访问任何特定的数据设备。

我尝试遵循的一个类似策略是，我将所有文档保存在一些云位置以及实际位置。就像我这样总是喜欢保持系统有序的人，我建议这是必须的。因为有时由于失误，我在维护系统时删除了一些文件。但是我可以在事后恢复数据。这给了我极大的安心。

# 备份所有内容

就像最后一点一样，将您的数据放在云上始终是必要的，这使您可以随时随地使用它。

![](img/dbf37c73957fe26644126d9a11e10aa9.png)

Photo by [Chris Yates](https://unsplash.com/@chrjy?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

但是我从我的软件工程师之旅中学到的另一个很酷的事实是数据恢复。作为一名软件开发人员，我们被教导说，用户数据是最终的东西，无论如何都不能丢失。

UI、后端、服务器都可以通过额外的努力来构建。但是用户的数据一旦丢失就很难恢复。这就是为什么总是建议以相等的时间间隔将数据库备份到另一台服务器。但愿不会发生什么糟糕的事情，数据可以从备份中恢复。

同样的事情，我也尝试在我的生活方式中实施。我所拥有的任何重要的东西，我总是认为它会丢失。像手机笔记本电脑这样的物理对象可以恢复，但像数据凭证、文档这样的产品可以存储在多个位置。

这就是数字世界的好处。所以我总是建议你也考虑你创建的任何重要文件可以被删除。总是考虑保留这些东西的备份一个外部硬盘笔驱动云存储永远觉得对你来说是正确的。
