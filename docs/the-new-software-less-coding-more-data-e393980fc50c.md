## 现代软件工程

## 阐明软件工程的当前趋势或为什么人工智能比以往任何时候都重要

软件就像一切事物一样在进化，但它的进化与我想象的不同。当我在大学学习计算机科学时，我认为未来是并行的。我们只上过一堂并行编程课。多核计算机正在崛起，这似乎是值得学习的东西。从那以后，我的看法改变了。确实需要并行程序员，但没有我预想的那么多。大多数需要并行编程的库和已经实现的代码都已经实现了。不时需要一些基本的概念。

他们还在大学教授“顺序”(常规)编程。到目前为止，常规编程都是关于给出程序应该采取什么行动的具体指令。每一行代码都包含一个有明确目标的具体指令。程序员编写代码，指出在每一步计算机必须执行哪些动作，而不给计算机留下即兴发挥的空间。所有情况都有这样或那样的定义。如果没有，程序就会中断。对于并行编程来说也是如此，在并行编程中，代码定义了要执行的操作以及执行的顺序。单核和多核的区别在于，单核以顺序方式执行所有指令，而多核可以多次执行不一定遵循相同路径的不同代码部分。在新的范例中，安德烈·卡帕西让我注意到，我也同意，软件是抽象的。软件成为神经网络的砝码，作为人类，我们不能直接解释或编程它。因此，在这些情况下的目标是定义期望的行为。然后，软件开发人员应该定义，对于这些输入集，我们需要另一组输出。为了让程序遵循正确的行为，我们需要编写神经网络架构来从域中提取信息，然后训练它，以便程序在空间中搜索最佳解决方案。我们将不再使用显式代码来解决复杂的问题；相反，机器会自己找出它们。

软件肯定会朝着一个新的方向发展。在许多情况下，收集更多的训练数据可能比实际硬编码算法来执行特定任务更容易。在新的软件时代，编码员的任务是管理数据集和监控系统。该系统经过优化，能够以最准确的方式执行任务。“老派”程序员仍然会被需要，就像仍然有人靠写低级语言为生一样。老派程序员会开发标签接口，创建和维护基础设施，执行分析等。如今，很明显，在许多不同的领域，神经网络是硬编码指令的明显赢家。有了目前的软件，[麦肯锡称](https://www.mckinsey.com/~/media/McKinsey/Business%20Functions/McKinsey%20Digital/Our%20Insights/Driving%20impact%20at%20scale%20from%20automation%20and%20AI/Driving-impact-at-scale-from-automation-and-AI.ashx)30%的公司活动可以自动化。机器翻译、图像识别、文本分析，以及像国际象棋甚至‘英雄联盟’这样需要对宇宙有高级理解的游戏，都可以通过计算机实现自动化。[多亏了 TensorFlow 和神经网络，谷歌将翻译程序的 50 万行代码减少到只有 500 行](https://twitter.com/DynamicWebPaige/status/915326707107844097)。这些是经典的例子，深度学习简单明了，可以大放异彩，但还有其他不太直观(也不太性感)的领域，如数据结构和数据库，可以取得巨大的进步。在[这个不性感出版物的例子](https://arxiv.org/abs/1712.01208)中，深度学习软件比传统软件快了 70%,并且使用的内存少了一个数量级。作为最后一个例子，我想带[这篇文章](https://arxiv.org/abs/1703.01041)在这里，研究人员将这个想法发挥到了极致。他们开发的软件甚至不需要定义要使用的模型，而是为用户找到模型。

总之，我们一直使用到现在的传统软件将会保留。有些情况下，比如法律和医学，暗箱操作是不可接受的，也是不可容忍的。其他时候，硬编码功能会更具成本效益，也更容易，而不是准备训练数据并让模型计算出来。新的软件时代将是这样一个时代，编码人员将不再为每种情况明确地编写行动过程，而是神经网络将为给定的输入找到最佳解决方案。该软件将找到一个给定问题的最佳解决方案，而不需要人类的明确指示。新的范例软件将会变得更加流行，同时，新的软件工具将会被开发出来。
