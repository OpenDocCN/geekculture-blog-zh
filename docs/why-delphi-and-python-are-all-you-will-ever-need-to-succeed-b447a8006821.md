## Delphi 和 Python 是实用的、稳定的和有用的

在实用、稳定和有用的通用编程语言中，有两种已经经受住了时间的考验:Object Pascal (Delphi)和 Python(T1)。

Object Pascal 是软件产品 Delphi 中使用的面向对象编程语言的原始名称，Delphi 提供了一个集成开发环境(IDE ),旨在实现各种软件的快速应用程序开发(RAD)。

对象 Pascal 语言是作为原始 Pascal 语言的扩展而发展起来的。这种对象增强版的 Pascal 语言是由苹果电脑公司开发的，是在 20 世纪 80 年代中期与 Pascal 语言的发明者尼古拉斯·沃斯协商后产生的。

苹果公司特有的 Object Pascal 被广泛用于他们的 Mac 操作系统和应用程序。这种语言足够抽象和“高级”,它可以简化复杂应用程序的开发，同时保留足够的“低级”能力，以允许广泛采用为苹果 Macintosh 软件的首选语言。

后来，Borland Software Corporation 对其中一个 Object Pascal 实现进行了一系列扩展，并在 1995 年 2 月发布了 Delphi 语言和 IDE 的 1.0 版本。

Delphi 很快变得非常流行，并很快确立了自己作为面向对象 Pascal 的主要实现的地位。由于这种广泛的市场认知度，Object Pascal 和 Delphi 这两个名字经常互换出现。

现代编程社区将 Object Pascal 语言的 Delphi 风格和集成开发环境(代码编辑器、编译器和调试器)都称为“Delphi”，在本文的剩余部分，我们将把它称为 Delphi。

从 2015 年至今，德尔福一直由总部位于德克萨斯州奥斯汀的 [Embarcadero](https://www.embarcadero.com/?utm_source=Medium&utm_medium=Leads%20Acquisition&utm_content=WhyDelphiandPythonareAllYouWillEverNeedtoSucceed&utm_campaign=WhyDelphiandPythonareAllYouWillEverNeedtoSucceed) 拥有和分销。

## Python 是一个纯文本发电站

20 世纪 80 年代和 90 年代肯定有一些特别之处。当 Pascal 演变成 Object Pascal，Delphi 准备展示其低代码、可视化驱动的设计时，另一个编程巨人正在崛起。这种新语言被称为 Python，它被证明是与它 90 年代同时代的 Delphi 完全不同的游戏改变者。

Python 是一种通用解释编程语言，由吉多·范·罗苏姆在 20 世纪 80 年代设计，作为他的 ABC 编程语言的继承者。它的正式发布是在 1991 年，名为 Python 0.9.0。

今天，Python 被公认为世界上最流行的编程语言之一。

Delphi 和 Python 在同一时间框架内共同出现，并且都在继续发展，以取得新的技术进步，结合新的编程范例和技术，并适应计算硬件和操作系统的进步。

让我们来看看它们有什么不同，它们的相对优势如何互补，以及它们实际上如何协调工作，将各自的能力增强为大于两部分的综合优势。

## Delphi 和 Python——优势对优势

总的来说，Delphi 比 Python 有更多不同的特性，比它的对手有更多的商业相关性，特别是对于最终用户消费者和企业对企业的市场部门。

Delphi 的“原生平台”，完全编译的代码应用程序更快，更安全，并且抵抗偶然的干扰。此外，Delphi 支持更广泛的执行平台，特别是移动设备。

Python 受益于它的开源特性和相对较低的入门成本，这帮助它获得了广泛的采用。

## Delphi 是完全编译的，Python 是在运行时解释的

编程语言需要将人类可读的源代码转换成可由计算机执行的原始机器指令。

在 Delphi 中，这种转换直接发生在其集成编辑器环境中，编译器生成完全编译好的应用程序，无需进一步的预处理，通常也不需要进一步的支持组件或可安装的运行时。

Python 采用了一种方法，当用户希望执行用它编写的程序或应用程序时，代码需要安装在最终用户自己的计算机上的运行时解释器来完成转换。

运行时解释速度很快，但仍然比不上已经预先编译好的程序，因此，用 Delphi 编写的程序运行速度明显快于 Python 应用程序。

## 程序的源代码通常是商业秘密

Python 程序要么以纯文本文件的形式交付，要么以部分预编译模块的形式交付。这两种形式都很容易检查——对于文本源文件，任何人都可以查看它们，根本不需要任何特殊工具。

在许多内部商业和家庭场景中，Python 程序的源代码易于检查可能不是问题，事实上，对于用户需要或希望定制 Python 程序的行为或结果的用例类型来说，这可能是一个显著的优势。

一般来说，对于一个旨在产生利润并以家庭用户或商业市场为目标的商业程序来说，程序的内部工作被认为是“知识产权”,并被严格保护以免被泄露。

对于涉及安全性或有价值的私有项目(如访问密钥和令牌)的场景，确保程序的源代码保持其纯粹的完整性通常是绝对必要的——在这两者中，只有 Delphi 可以确保对这种不折不扣的保真度有最高的信心。

容易修改和减少未经授权的修改(如黑客的行为)之间的平衡在很大程度上取决于 Delphi 和 Python 两者中哪一个最适合所需解决方案的要求。

## Python 和 Delphi 语言的机制有什么不同？

Delphi 是静态类型的编程语言，而 Python 是动态类型的。

这意味着 Delphi 的代码在编译时是“类型检查”的，但是对于 Python，这发生在运行时。

Delphi IDE 结合静态类型有助于在最终用户使用程序之前识别代码中的潜在问题。

静态类型的代码执行通常更快，因为在程序运行时不需要识别和检查变量类型，从而允许进行优化，这在 Python 的运行时解释中是不可能的。

一些开发人员可能仍然更喜欢 Python 的动态类型特性，因为它为限制较少的代码结构提供了一些好处。

## 缩进:最大的区别

两种语言对实际代码的布局采取了略微不同的方法，但是可能一个主要的区别是 Python 和 Delphi 代码块的缩进方式。

对于代码缩进，Python 要求严格的一致性，因为它是语言语法的一部分。

例如，在 Python 3 中，如果最初使用制表符而不是空格进行缩进，则必须在整个代码中保持这一点。“空白”缩进的级别是 Python 程序代码的操作所固有的。

相比之下，在 Delphi 程序中，缩进的形式和空格或制表符的数量对代码的语法没有相关性或直接影响。

最后，虽然 Python 的代码区分大小写，但 Delphi 的代码不区分大小写，这有助于 Delphi 开发人员避免几种形式的与大小写相关的错误，这些错误很难诊断和修复。公平地说，Python 的大小写敏感性可能是 Python 开发新手的主要障碍。

## 哪些设备平台最适合 Delphi，哪些最适合 Python？

当谈到它可以瞄准的平台范围时，Delphi 是非常通用的。它允许从 Windows 环境开发 Windows、macOS、Linux、iOS 和 Android 应用程序。

Python 运行时解释器广泛适用于大多数平台。

Python 常用的 ide 包括 Pycharm、Visual Studio Code 和 PyScripter，它们可以在更广泛的平台上用于实际代码的初始开发。

然而，与 Delphi(和 RAD Studio)不同，这些 Python 兼容的 ide 不能为 Android 和 iOS 生成本地应用程序，尽管可能有第三方解决方案来打包 Python 代码以便在这些平台上使用。

## 开源 Python 和 Delphi

Python 是一种开源语言，允许免费公开和自由地检查、使用、修改和分发。

Delphi 的编译器和 IDE 是商业封闭源码；然而，Delphi 运行时库的源代码是随产品一起提供的，也可以被免费检查和用作学习资源，或者被修改以提供附加功能。

Delphi 提供了一个免费的社区版，允许创建开源项目，并为学术和研究机构提供定价方案。

## 运行时间开销

Python 的解释步骤会导致移动设备上更显著的内存使用和更高的电池消耗。

除了可执行包的初始大小之外，Delphi 的内存开销要低得多。Python 程序。py 文本文件通常从很小开始，因为整体大小只有在解释后使用 Python 程序的运行内存映像才能真正实现。这种整体上更大的大小主要是由指令的小文本文件扩展成标记化的形式，然后转换成实际的内存中的“真实”代码。

## Delphi 可能是原始的低代码系统

Delphi 优秀的 GUI 设计工具允许开发人员“绘制”和布局他们的应用程序的屏幕，而不需要任何额外的原型或线框工具，以使快速开发和准确遵守客户的可视化设计文档成为可能。

这些工具被集成到开发环境中，意味着零时间浪费，因为不涉及切换或加载时间。

在这些工具的帮助下，生产原型的时间大大缩短了，甚至可以完全避免。

Python 在很大程度上是一种纯代码开发体验，在这种体验中，程序以长格式作为文本输入到编辑器或 IDE 中。

德尔福的生态系统更加以商业为中心，许多专家在丰富其功能和传播其功能方面发挥着作用。另一方面，Python 的生态系统更倾向于广泛的业余爱好者、业余爱好者和 IT 专业人士。虽然这使得 Python 对新手和中级开发人员更加开放和包容，但对大型项目的高级支持可能是有限的。

## 易用性和最佳使用案例

总的来说，大多数开发人员似乎认为 Delphi 和 Python 都很容易使用。然而，对于那些从其他静态类型语言过渡过来的人来说，学习 Delphi 可能更容易。总的来说，使用的目的决定了一个人选择这些语言的舒适程度。

所以让我们深入研究一下。

Delphi 更适合于本地(x86/x64)桌面和移动(ARM)应用程序。它提供了带有交互式按钮、菜单和图标的图形用户界面(GUI)应用程序的快速而简单的开发。

Delphi 也是数据库开发的最佳选择。它可以访问所有主要的数据库，包括 Oracle 和 SQL Server，很少或不需要额外的库和大量的支持库下载。

在处理大型数据集和统计数据时，Python 很受欢迎。

这种专业化理所当然地使其成为数据科学界的宠儿。

还有许多 Python 分析库，它们极大地简化了人们处理数据的工作。

作为一种脚本语言，Python 在处理一个或多个相对小规模的特定或“快速而肮脏”的编码活动时表现良好。

当然，这使得它与许多行业、企业和个人用户直接相关，因此它很受欢迎。

由于其快速原型、商业支持和快速开发，Delphi 在处理大规模应用程序时占了上风。

Delphi 应用程序能够运行许多年是很常见的，经历了许多操作系统升级和安全补丁，继续按设计运行而不受影响。

对于 Python 程序和其他语言(如 C#)来说，情况并非如此，它们在运行环境中很容易受到这种变化的影响。

因此，也许与 Delphi 的企业级使用和广泛采用相关的最关键的因素是对代码维护的需求很少。

Python 在与一些 API 接口方面有特别的优势，尤其是一些人工智能和机器学习 API 即服务系统。
