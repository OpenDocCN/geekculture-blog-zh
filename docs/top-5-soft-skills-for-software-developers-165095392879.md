# 什么是软技能？

软技能是非技术技能。它们包括你如何与同事互动，如何解决冲突，如何安排工作的优先次序，以及如何管理自己的情绪。

# 为什么软技能对你很重要？

几乎所有的开发工作都需要开发人员以某种方式与他们的同事或客户接触。你可能在技术技能方面很优秀，但如果你不能与你的同事很好地沟通，如果你不能很好地管理你的时间，那么它不会帮助你超越你的职业生涯。

从经理和人力资源的角度考虑新的招聘或晋升。如果他们必须在以下选项中做出选择，他们会做出什么选择？

*   一个非常有技巧的候选人/员工，善于沟通，看起来情商很高，而且非常讨人喜欢
*   同样有技能的候选人/员工，反社会，缺乏软技能

选择是如此明显。所以仅仅拥有技术技能是不够的。拥有软技能很重要，这样才能被录用，然后在职业生涯中脱颖而出。

# 软件开发人员的五大软技能

让我们来看看软件开发人员的 5 大软技能是什么:

# 1.沟通

![](img/0ecb3293109e91c079b0b8ae2d1fd862.png)

Communication

作为一名软件开发人员，你可能会花几个小时独自完成一项开发任务。但是你很少独自编写整个系统的代码。大多数时候，你作为团队的一部分工作，团队成员包括业务分析师、质量保证工程师、scrum masters、项目经理、代码审查人员、技术经理和其他利益相关者。

对于软件开发人员来说，有两种沟通技巧非常重要:

*   口头交流:软件开发人员必须一直倾听在不同会议上传达给他们的开发需求:待办事项整理会议、每日站立会议、QA 会议、回顾会议等等。因此，你能够有效地倾听并理解这些讨论是很重要的。与此相关的是提出正确的相关问题，对建议的解决方案给出反馈，甚至提出更好的解决方案。这一切都需要优秀的语言沟通能力。
*   **书面交流:**作为一名软件开发人员，你也必须进行书面交流。做技术文档，在 confluence 上合作，阅读和回复 Slack 或团队渠道消息，回复电子邮件等等。拥有良好的书面沟通技巧可以确保人们理解你真正想说的话。

# 2.时间管理

![](img/7ee81a69737ee08859ebf1b94cc5c18e.png)

Time Management

您正在参加 sprint 计划会议，并被要求对您的任务进行时间估计。你用你的经验和最好的判断技能来估计时间并分享时间，现在时钟开始滴答作响。

为了实现你的目标，你需要合理规划你的一天。试着把你的任务分成更小的任务，用**番茄工作法**来计划你的一天。

技术很简单:

*   把你的任务分成更小的任务
*   将你的一天分成短时间的增量和周期性的休息
*   工作 25 分钟，然后休息 5 分钟
*   每 25 分钟的工作时间被称为“番茄工作法”
*   在 4 次番茄大战后，休息 15-30 分钟。

这将有助于你更好地专注于你的工作，也将避免筋疲力尽。

当你在完成任务时，在冲刺阶段的每一天都会有常规的中断。某个同事在他的任务中需要您的帮助，您的经理在其他一些事情上需要您的专家意见，某个软件的其他部分出现了问题，DevOps 需要您帮助进行热修复等等。

*   总是尽力帮忙，但避免不必要的干扰
*   设定专注的编码时间，尽量避免被打断
*   如果你陷入困境，请不要犹豫寻求帮助
*   如果你是一个团队领导，可以授权，那么就去做吧

# 3.有责任

![](img/d8d59c3428c65a496766494d51c8a10e.png)

Accountability

责任就是对你的错误负责。错误是软件开发不可避免的一部分，所有的软件开发人员都会犯错误；小调还是大调。

处理错误有两种方法。

*   **隐藏自己的错误:**当你隐藏自己的错误时，说明你不成熟，没有诚信，不虚心学习，还会再犯类似的错误。在这种情况下，因为你隐瞒了你的错误，所以不能及时采取纠正措施。你对整个软件的工作和错误的影响没有更广阔的视野，所以即使是你认为很小的错误，最终也会给组织带来很大的损失。
*   **接受你的错误:**当你接受你的错误时，这表明你成熟了，有诚信，乐于学习，并想把你的组织从代价高昂的错误中拯救出来。

所以你应该承认责任，而不是隐藏和逃避发生的事情。利用这个机会后退一步，分析哪里出了问题，如何解决，最重要的是，如何在未来防止出现问题。与你的同事分享这个结论，作为一个学习的机会，这样可以避免类似的错误。

# 4.协力

![](img/d60a95b765cbb561cec89c1e924483ac.png)

Team Work

大多数软件开发人员是作为团队的一部分工作的。在一个团队中，并非所有成员都有相同的个人特质。有些很合理，很好相处，有些很难相处。有时，你可能在某些事情上与你的团队成员意见一致，也可能有你不同意他们意见的情况。因此，你应该具备软技能和个人特质，比如耐心、开明和同情心，来处理这种情况。

团队合作的理念是与他人合作愉快，并在你的方法中找到平衡。

# 5.神入

![](img/bf1562d6c2da6f7980e53379ef66d8c4.png)

Empathy

同理心是你理解队友感受的能力。当你感同身受时，你把自己放在队友的位置上，试着感受他们的感受。

让我们举几个例子:

*   **同情非 IT 同事:**你的组织中有非 IT 同事，你正在和他们谈论某个项目。那些同事不懂编码术语。因此，如果你使用这些术语，他们将无法跟上整个讨论。出于同理心，你会尽量不使用编码术语，而是使用简单的词语，这样他们就可以很好地参与讨论。
*   **同情你的队友:**有一些初级开发人员，他们在代码库方面需要你的帮助。感同身受，你会试着把自己放在他们的位置上，看看在类似的情况下你会希望得到怎样的对待。所以你可以通过提供代码库的文档，解释代码的逻辑，或者任何你能做的其他帮助来帮助他们。
*   **对软件用户感同身受:**在编写软件时，你会把自己放在软件用户的位置上，试图了解“你”会如何使用软件。感同身受将有助于你交付一个更好的软件，用户会喜欢，因为你已经考虑了他们的需求。

软件开发人员通常会忽略软技能。我希望这份基础软技能指南能对你的职业生涯有所帮助。
