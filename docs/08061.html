我希望在我的软件工程生涯中早点知道的21课。

原文：https://medium.com/geekculture/21-lessons-i-wish-id-known-earlier-in-my-software-engineering-career-751b2158e8e1?source=collection_archive---------0-----------------------#2021-10-13

学习编程很难。6年前，当我开始我的web开发之旅时，我觉得自己好像要辞职了。

但我在科技行业生存了下来，现在我热爱它。以下是我希望在职业生涯中早点知道的21个教训。

{ 1 }选择一种语言

有令人难以置信数量的编程语言。但是，问题是，你需要慢慢开始。

一次挑一件事，尽可能多的练习。然后，你可以继续下一个。

{ 2 }理解编程概念

学习一门新的编程语言时，你可能会想抄近路。虽然使用谷歌来寻找答案是可以的，但是不要记住语法。

如果你不记得语法没关系，但是记住概念是必须的。

{ 3 }使用在线教程

这是一个救命稻草，学习新技能比以往任何时候都容易。为什么要用教程？

—你获得多种视角
—没有时间限制
—它帮助你更快更有效地学习

{ 4 }在您的日历中每天安排几个小时用于学习

每天学习没那么难，但是一开始很难做到。每天学习都需要很大的奉献和自律。

—阅读博客
—观看视频
—不断学习新事物，不断建设

{ 5 }练习，练习，练习！

构建辅助项目或演示项目是精通编程的捷径。

俗话说“熟能生巧”是非常正确的，如果你愿意花时间去做某事。

{ 6 }不要放弃

一旦你开始你的旅程，事情可能不会对你有利。成为优秀程序员的一个关键是永不放弃。

你会惊讶地发现你能如此迅速地掌握最新的概念。任何事情的开始总是艰难的。

{ 7 }不要重新发明轮子

除非你需要保持事情简单，利用库来帮助你简化工作。

并非每个解决方案都需要从头构建，请密切关注不同的包和库。

{ 8 }学会使用命令行

知道这一点非常有用，而且实际上比大多数人想象的要简单。虽然一开始看起来很复杂，但是一旦你掌握了窍门，就很容易了！

一旦你掌握了它，就没有回头路了。

{ 9 }提高你的软技能

拥有良好的编程知识是好事，但如果你掌握了软技能，你将直接跳到前5%。

练习写作，从长远来看对你有帮助。掌握软技能会帮助你脱颖而出。

{ 10 }构建强大的项目组合

展示你的项目和工作是吸引注意力的最佳方式。

学习+建造东西，用这些战术你很容易脱颖而出。这是双赢的局面。

{ 11 }记录您的旅程

在推特上分享你的学习，并从他人那里获得灵感。

写博客今天有许多人正在开始他们的旅程，你肯定会激励他们。

{ 12 }边学边赚

我在学习无代码工具时赚了750多美元。以下是你如何赚取你的第一美元。如果我能甚至你也能:

{ 13 }定义小目标

不要从每一种编程语言和概念开始。

一次做一件事，定义小目标，粉碎它们。这种方法是最终的信心助推器。设定提高能力和生产力的目标。

{ 14 }没有单一的路径

在学习的时候，不要试图走某条路，你可以有自己的路。

有些东西对你可能不适用，但对其他人可能适用。专注于你已经计划好的事情，而不是担心其他人。

{ 15 }不要害怕寻求帮助

在学习过程中，如果有任何疑问，请联系你的朋友或导师。即使你有愚蠢的问题，也不要不好意思问。

这可能是解决你的疑惑并继续前进的最快方法。

{ 16 }挑战自我

不要和别人比，和自己比，看看自己随着时间的推移进步了多少。

与他人比较会让你失去动力，并在你的道路上设置很多障碍。

{ 17 }学习基础知识并构建小型应用程序

你构建的小应用越多，你的基础就越强大。

如果你更专注于构建例子来更好地理解概念，你实际上是不可阻挡的。

{ 18 }拒绝负面言论

不是所有的日子都是一样的，有些日子你可能会因为事情对你不利而想要放弃。

休息一会儿，重新振作起来。记住程序员从不放弃，总会有解决方法。

{ 19 }请记住，没有人天生就是专业人士

只要坚持不懈，努力工作，你就会成为职业选手。继续全力以赴。

一旦你在一段时间内不断重复做同样的事情，你很快就会掌握它。

{ 20 }除非你去做，否则什么都不管用

只看文档，看视频，会让你一事无成。研究你读过的任何东西，并有一个坚实的理解。实践使人完美。

{ 21 }关注过程，而不是结果

让它变得有趣不要太担心未来，会有许许多多新的库和框架出现。

如果你基础扎实，剩下的就容易破解了。享受旅程，保持乐观。
