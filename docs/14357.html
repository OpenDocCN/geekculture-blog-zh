  • 新西兰网络弹性框架认证

什么是新西兰网络弹性框架？

“这是一个框架或指数，我们正在寻求开发，可用于私人、公共和个人。这不是一个网络安全框架，你有一个勾框，你去，哦，我需要做这些步骤来多一点，”洛佩兹说。

文章进一步解释道:

CERT NZ已经开发这个框架大约一年了，在最初的研究中，它还没有在国际范围内发现任何类似的东西。目前只有基于公共信息的索引或系统可用，没有任何东西像政府机构想要开发的那样全面。洛佩兹说，那里存在的许多东西也是相当技术性的。他们不一定考虑网络攻击对个人和社区的社会影响。

CERT NZ正在寻求帮助，这对希望参与上述框架开发的专业人士和组织来说是一件好事:

对于认为该框架有用的企业和行业，Lopez要求他们联系CERT NZ。她表示，该机构目前正处于寻求理解问题所在以及如何整合框架或指数的阶段。CERT NZ希望它对任何人都有用，无论是个人还是企业，以了解他们的行业中是否有特定的弱点或优势。

  • 威胁参与者正在使用新发现的技术来在受损环境中保持持久性。被MSTIC发现后，他们将其命名为“MagicWeb”

微软安全研究人员发现了一种我们称之为MagicWeb的危害后功能，我们跟踪的威胁参与者NOBELIUM使用这种功能来保持对危害环境的持续访问。

MagicWeb的工作原理:

MagicWeb是一种危害后恶意软件，只有在获得对环境的高权限访问并横向移动到AD FS服务器后，威胁参与者才能部署它。为了实现他们的目标，即通过验证AD FS服务器上任何用户帐户的身份验证来保持对环境的持久访问，NOBELIUM通过复制AD FS操作中使用的合法Microsoft.IdentityServer.Diagnostics.dll文件创建了一个后门DLL。

如何缓解:

将您的AD FS服务器视为第0层资产至关重要，要像保护域控制器或其他关键安全基础设施一样保护它们。AD FS服务器为配置的依赖方提供身份验证，因此获得AD FS服务器管理访问权限的攻击者可以完全控制对配置的依赖方(包括配置为使用AD FS服务器的Azure AD租户)的身份验证。实践凭证卫生对于保护和防止高特权管理员帐户暴露至关重要。这尤其适用于更容易受到攻击的系统，如带有登录限制等控制的工作站，以及带有Windows防火墙等控制的防止横向移动的系统。

建议迁移到Azure Active Directory (Azure AD)身份验证，以降低本地危害横向迁移到您的身份验证服务器的风险。客户可以使用以下关于迁移的参考资料:
