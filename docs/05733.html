为什么我决定学习计算机科学

原文：https://medium.com/geekculture/why-i-decided-to-study-computer-science-68fc5f601281?source=collection_archive---------49-----------------------#2021-07-28

毕业生在申请初级工作时经常被问到的一个重要问题是“你为什么选择你的研究领域？”。现在我已经是大学的最后一年了，我正在为毕业生的工作面试做准备，我意识到我以前没有真正考虑过这个问题。学习计算机科学对我来说似乎是一种天赋。这是我想做的，但我从未真正停下来思考过，“为什么？”。

为了开始我的回答，我回到了我第一次与编程的互动。早在高中(初中)，我被介绍到Scratch。Scratch是一种高级编程语言，是为孩子设计的教育工具。代码是通过“拖放”块编写的，偶尔会输入诸如循环和if语句之类的内容。

显然，14岁时，我和Scratch的目标不是创造下一个脸书。相反，我想做一个视频游戏，我可以玩，给我的朋友看。在使用Scratch和观看一些教程的第一个小时左右，我已经完成了一个非常基本的游戏。

这个游戏的目的是让一个角色通过我在谷歌图片上找到的2D迷宫。然而，我随后决定，我希望我的主角是动画。所以，又过了一个小时，我看了一些youtube视频，并想出了如何给我的主角制作动画。但我仍然觉得我的游戏可以更酷。我想要多个关卡，一个boss战，一个主菜单…到最后我可能花了五个小时左右的时间在我的迷宫游戏上，当时，我认为它非常完美。

现在回头看看这个游戏(实际上你仍然可以玩这个游戏)，它没有我记忆中的那么完美，但我可以有把握地说，这是我走向计算机科学学位的那一刻。我真的为此感到骄傲。这是我想展示给人们看的东西，并说“看！我做了这个”。

我想我也发现了仅仅通过运行游戏就能立即看到我的代码是正确还是不正确的过程，真的很令人满意。这肯定比写论文要容易得多，因为我的老师要找的答案并不那么明显。

当时，我记得我在想，“如果我能每天都做这样的东西，并为此获得报酬，那不是很好吗？”。然而，这仍然不能完全回答问题。如果我那么喜欢做那个游戏，为什么不去学游戏开发？为了回答这个问题，我提前了一点到了高中的最后一年。

到这个时候，我已经参加了高中开设的所有数字技术课程，并且从各种事情中获得了很多乐趣。从Corona SDK中的简单android应用程序到远程控制，arduino动力坦克由激光切割丙烯酸树脂制成。我认为这些数字技术课程真的让我大开眼界，让我认识到编程和软件开发总的来说比仅仅制作视频游戏有更多的乐趣。

然而，真正决定性的时刻是在我在学校咖啡厅做兼职咖啡师的时候。

我们学校的咖啡厅很受学生和老师的欢迎。每天早上，在上学前或30分钟的早间休息时间，我们会看到学生和老师排着长队等着喝咖啡。事实上，咖啡馆非常繁忙，我们不得不经常拒绝顾客，这样我们的学生员工才能按时上课。

虽然咖啡馆的目标是培养学生成为咖啡师，但它的运营也很像商业。为了继续购买咖啡豆，购买杯子，并支付我们的培训师/经理，我们真的不能每天拒绝这么多顾客。这让我思考“我们能做些什么来加快我们的服务？”。

我们的咖啡馆用积分卡奖励回头客，每买五杯咖啡就送一杯免费咖啡。这些卡片使用邮票进行标记，并保存在现场，以确保客户不会添加自己的邮票。这意味着为了在顾客的卡片上盖章，我们必须记下他们的名字，并把卡片放在一个按字母排序的盒子里。

如果你能想象大多数顾客(在一所学生人数约为2000人的学校)都有一张这样的卡，你就能想象这个忠诚卡盒子有多满了。为每位顾客筛选数百张卡片的过程，即使我已经知道他们的名字，也可能给每笔交易增加了几分钟的时间。

这让我想到，如果我能消除对那个盒子的需求，同时确保客户不会只是给自己更多的邮票，我就能更快地为每个客户服务。因此，我将能够服务更多的顾客。正是在这里，我产生了将我在数字技术课上学到的知识与我的兼职工作相结合的想法。

我的想法是建立一个渐进式的网络应用程序来数字化这个积分卡系统。这种运作方式是，咖啡馆将有一个商业版的网络应用程序，他们将使用它来代替邮票和一盒卡片。另一方面，顾客将拥有他们自己版本的网络应用程序，以取代他们的实体会员卡，并允许他们在任何时候(不仅仅是当他们在咖啡馆时)跟踪他们下一次免费咖啡的进展。

在此之前，我在构建渐进式web应用程序或一般的web开发方面经验很少。因此，我决定使用React来开发我的解决方案。React是由脸书开发的JavaScript库。React有许多我在开发这款应用时学到的优秀特性，但我选择React的主要原因是:

  1. 这是一个很受欢迎的库，所以有很多可用的包和教程。
  2. 我可以用它来开发一个渐进式的网络应用程序，可以在IOS、Android和网络桌面上运行，而不需要开发者许可证。
  3. 我知道Firebase(Google开发的一个移动平台，支持免费托管和实时JSON数据库)可以很好地工作。

虽然一开始令人畏惧，但我实际上非常享受学习React的过程，学习它是如何工作的，以及如何使用它来实现我的想法。一旦我制作了一些基本的应用程序来测试一些特性，比如生成独特的二维码，并弄清楚如何将我的应用程序部署到firebase，我就准备好开始我的主要实现了。

我首先用UXPin制作了一些UX模型。我把这些拿给我的老板/导师，以检查他是否喜欢这个应用程序的外观，以及他是否真的能够使用它(稍后将详细介绍)。这是我真正喜欢的软件开发过程的另一部分。定期与我的“客户”互动，以确保我们对应用程序的外观和功能的愿景保持一致。

正是在这一点上，我开始意识到我的项目的意义。与我的迷宫游戏或数字技术项目不同，我要做的实际上是解决人们生活中的一个真实问题。如果我的想法成功了，我不仅会为我所做的事情感到骄傲，还会为我对同学和同事的生活产生了切实的积极影响而感到骄傲。

快到年底的时候，我已经基本完成了我的实现(实际上它还在我的 GitHub 上)。我已经使用了我的实体模型中的UX设计，在家里测试时，我发现我所有的原始需求都正常工作。我添加了一些功能，如Google OAuth来验证用户和异步firestore，这样客户忠诚度的变化就会实时更新。总的来说，我觉得这是准备实施到咖啡馆。

不幸的是，在这里我了解到了软件开发的一些困难，以及正确测试的重要性。

虽然我的应用程序在我的家庭网络和移动数据上工作没有问题，但学校无线网络的不一致性和较差的覆盖范围经常导致应用程序无响应和不可用(特别是在咖啡馆周围)。此外，虽然我的老板/导师最初说我描述的使用应用程序的方式是理想的，他使用它不会有问题，但当把成品交给他时，他对技术的了解远低于我的预期，甚至连应用程序的基本功能都很难掌握。除了这些问题之外，我的老板/导师明年就要离开学校了，学校咖啡厅是否还会继续存在还不确定。

因此，我的应用程序最终从未被咖啡馆或我为之开发的学生使用过。虽然这相当令人失望，但我认识到，下一次，我需要“在现场”测试我的解决方案，并与实际的目标用户一起，在整个开发过程中定期这样做(不仅仅是在结束时)。我还学到了很多关于web开发和为真实世界的客户开发真实世界的解决方案的知识(相对于高中和大学经常给的模拟任务)。回顾整个项目，这仍然是一次非常积极的经历。

尽管我的应用没有实现，但我的软件可以解决现实世界问题的想法一直困扰着我。我觉得，虽然用软件解决问题可能不如制作视频游戏那么“有趣”，但从长远来看，这对我个人来说更有价值。这并不是说我将来不会从事游戏开发，而是意识到为什么我选择了计算机科学的学位而不是从事游戏开发的研究。

那我该怎么办？如果在面试中被问到，我现在可以自信地说出我选择我的研究领域的地点、时间和原因，但我认为还有更多。

一位前毕业生招聘人员曾告诉我，为了在职业生涯中取得成功，你应该努力找到一份你的价值观与雇主的价值观以及他们期望你担任的角色的价值观相一致的工作。由于还没有从事过软件开发，我想我还不能准确地说出我在职业生涯中看重什么。

然而，在找到我对“你为什么选择你的研究领域”这个问题的答案的同时到目前为止，我已经回到了这个领域中让我感兴趣和激励我的根源。

我想现在，我开始对我的价值观有了一些了解。
