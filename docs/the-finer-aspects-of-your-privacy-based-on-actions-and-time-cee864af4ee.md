我从不相信人们的断言，我总是根据他们的行动来判断他们。—安·拉德克利夫

[早些时候](/geekculture/can-differential-privacy-hide-your-love-for-hawaiian-pizza-3b5cf78612d9)，我已经谈到了差分隐私(DP)如何从属于所述数据记录的人的存在中分离获取的信息。**默认情况下，我们假设差分隐私将为调查受访者或服务用户提供可信的否认，即他们不是记录数据的一部分。**这种可否认性是一种强有力的保证，确保用户会毫不犹豫地分享他们的信息，作为未来分析的丰富资源。毕竟，DP 将数据世界划分为两个平行的现实。在一个例子中，某个特定的用户存在于记录中，而在另一个例子中，他们不存在。

我们想象中被矿工和分析师觊觎的用户数据(兴奋地摩拳擦掌)不仅限于包含人及其属性的记录。数据集还可以包含更细粒度的用户属性，比如他们的行为或长期行为。还记得一个人的公共行为是如何通过巧妙的算法得出结论的吗？那么，如此重要的东西难道不应该受到像 DP 这样酷的概念的保护吗？好消息是，是的，这是可能的。让我向您介绍一下 DP 提供的不同粒度的隐私:

*   **行动级隐私**:用户在社交媒体上的存在通过他们的反应、评论和他们可能与联系人分享的 URL 来标记。因此，这样的平台会记录一个人的行为，而不是他们的属性。想象一下，你正在浏览脸书或 Instagram，你喜欢一个关于吉卜力工作室艺术的帖子，然后你在一个关于失败旅行计划的 meme 上标记了一个朋友，然后你看到了这款酷手表的广告；点击那个广告会把你带到出售它的门户网站。但是你不确定它戴在你的手腕上是否好看，所以你通过一些外部应用程序与另一个朋友分享它的链接。这个跨越十分钟的动作序列可以成为记录的一部分，也称为数据集。虽然这种动作中的重复趋势可以形成用户行为的指示符，但是 **DP 可以再次向用户提供对单个动作的似是而非的否认。例如，现在会有两个平行的宇宙。在一个现实中，你不属于嘲笑计划的那一类人——宠坏了，令人失望的朋友，而在另一个现实中，你是那些给予应得嘲笑的人之一。**
*   **周(或月，或年)级别的隐私**:现在我谈论的是在一个时间范围内打上时间戳的行为，这个时间范围可以是一周、一个月或一年。**对于月级别的隐私，这两个数据集的区别仅在于特定月份内发生的用户行为。**例如，在你妈妈生日的前一个月，你可能会看到很多与访问珠宝网站、论坛上的恐慌讨论、访问电器供应商网站、分享 URL 以及最终购买珠宝有关的用户操作。因此，一个月级别的粒度 DP 可以让你否认在某个特定的月份里，你可能讨论了很多送礼的想法，也可能没有。

现在，我们已经在用户层面上建立了差分隐私的数学公式。校准噪声的增加由两个参数ε(ε)和δ(δ)决定。简单来说，ε的指数决定了 DP 创造的两个平行世界的差别。**当我们谈论动作级隐私保证时，它类似于用户级粒度，但是 N 次，其中用户采取了 N 个动作**。因此，ε = 0.001 的动作级 DP 的基于 DP 的计算类似于ε = 1 且用户已经采取 1000 个动作的用户级隐私，但是弱于用户级隐私。因此**保证会随着动作的数量**而降低。例如，它为任何采取了最多 2000 个动作的用户提供了类似于ε = 2 的用户级隐私的保证，为采取了最多 3000 个动作的用户提供了类似于ε = 3 的用户级隐私的保证，等等。由于系统不能限制任何用户的动作数量，因此这种保证的弱点对于高级用户来说变得更加明显。

虽然使用数字的效果对动作级别的保证有很大的影响，但是即使我们假设 99%的用户采取了特定数量的动作，仍然会有更多的陷阱。**通过添加虚假记录进行的数据操作会削弱行动层面的隐私保障。**例如，一个包含 100 万用户操作的数据集有 90 万个虚假用户。与真实用户相比，假冒用户所采取的行动的数量微乎其微。由于低阶ε给出的大大削弱的隐私保证，真实用户的行为因此将是明显可见的。现在，您可能会说，让我们将操作的数量截断为 1，000，以实现一致性。由于用户级隐私更强，我们也来应用一下吧！由于我们也对高级用户进行了限制，这样的限制会带来偏见。这让我们接受了一个事实，即在所有情况下，**用户级隐私是更强有力的保证**，但这通常会以降低数据发布的准确性为代价。
