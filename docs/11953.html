世界各地的公司都在争先恐后地在工作场所实施人工智能。从自动化的性能评估到人工智能写作助手，人工智能完全融入管理堆栈只是时间问题。

为什么？因为AI集成公司是更好的。他们生产更多的商品，更具前瞻性，对市场变化反应迅速。

不要让即将到来的技术变革占了你的便宜——利用即将到来的技术变革，而不是。这里告诉你如何在工作场所实施人工智能，几乎可以在一夜之间发展任何一家公司。

为什么要在职场中使用人工智能？

现代人工智能的效用正在爆发，很大程度上是因为它的效用。在许多任务上，人工智能的输出现在与人类输出相似，甚至更好:写作、设计、编程、数据分析等等。大多数人还没有完全理解其中的含义。

这意味着我称之为生产力过剩的存在，存在显著提高产量或性能的能力，但还没有人利用它。如果你能成为拥抱这项技术的前瞻性思考者之一，你将收获巨大的回报。

这里有一些具体的方法，你可以利用生产力过剩，在工作场所应用人工智能:

改善沟通

双方之间交换文本信息的任何地方都充满了人工智能优化的潜力。

目前，您可以从使用现代自然语言处理的要点式“精化”提示中获得大部分这种功能(为了简单起见，以下所有示例都将使用 OpenAI API )。

简而言之:

  • 你不再需要担心表达思想的艰巨任务
  • 相反，用简短的要点来描述你的想法或想说的话
  • 人工智能将为你编写长格式的交流，对你的要点进行排序，并插入单词以填充和格式化输出

销售和营销邮件占据了公司大量的时间。内部邮件也是如此——你参与过多少15封邮件的抄送链，花了一天的大部分时间去处理？大多数工作场所可以将员工花在电子邮件上的资源减半，让他们专注于可以显著提高他们利润的事情。

节省时间/成本:电子邮件节省50%或更多。

提高设计效率

艺术与设计是一个迅速崛起的人工智能行业，将在未来几年内完全自动化。如果你读过我的上一篇文章，你就会知道这是多么的接近。总结一下:现在有人工智能可以在15秒内一个像素一个像素地创造出几乎在所有方面都优于人类质量的设计。那为什么没人用呢？

答案是:巨大的生产力过剩，加上行业对先进技术的厌恶。创造力是人类自我认同的核心；设计部门不喜欢知道或承认他们的工作是可复制的。这还需要一段时间才能全面加速，但是任何现在实现它的人都可以远远领先于他们的竞争对手。

方法如下:

  • 在LAION-400这样的大规模数据集上训练的开源模型可以创建图像，设计师稍后可以将这些图像放入Photoshop或Procreate进行增强。
  • 在确定一个可行的长期解决方案之前，要么使用少数可用的拥抱面部空间，要么为每个设计师注册Colab Pro，并让他们使用像这样的笔记本(花几个小时玩参数)。
  • 这将产生一个256x256的图像。您可以使用像 SWINIR 这样的模型放大(提高分辨率)图像结果。注意:在撰写本文时，SWINIR是第一名，但在接下来的几个月里，这种情况可能会发生变化。
  • 您的设计师现在拥有大约1，000 x 1，000像素的图像，可用于徽标、广告设计、品牌、汽车包装、文化基因、艺术佣金等。你可以使用像Topaz Gigapixel这样的大型系统进一步升级，尽管这可能会导致质量下降。
  • 将你的设计师变成编辑而不是纯粹的T2创作者有助于为每件作品节省几十个小时，并且比你的艺术家必须从头开始创作每件作品时给你更大的风格多样性。

节省时间/成本:设计时间节省50%或更多。

视频旁白

贵公司录制了多少视频，是内部的还是为顾客/客户录制的？有了声音克隆技术，你现在可以高保真地重建人们的声音，这可以用来修复录音错误，创建新的内容，或者发送个性化的信息。

声音重建是营销部门的一个梦想。你与终端客户的交流越个性化，转化率就越高。想象一下定制的语音提示，说话者实际上说的是潜在客户的名字，而不是我们倾向于在电子邮件中使用的冷漠的“嗨，约翰”方式。

此外，语音重建技术可用于内部通信。你再也不用担心内部视频中的失误:你可以将你的录音转换成文字稿，然后编辑文字稿以使你的视频清晰、无错误、可展示。对于全球的大公司来说，记录错误每年会造成数十亿美元的损失，因此这是非常宝贵的(尤其是对于提供基于视频的服务的公司)。

节省时间/成本: 30%或更多，并能显著提高营销活动的质量。

拥有大型IT或编程部门的公司会注意到这一点；现在有一种技术可以为你编写大量的代码，并且可以检查你的工作并指出错误。例如，Copilot 是一个自动化的人工智能编程助手，它接受了数百万高质量企业级代码示例的训练。

使用它的最简单的方法是简单地下载 Copilot 扩展，用于您团队选择的IDE。如果您的编程部门使用VSCode，这很容易——只需前往扩展选项卡，键入 Copilot ，并安装第一个结果。

之后你得到的绝对是非凡的。从历史上看，代码完成扩展可能会提高您的工作效率5–10%， Copilot 可以定期编写完整的函数和错误处理流程。就我个人而言，我用它来编写超过30%的代码——而且它在你不熟悉的库或工作流上工作得特别有效。

节省时间/成本:编程时间节省30%或更多。
