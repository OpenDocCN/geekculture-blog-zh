## 我为管理你的事业所做的微薄努力

几个月前，我换了工作。作为这一举措的一部分，我已经将我日常使用的编程语言从使用了八年多的 C#改为 Java。这可能会让一些人感到困惑，并到处引起一些人的怀疑，问我为什么要放弃我的经历，只为了开始一门新的语言。

在下面，我将解释这样做的理由和所涉及的过程。

## 你是怎么做到的？

Did I do that? — [src](https://media.giphy.com/media/BxWTWalKTUAdq/source.gif)

从 C#到 Java 编程语言的转换很简单。在技巧、局限性等方面获得经验……只是时间问题，我多年的软件开发经验肯定会派上用场。

另一方面，从。从 CLR 到 JVM 是一个更大的飞跃。虽然有些概念是相似的，但实际上是一组完全不同的 API，它们的组织方式也不同。除此之外，我还改变了工作方式，从使用单块芯片转向使用基于微服务架构的东西，因此语言和平台本身必须添加到操作系统、工具(如 Docker)以及分布式系统带来的固有困难中。那是相当大的挑战。那正是我所希望的。

一个很大的障碍是大量的库/框架在做类似的事情，各种 Java APIs 试图处理这些混乱。的。NET universe 在这方面要简单得多(无论你想做什么，都有明确的工具选择)。一种“杀不死你的东西会让你变得更强”的情况。

## 把它包起来

总的来说，转换编程生态系统的经历是你成长过程中的一个重要里程碑。为以后潜在的转行做铺垫(既然你向自己证明了“我能行！”).这样做真的可以拓展思维，将你的思维从你已经习惯的技术堆栈所建立的边界中释放出来。

当人们听到交换技术时，他们会有不同的反应。在我看来，许多人过于重视语言、平台和工具，而对使用它们的目的、产品的实际开发以及从开发中获得的经验重视不够。

或者换句话说，代码是业务的**手段**，而不是目的(目标)。
