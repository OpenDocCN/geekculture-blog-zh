**‍** 消费者对移动商务以及银行、医疗和教育等其他在线互动的普遍依赖因封锁而加速——而欺诈者也乘虚而入。

现在，不仅仅是高调的案件，如 Twitter 首席执行官杰克·多西账户被接管，或科技企业家罗伯特·罗斯在 crypto 上的 100 万美元救命损失，都是欺诈活动的目标。

许多种类的手机诈骗，尤其是 [SIM 卡互换](https://tru.id/blog/what-is-sim-swap-prevent-account-takeover)，正危险地成为主流。就在最近，《连线英国》报道了皇家邮政短信诈骗的[【无情崛起】](https://www.wired.co.uk/article/royal-mail-text-message-scams)，例如*《太阳报》* [警告](https://www.thesun.co.uk/tech/15014487/whatsapp-scam-access-code-message/)警惕 WhatsApp 诈骗访问代码。根据战略和研究公司 Javelin 的数据，账户接管欺诈同比增长了 72%(2020 年)。

实施 SIM 卡互换欺诈最常见的方法之一是截取 PIN 码，然后接管客户账户。银行、金融科技公司和任何使用短信发送安全 PIN 码的公司都容易受到攻击，他们的用户和客户现在面临着身份被窃取和存款被盗的风险。

# SIM 卡互换欺诈是如何运作的？

**‍** 如今，大多数基于电话的身份认证方法只是使用手机号码，并依靠通过短信或语音呼叫发送的 PIN 码。公司认为这是一种所有权因素认证方法，但问题是它不能可靠地证明所有权。有一些根本性的缺陷——坏演员正在利用这些缺陷。

越来越多的不法分子通过说服移动运营商给他们发放一张新的 SIM 卡来进行 SIM 卡互换欺诈，这张卡将取代同一个手机号码。

然后，他们能够接收发送到该号码的所有语音呼叫和短信(包括 PIN 码)，然后使用这些代码接管该用户的帐户。SMS 2FA 还有很多其他问题；有关完整的比较，请参见此 [SMS 2FA 安全分析](https://tru.id/why-tru/sms)。‍

# 基于 SIM 卡的认证如何对抗 SIM 卡互换欺诈

[基于 SIM 卡的认证](https://tru.id/products/active-simcheck)是一种简单的交钥匙解决方案，适用于任何使用 SMS PIN 码进行用户认证的在线企业，以保护其客户和品牌免受身份盗窃和账户盗用。

当今企业面临的许多安全挑战都是由过时的对密码和 SMS PIN 码的依赖造成的。基于 SIM 卡的身份认证提供移动本地、无缝、安全和私密的用户身份认证。

通过基于 SIM 卡的认证，基于 API 的服务可以直接实时连接到移动网络运营商，以验证用户手机中 SIM 卡的身份。如果该 SIM 卡最近发生了变化，API 将标记该变化，从而能够采取行动并阻止潜在的欺诈者拦截包括 SMS 2FA PIN 码在内的 SMS 消息。

这种新的安全检查可以由开发人员快速轻松地与现有的 SMS 2FA 解决方案集成在一起。不需要对用户体验做任何改变。
