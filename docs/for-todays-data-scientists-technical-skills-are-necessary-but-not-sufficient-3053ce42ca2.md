# 对于今天的数据科学家来说，技术技能是必要的，但还不够。

益普索·莫里(Ipsos MORI)最近对英国人工智能(AI)劳动力市场进行的一份报告支持了一个在该行业经常被忽视的事实——技术技能对于今天的人工智能工作来说是必要的，但不是充分的。

当然，专业技能是必要的。你需要理解人工智能概念和算法，有编程经验(Python 是当今主要的数据科学语言)，熟悉当前人工智能相关的软件产品(云人工智能产品和基础设施，开源工具等)。)，擅长数据扯皮，对数学、统计等有扎实的掌握。

但这就够了吗？

根据我的经验——并得到上述研究的支持——答案是否定的

让我们来看看我在面试未来的数据科学家时问的一些问题。

# 技术的

*   给我一分钟的时间介绍一下你的背景，以及你是如何演变成今天这个角色的？
*   能告诉我 AI/ML/DL 的区别，以及你用的最多的是哪个术语吗？
*   说说你的机器学习背景。
*   机器学习最难的是什么？
*   说说你对 AWS/Azure/GCP 的体验，尤其是在大数据、分析和机器学习方面。

这些问题让我了解候选人的背景和理解力，并确保他们的简历是合法的(你会惊讶于人们在简历上有多少夸大或不真实的地方！).如果有任何技术方面的问题，这是我们可以深入探讨的地方。

# 软技能

*   你会如何向没有经验的人解释机器学习？
*   给我一个推销让我对使用机器学习感到兴奋？
*   你会建议某人如何开始机器学习？
*   你喜欢和人群交谈和指导研讨会吗？

公开演讲，将先进的概念分解成易懂的术语，深入理解价值主张(并有效地交付)，理解学习过程——这些都是当今人工智能角色的关键方面。

当然，在过去，有一些技术人员非常擅长编程之类的东西，但是你不敢让他们与客户或管理层交流，因为他们的社交技能不够出色。但在当今世界，尤其是新兴技术，如云、人工智能、区块链、量子计算等。大多数人不了解技术，所以你有效解释事情和激发激情的能力是至关重要的。

你可以有惊人的技术才能，但是如果你不能提出一个可以理解和令人兴奋的提议，你很可能没有机会展示这些才能。理解如何构建神经网络很重要，但如果你不能从商业角度把握问题的大背景，那还有什么意义呢？如果你是一名学术研究者，那就为了科学去解决一些新奇的问题吧。在现实世界中，人工智能只是一个用来解决问题的工具(只是众多工具中的一个)。

# 人类

*   你自己学到的帮助你更好地完成工作的最酷的事情是什么？
*   给我一个你对除了你自己以外的人真正感兴趣的例子。

归根结底，我们的合作伙伴和客户都只是普通人。人们想和他们喜欢的人做生意。给我一个普通的数据科学家，一个比自大狂摇滚明星更酷的人。你如何向别人谈论你的兴趣？你对什么感兴趣(为什么)？你如何与其他人相处？这些都是同事身上极其重要的方面。懒惰、戏剧性、自我中心、毒性——这些都是要不惜一切代价避免的事情。

# 准备和好奇心

*   你有什么问题要问我吗？

我试图让我们的谈话(是的，面试是一次值得重视的谈话，而不是一次令人恐惧的审问)有整整一半都只集中在这个问题上。候选人问的问题几乎总是比我问他们的任何问题都更有见地。对他们来说什么最重要？他们之前的经历是怎样的？这些通常会推动他们提出问题。他们的备考技巧是什么？如果你在这一点上没有问题要问我，你可能得不到这份工作。

当然，有一些好的，但是相当标准的问题可以问面试官。

*   你的理想人选是什么？
*   这个角色最重要的是什么？
*   你如何解释公司/团队的文化？
*   典型的一天是什么样子的？

当然，也有不好的问题要问(除非你在最后的谈判中，细节很重要)。

*   我的确切时间是什么时候？
*   我每年有多少假期？
*   我永远不会被期望工作超过 40 小时，是吗？
*   这个角色的薪水是多少？
*   您使用哪些特定的软件工具？

也有更有见地的问题。

*   我想成为团队的一员。团队结构是怎样的？我将和谁一起工作，他们是什么样的人？
*   我喜欢更快的节奏。你如何划分这个角色的节奏？
*   你觉得面试进行得怎么样？你认为我的优点和缺点是什么，还有哪些地方需要改进？

前两个问题是候选人知道他们想要什么，并调查在这个特定角色中是否存在的例子。如果问得正确，这可能是非常强大的。

最后一个问题(或类似的问题)是每个求职者在每次面试中都应该问的问题。它显示了你作为一个人的脆弱，你接受批评的能力，你改善自己的愿望，你的好奇心。这些都是值得展示的好东西。你从中得到了什么？你的面试技巧将会从反馈中得到极大的提高。你不会几天/几周坐着想你做得怎么样，或者你是否会接到电话。可悲的是，这些年来我面试过的数百名候选人中，只有三人问过这个问题——我记得他们每一个人。

想更进一步给人留下深刻印象吗？对你提出的问题要有创意！最近的一个例子是，我问了一位候选人一个“你自己学到的最酷的东西”的问题。他们的回答(至少是前两句)是:“贝叶斯统计的整体理论。这是一种哲学，认为可能有一个核心真理，但问题不是找到正确的答案，而是越来越接近，知道自己有多接近。”我发现这个回答很有趣。然后到了他问我问题的时候，他的第一个问题是:“你的贝叶斯统计是什么[意思是什么让我兴奋，面试官？]这在你的日常生活中有多大作用？”把我们谈话的不同部分编织在一起，表现出对我这个人的兴趣，以独特的方式询问日常工作。我发现这个问题在几个层面上都很有趣。他得到了那份工作。

# 回到手头的话题

如你所见，我在面试中关注的大部分都是非技术性的，而是基于软技能的。有效的沟通、理解大局、商业意识、好奇心、做一个正派的人——这些是我倾向于关注的事情。

当然，你需要一些熟练的技术——我的意思是，如果你从未听说过互联网，你可能有一个相当陡峭的上坡攀登。但是只要你有基本的技术能力，我可以教你剩下的。教某人 XGBoost 或语义分段要比教他们保持好奇心、成为一个有效的沟通者或不是一个混蛋容易得多。

今天，要想在这个职位上取得成功，你不仅需要成为销售人员，还需要成为开发人员。

大学和在职培训通常会教给人们技术技能，但仅靠这些还不足以在人工智能领域取得真正的成功。如果我为人工智能学位开发一个课程，我会包括数学、统计、工程、编程、数据争论、云基础设施和机器学习(框架、算法、超参数优化、模型漂移等)等课程。)—所有你期待的事情。但我也会包括商业、伦理、哲学、演讲、交流、批判性思维和领导力等课程。

> “如果我有一个小时来解决一个问题，我会花 55 分钟思考问题，5 分钟思考解决方案。”
> ――阿尔伯特·爱因斯坦

所以，下次你看你的简历时，不要只看“我利用 PCA 进行特征工程”或“我用 BlazingText 算法开发了一个模型”。展示你对你正在解决的商业问题的理解，以及这些人工智能技术是如何帮助你的。投资回报率是多少？可以考虑用 STAR(情境、任务、行动、结果)法，把自己的想法表达清楚。

下次你参加面试时，放松并享受双向交流。要真实、好奇、有思想、有趣。你是那种人们想与之共事的人吗？面试官看起来像是你愿意与之共事的人吗？记住，公司在面试你，但你也在面试公司。太多人似乎不明白这一点。

当你在工作时，请记住，能够用简单的语言向客户或团队成员解释事情比将模型的准确性从 90%提高到 93%更重要。寻找机会利用(或增强)你的领导能力。不仅要对如何用人工智能解决问题感到好奇，还要对围绕人工智能的隐私和伦理问题感到好奇。理解我们试图解决的业务问题的全貌应该是你的首要目标。没有这种理解，你很可能会用你的技术技能来解决错误的问题。
