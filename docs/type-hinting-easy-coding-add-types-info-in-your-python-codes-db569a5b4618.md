# 为什么添加类型可以使编码更容易？

我宣传 Python 提示的主要原因是**通过提供类型提示，你的文本编辑器有更多的信息来帮助你**。

现在很多人使用相当智能的文本编辑器(比如 *PyCharm* 、 *Visual Studio Code* 、 *Atom* 、*Sublime Text* 、 *vim* 等)来提高编码效率。这些文本编辑器有很好的功能，例如语法高亮、自动完成建议以及在您键入时在工具提示框中显示帮助信息(参见下图中的示例)。

![](img/7cd30b624a03d31fc71ad5075eb078fa.png)

Whenever possible, modern text editors automatically display help information in a tooltip box while you type. (screenshot of an example in Visual Studio Code)

然而，由于 Python 的动态类型特性，您的文本编辑器通常没有足够的信息来决定如何帮助您。而当您使用的变量大多是原始类型(`str`、`int`等)时，这似乎不是一个很大的优势。)，当处理具有复杂内部结构的类时，您的编码体验可以提高很多。

## 可迭代类型

有经验的读者可能已经从上面问题 1 的描述中认识到`l`到底是什么。这是一个 **Python 可迭代的**。从技术上讲，Python iterable 并不是真正的类型，而是一种协议。任何支持迭代方法(类定义中的一些特殊方法)的类都被定义为可迭代类型。关于 Python iterables 的讨论值得单独写一篇文章。这里你需要知道的是，如果你能做`for x in obj`，那么对象`obj`就是可迭代的。比如列表，元组都是可迭代的。文件处理程序(由`open`方法返回的对象)是可迭代的，因为您可以通过`for line in fhand`来迭代文件的行。

我们如何暗示`l`是一个可迭代的对象？好消息是，iterables 在 Python 中无处不在，它的类型提示已经被创建了。你需要做的就是从`typing`模块导入`Iterable`。

## 联合和可选

解决这个问题至少有两种方法。首先，我们可以声称`upper_limit`是`int`或`NoneType`类型。`typing`模块中的`Union`类型用于解决这种“非此即彼”的情况。利用`Union`，我们可以写出`upper_limit: Union[int, NoneType]`。

由于`None`是许多可选变量非常流行的默认值，`typing`模块还引入了另一个名为`Optional`的家伙来满足我们的需求。比如`Optional[int]`和`Union[int, NoneType]`本质上是一样的。

# 给初学者的最后几句话

如上一节所示，您可以深入研究 Python 的类型提示系统，很快事情就会变得很糟糕。虽然掌握类型提示非常有价值，特别是对于一些大组织和大项目，但是对于初级开发人员来说却是不必要的复杂。幸运的是，Python 允许您逐渐采用类型暗示知识和实践。如果你的代码没有类型提示，甚至是错误的类型提示，Python 不会拒绝运行你的代码(同样，类型提示不会改变 Python 的动态类型本质，在运行时会被解释器忽略)。我个人的建议是，利用这个渐进的特性。当您添加类型信息时，请确保您添加了正确的信息，但没有必要键入任何提示。

还要注意，有类型检查工具(例如:`mypy`)可以用类型提示检查你的代码(就像编译源代码时编译器所做的)。然而，它可能会对初学者造成一些最初的挫折，并使您过早地陷入更高级类型的泥淖中，这暗示了一些不必要的技术性问题。如果这让你感到沮丧，那就先不要这样做，只有在你对类型提示更加熟悉之后再来做。

在我看来，下面是一些添加类型提示会对你或你的代码用户有所帮助的例子:

1.  **参数是某些类的实例的函数**。正如我们在上面所演示的，如果类本身有很好的文档记录和类型提示，当您使用这个参数时，您可以立即从您的文本编辑器中获得有用的帮助。
2.  **如果你正在编写一个模块(供自己或他人使用)，试着给每个函数(参数和返回值)或类(类变量和方法)添加类型提示**。这将确保模块的用户可以获得帮助。事实上，在这种情况下，我建议编写格式良好的 docstrings，但这将值得另一篇文章讨论。
