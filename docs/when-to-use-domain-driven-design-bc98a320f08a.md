# 三种心智模式

在任何软件中，至少有三种心智模式在起作用。首先，是用户的心智模型，她对软件如何工作的理解。第二，领域专家关于软件应该如何工作的心智模型，最后，开发人员关于软件实际如何工作的心智模型。一般来说，DDD 很少谈论用户的心理模型，因为它试图将核心域与表示和持久性问题隔离开来。

## 为什么我们要让这些心智模型保持同步

> 如果你练习学习不仅仅是为了积累知识，而是为了建立信息可以依赖的心智模型，那么你就进入了学习促进学习的反馈循环。查理·芒格

当这三个心智模型同步时，客户/领域专家心智模型和代码之间存在一对一的映射。任何新的需求都变得相当容易/更快地实现。当不同步时，我们需要将概念从一个心智模型转换到另一个。

如果不保持这些心智模型的同步，您可以随时以您想要的方式修改代码，而无需咨询领域专家，但是，开发人员和领域专家的心智模型是不匹配的。开始时，软件将会很小并且容易理解，用很少的翻译来同步开发者和领域专家的心智模型并不难，然而，复杂性将会随着时间的推移而增长，心智模型将会逐渐分离。

考虑到变化的需求，软件应该如何表现是由领域专家来决定的，但是当心智模型不同步时，领域专家不得不依靠开发人员的心智模型来决定软件应该如何表现。将会有大量的翻译发生，信息必然会被遗漏或误解。

## 保持心智模型同步

你可以通过非常严谨地构建这些心智模型来实现这种同步。它不一定要反映现实，但无论我们选择什么心理模型，我们都应该严格对待它，并确保每个人都有相同的心理模型。

## 什么时候让这些心智模型保持同步，也就是什么时候使用 DDD

如果你的软件没有太多这样的翻译，你就不需要 DDD。当项目足够小或刚刚开始时，通常会出现这种情况。然而，请留意这些翻译，如果它们失去控制，是时候做适当的领域建模了。

## 使用 DDD 来简化，而不是复杂化

使用 DDD 以最简单的方式对一个复杂的领域建模。永远不要用 DDD 让你的解决方案变得更复杂。
