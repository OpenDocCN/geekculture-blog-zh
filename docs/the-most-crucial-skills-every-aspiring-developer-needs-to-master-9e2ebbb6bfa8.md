# 每个有抱负的开发人员需要掌握的顶级技能

![](img/279962273dcabcfbb1a6979849c911c5.png)

如果你正在踏上成为一名软件工程师的旅程，那么这是为你准备的。这个故事的见解来自我在网上编码学校 Microverse 学习期间的个人经历和奋斗。

作为一个相对来说对编程世界还比较陌生的人——我是一年前加入的——我整理了一份我认为对任何有抱负的程序员都必不可少的技能清单。

# 谷歌搜索的艺术

你说，谷歌搜索是一门艺术？虽然看起来很琐碎，但这实际上是每个初学者需要掌握的第一个技能。相信我。

如果不知道如何有效地搜索，你将几乎不可能在互联网的浩瀚信息海洋中找到答案。

![](img/8a886aee1e50dbb36549eacf962d2e15.png)

*Photo by* [*Glenn Carstens-Peters*](https://unsplash.com/@glenncarstenspeters?utm_source=medium&utm_medium=referral)

# 有效地使用谷歌需要三样东西:

1)了解提出什么基本问题来解决你的问题

2)知道如何提问/表达关键问题和搜索

3)能够仔细评估和选择要参考的资源

这意味着能够快速浏览最常访问的站点和程序员讨论的论坛；堆栈溢出、Reddit 线程、Dev.to 和黑客新闻等等。同样，能够阅读和钻研大量的文档，以理解语言和框架实际上是如何工作的。

这还没有结束。使用谷歌来摆脱困境将是经常发生的事情。那么，秘密是什么？

让谷歌成为你新的最好的朋友。

俗话说得好:

> “如果你不知道什么，去谷歌一下！”—古代谚语

# 耐心坐下的艺术

你可能会问，坐下和编程有什么关系？嗯，我的朋友，如果你想学习如何编码，就必须能够在你的笔记本电脑上不间断地坐下来(或者站起来，对于那些喜欢站着办公的人来说)。

![](img/c37af4d8274387923e7496520251ed2e.png)

*Photo by* [*Scott Winterroth*](https://unsplash.com/@contentacademy?utm_source=medium&utm_medium=referral)

没有这项技能，成为一名程序员几乎是不可能的。编程不适合胆小的人，也不适合那些粘在椅子上无法理解的人，他们会感到无助，因为他们的大脑在寻找解决方案。

事实上， [TechLead 的 Patrick Shyu](https://www.youtube.com/channel/UC4xKdmAXFh4ACyhpiQ_3qBw) 说，那些不能将这种技能转化为习惯的人最终很可能会辞职。好处是，这种技能可以通过有意识的努力和日常练习来学习。而且，一旦掌握了，这将是学习如何编码的最快和最简单的方法！

掌握这项技能如此重要的原因是它开启了进入状态的能力。这是程序员能够想出如何让一件事情工作并让奇迹发生的时候。

这种现象也被称为深度工作，这是学习和做任何努力的根本技能。

> 深度工作是指专注于一项认知要求很高的任务而不分心的能力。这是一种技能，可以让你快速掌握复杂的信息，在更短的时间内产生更好的结果。”——[*卡尔纽波特*](https://www.calnewport.com/books/deep-work/) *‍*

# 通过三个简单的步骤进入状态:

1)通过像 [Flow](https://apps.apple.com/app/id1423210932) 这样的工具划出大块的时间来专注于一件事(我个人使用并且喜欢这个！)，这是一个番茄定时器，可以帮助你集中注意力，提高工作效率。

2)通过使用像 [Forest](https://www.forestapp.cc/) 和 [Newsfeed Eradicator](https://chrome.google.com/webstore/detail/news-feed-eradicator/fjcldmjmjhkklehbacihaiopjklihlgg?hl=fil) 这样的应用程序来限制和约束访问，从而消除所有干扰，断开与社交媒体的连接。

3)抵制多任务的冲动，从一件事跳到另一件事。

一旦满足了这些条件，肯定会更容易进入地带。

# 将事物分解成更小的部分

我从我的三位摇滚明星导师那里直接了解到这项技能的重要性，他们是:达米安·坎、艾伯特·帕丁、保罗·萨蒙塔内斯。他们都教会了我这一课——能够将事情分解成更小、更易管理的块，这是编程的本质。

![](img/fceee3190991c5f294edd159b8aefeae.png)

*Photo by* [*Ross Sneddon*](https://unsplash.com/@rosssneddon?utm_source=medium&utm_medium=referral)

通常情况下，会有你陷入困境的时候。老实说，这是一种非常糟糕的感觉。不，你不是一个特殊的独角兽，所有的开发者都会经历这个。事实上，如果你真的想成为一名程序员，那么你必须拥抱这种感觉，因为它永远不会消失。即使是最资深的程序员也经常被卡住。

正是在这一刻，当我觉得一切都失去了，我记得我的导师的话和建议。

# 分解东西时要记住一个简单的过程:

1)缩小视野，意识到如果你把这个项目看成一个巨大的问题，那么，当然，你会感到不知所措。相反，把一个大问题重新组织成你能真正解决的小问题。

2)从这里开始，确定目标是什么，问自己:“我能完成这件事的最简单、最快的方法是什么？”记住这一点，你现在可以规划出阻力最小的路径，知道什么需要优先考虑。

3)现在您已经有了构建框架项目的地图，请评估您工具箱中的工具。看看你目前拥有的技能，用最简单的方法解决每个问题，一步一步地使用它们。

通过遵循这一过程，你可以避免增加不必要的复杂性，并学会如何组织、规划和构建你的想法，使之成为一个清晰明了的行动计划。

正如我从我的导师那里学到的，在一天结束的时候，知道如何把事情分解，就是编程的意义所在。项目不会一下子就奇迹般地自行实现。

就像建筑一样，只有在有了蓝图之后，实际建设才会发生。即便如此，事情仍然按顺序发生，每一块砖都是小心翼翼、深思熟虑地垒起来的，一次一个例子。

令人惊奇的是，这不仅仅适用于编程。威尔·史密斯采用了这个原则，他认为这是他能够克服阻碍他成功之路的看似不可能的障碍的原因之一。

> *“你不要试图建一堵墙。你不打算建一堵墙。你不会说‘我要建造有史以来最大、最差、最伟大的墙。…你说‘我要把这块砖砌得尽可能完美’，你每天都这么做，很快你就有了一面墙。”—威尔·史密斯*

# 拥抱奋斗&投入时间

最重要的是，最重要的技能是能够磨出来。事实是——如果你是编程新手，那么你真的需要拥抱奋斗。这是无法回避的。如果不投入时间，一个人就无法学习和处理大量信息。

当你努力学习时，你可能会想放弃。不是一次，而是多次。感觉就像你被困在这个深坑里，如果你想要自由，除了爬上来，你别无选择。作为一名有抱负的软件工程师，我有时会有这种感觉。

学习编码意味着习惯于不时感到不舒服、不聪明和失落。你不应该为此自责。

你对自己的教育负责。如果你想学什么，那么你只需要练习。这和你学吉他或者其他任何东西是一样的。

你尝试，然后失败。尝试，然后失败。重复一遍。

你试了一次又一次，直到最后，它坚持。这样的循环，虽然不可避免，也很痛苦，但一定会值得。重要的是你每天都要坚持练习。

一个简单的事实是，如果你不花时间和精力去奋斗和磨砺这一切，那么你就不会学习，或去任何地方。

![](img/0015c0f7f4deb65ec71f69e3e5998e42.png)

*Photo by* [*Logan Fisher*](https://unsplash.com/@loganrfisher?utm_source=medium&utm_medium=referral)

正是通过拥抱斗争和投入时间，一个人学会了如何编码、理解数据和构建有意义的产品——影响数百万人的应用程序、系统和解决方案，影响着我们生活的每一部分。

简单地说，我相信这些话说明了一切:

“学习如何编码没有捷径可走。我完全相信任何人都可以学习编码。这可能不容易，但绝对可行。我看到的成为最好的人是那些投入最多时间的人。投入更多的时间，你一定会变得更好。句号。”—戴米恩·坎

这正是成为一名成熟的软件工程师所需要的，也是为什么掌握这些技能至关重要。
