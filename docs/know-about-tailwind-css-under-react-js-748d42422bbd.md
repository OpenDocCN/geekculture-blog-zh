React.js 是一个创建应用前端的绝佳工具。您可以使用 React 充分发挥其潜力，创建有史以来最复杂的应用程序，但是如果它看起来不美观，没有人会想使用它。为了让我的应用程序看起来更好，你可以使用普通的 CSS、Bootstrap 和 Tailwind。它们都有各自的优缺点，但是为了这篇博客的目的，我们将了解什么是 Tailwind CSS 以及如何在 React 应用程序中使用它。

# 什么是顺风 CSS

Tailwind CSS 是一个区分功能优先级的 CSS 系统。这意味着您用类名(实用程序)来装饰您的应用程序。你可能会认为这和普通的 CSS 是一样的。如果您使用的是直接的 CSS，您可以定义自己的类名，并将样式属性和值放在一个单独的 CSS 文件中(除非您使用的是内联样式……稍后会详细介绍)。

Tailwind CSS 带有预定义的类名，对应于 CSS 文件中的 CSS 属性。您可以在组件中使用类名，而不必离开文件。这有助于您更快地开发，尤其是当您在小屏幕的笔记本电脑上工作时。

# React 内嵌样式怎么样

如果使用内联样式可以获得相同的结果，为什么还要使用框架呢？当您使用 reactions 内联样式时，您正在构建 JavaScript 对象来管理您的样式。为了做到这一点，你必须用双花括号`style={{}}`将你的样式括起来。你还必须用不同的方式书写你的风格。因为这些是我们注入到 JSX 中的 Javascript 对象，所以我们必须用 camelCase 编写我们的样式。`background-color: red;` == `{{backgroundColor: “red”}}`必须声明整个属性及其值会使你的代码非常拥挤，难以阅读。在 Tailwind 中，该属性由一个类名表示。为了有助于可读性和一致性，这些类名较短，并且包含设置值。

Tailwind 允许您直接在组件中控制状态变量和响应，这是一个显著的区别。此功能不适用于内嵌样式。使用 Tailwind CSS，您可以针对不同的媒体大小调整组件，并通过更改类名来添加悬停和聚焦等效果。Tailwind 本质上提高了内联样式的可读性和功能。

# 顺风 CSS-基础

现在我们已经确定了什么是顺风 CSS，让我们来谈谈顺风 CSS 的原则。Tailwind 的初始设置可能有点困难，但是文档可以很好地指导您完成它。

# 了解— tailwind.config.js

这是您系统的配置文件。默认情况下，Tailwind 包含以特定方式设计的专用类名(实用程序)。这对于 starting started 来说是极好的，但是如果你想改变一些默认设置呢？您可以在配置文件中更改这些类的名称和行为，而不是将自定义类添加到 css 文件中。设置和更新都很简单。每个主要实用程序的文档都将向您展示如何编辑配置文件，以便在页面底部添加您自己的版本。

# 效用第一

顺风 CSS 以此为秘制酱料。如果你以前从未使用过实用优先的框架，这可能需要一点学习过程，但是请相信我，这是非常值得的！不同的类名一开始可能会令人望而生畏，但是它们的命名方式非常直观。记住这些名字要比不断想出新的唯一类名容易得多，就像 CSS 一样。同样，每个工具都有一个设定值，它允许你为你的设计带来更多的恒定性。不需要记住你三天前创建的类和它的值，你只需要记住你的标题使用了`m-8` ( `margin: 2 rem`)。简单多了。要使用这些工具，你只需将相应的工具添加到你的元素的类名中。如果我想创建一个有`border-style: solid; border-width: 2px;` `max-width: 100%`和`box-shadow`的`div`，我要做的就是:

```
<div className="width-full border-solid border-3 shadow">
Hi there!
</div>
```

# 基本样式和提取组件

一次又一次地向相同类型的元素添加相同的类是一个巨大的麻烦。这使得开发变得乏味且难以持续。幸运的是，Tailwind 允许您定义自定义组件类和基本样式。基本样式可以直接应用到你的 CSS 文件中，就在你安装 Tailwind 的地方。对于某些标签，您可以设置默认样式。如果你确定这个标签将来需要以这种方式设计，我会建议你这么做。不需要使用标签并一遍又一遍地添加所有的实用程序，您可以只使用标签。
