两年前，当我把 Alexa 带回家时，我的孩子们喜欢玩这个新玩意儿。Alexa 在我儿子面前表现得很好，他问了我所有关于体育的问题，密歇根本地人的分数，汤姆·布拉迪的年龄，洛杉矶道奇队的交易等等。然而，Alexa 在与女儿互动时似乎有困难，“Alexa，我最喜欢的歌是什么？”Alexa 回答，“嗯，我不知道”。“Alexa，如果 2+2=5，那么 5+5=2 吗？”Alexa 骄傲的回答，“5+5 等于 10”。当我女儿玩完 Alexa 后，她说:“Alexa，你真笨！”

我在工作中也经常遇到类似的情况。许多高管或客户问我，机器学习(ML)可以做什么，以确定他们的问题是否可以通过 ML 解决。特斯拉的自动驾驶汽车，亚马逊的 Alexa，网飞的推荐算法，谷歌的 AlphaGo，IBM 的 Watson 都是成功的 ML 故事的例子。那些成功的故事给 AI 和 ML 带来了兴奋，但也教会了我们 ML 的局限性。特别是在资源和数据有限的非大型科技公司工作时，我通过经验了解了 ML 项目的成功和失败。

**机器学习可以做任何与模式识别**相关的事情:例如，Yelp(你尝试一家新餐厅，然后评论这家餐厅)使用 ML 将 Yelpers 上传的数千万张照片分类为不同的类别，如食物、饮料、内部、外部和菜单。每个类别都有不同的模式，例如，菜单照片会比其他类别有更多的字符。另一个例子是，大通银行使用 ML 来检测潜在的欺诈交易。欺诈检测模型从大约 35 亿个用户帐户中学习可疑交易模式。当交易到来时，模型将查看该交易是否具有与过去欺诈交易相似的模式(监督学习)，或者该交易是否具有与大多数其他交易不同的模式(非监督学习)。如果您的业务问题是关于识别模式，那么它可以从 ML 中获益。

我在现实世界中见过的最赚钱的用例是自动化繁琐的人工任务。让我们以发票处理为例。当发票通过电子邮件送达时，您的客户服务代表(CSR)打开电子邮件，下载发票，然后手动将相关信息输入 ERP 系统，如发票号、卖方的企业名称、您的地址、发票到期日、产品描述、产品数量、单价等。可以想象，每条信息都有模式。例如，产品的数量将主要是数字，不会出现在卖方的地址之前。在美国，发票到期日的格式通常为 mm/dd/yyyy。发票的产品说明通常由文本而不是数字组成。对于这种模式识别任务，ML 一直在迅速改进，许多企业已经证明了它的效率。

机器学习不能做任何与推理相关的事情:谷歌大脑的创始人、前百度首席科学家吴恩达表示，如果一个普通人可以在不到一秒钟的时间内完成一项脑力任务，我们很可能可以用人工智能来解决它。换句话说，如果一项任务需要一个思考过程，那么获得 ML 全部好处的时机可能还不成熟。一个例子是当有四个数字时；1, 3, 5, 7.这个数列中的下一个数字是什么？你很容易猜到，对吧？这个问题问你最喜欢的 AI 助手。你得到滑稽和不合逻辑的回答的原因是，人工智能是建立在机器学习上的，而不是机器推理。尽管 ML 有时看起来很聪明，但它还不具备有效推理的能力。

让我们回到我们的发票处理示例。ML-powered 解决方案将在人类 CSR 打开发票之前，在眨眼之间提取最相关的信息。ML 可以提取你所有发票的 100%信息吗？很可能不是。总有一些例外，ML 没有从提供的数据中学习到。例如，在某一行中，“17 英寸显示器”在“价格”列中，而“说明”列中则为“$100”。根据经验，人类 CSR 会推断这是一个数据输入错误，并交换两个数据值。简单来说，人类知道错误可能发生在任何地方。然而，这个例外对 ML 来说并不明显。ML 不会告诉你这看起来像是一个错误，也不会告诉你这个值看起来像是用“$100”交换的，等等。

**在您的组织中使用 ML 的最佳方式是什么？**

即使科幻电影已经用冷血的有知觉的机器人描绘了我们的未来，幸运的是，这种场景似乎不会很快实现。正如它的名字所说，今天的机器学习意味着计算机自动学习一些东西。有些东西是一种模式，人们是如何感知和决定的。ML 擅长模仿这种人类的动作。环顾你的组织。是否存在重复的手动任务？在这些任务中，ML 将能够迅速带来切实的好处。然而，ML 不会取代所有的手工任务，人类仍然需要做 ML 不能做的事情。曼梯·里比我们快，但并不比我们聪明……到目前为止。
