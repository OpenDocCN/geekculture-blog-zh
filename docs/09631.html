当我已经安装了conda时，我再次运行了conda安装程序

几天前，我必须进行一次物理迁移，所以我必须将我的项目迁移到另一个我已经有一段时间没有使用的终端/环境。我不知道为什么我认为我没有安装conda，然后直接开始下载并运行。pkg安装程序。这出现在我的屏幕上:

Installation failed.
conda already installed at [path].

哦。我耸了耸肩，做了一个小小的心理手势。我一开始没有注意到任何问题，并认为一切只是简单地回滚。没问题，我想。

不对。我最终花了几个小时来修复这个愚蠢的错误(还想知道为什么安装者没有在安装之前检查一下)。

提示:务必快速检查康达是否已经安装。你只需要一个简单的命令就可以做到-

conda list

免责声明:我在这个特定的场景中使用的是Macbook Pro(英特尔)，所以一切都只基于Mac。

conda装置损坏/混乱的迹象

  1. 环境不能被激活(但你知道它存在)
  2. 环境有不同的父路径(当您执行conda env list时)
  3. 格式化的打印语句变成了错误的语法(当您的Python 3 env变成Python 2时)
  4. 找不到安装的模块

我是怎么修好的

  1. 使用官方说明卸载conda。我在Mac上，所以我删除了~/opt文件夹。我还删除了。bash_profile文件。
  2. 重启电脑。
  3. 当我尝试重新安装conda时，我很惊讶仍然得到同样的错误“conda已经存在”。然而，现在我所有的环境都来自一个父目录。我认为这意味着我已经卸载了“额外”的副本，但仍有一个副本。我尝试继续按原样使用conda，可惜还是损坏了python版本不对，安装的模块总是找不到。
  4. 我重复了上面的步骤1和2。
  5. 这一次我没有在终端中看到'(base)'，我想我一定是成功卸载了conda。然而，当我尝试重新安装conda时，我仍然得到“conda已经存在”的错误。
  6. 诀窍就在这里！在“选择安装目标”步骤，单击“选择目标”按钮。它将显示一个错误，说明它无法安装在上述目的地。点击它，进入“选择磁盘”，但不要前进。回到之前的状态，选择完全相同的文件夹。现在错误消失了。
  7. 继续安装，交叉手指。

其他破损

VS代码中的端子可能会继续断开。我通常点按“终端”>“新终端”,然后在那里键入我的命令。但是现在我发现我必须使用导航条顶部的小“播放”三角形按钮来触发终端。
