# Git 是什么？

在告诉你这背后的理论之前，Git 最初是在 2005 年开发的

正如我之前告诉你的，Git 是一个版本控制系统，通常用于跟踪你在创建任何项目或程序时所做的变更

就像，如果你正在构建任何软件，那么这将跟踪你拥有的你正在制作的软件的源代码

所以，对你来说，跟踪和记录你的软件也很容易

您可以将这个 Git 下载到您的本地系统(计算机或笔记本电脑)中，这是免费使用的

一旦你把它下载到你的本地系统中，你甚至不需要互联网连接就可以工作，除非你在下载 Git 的时候需要互联网连接

此外，您可以创建分支模型的不同分支

这将允许你尝试新的想法，这样你可以探索更多的东西，也可以回到你的旧分支，如果你想，你可以删除或合并它

当你把它下载到你的系统中，你只需要安装它，然后你就可以开始了

如果你正在学习数据科学，我相信你一定听说过这个术语，因为这对于你添加到简历中非常重要

这将有助于你在人群中脱颖而出，这对你也有好处，因为你已经学会了一些新的工具

现在，我们已经讨论了什么是 Git？现在，让我们来谈谈什么是 Github？

因为，如果你刚刚学习了一个工具，那么你应该也很擅长另一个工具，因为这将帮助你把你的项目直接放入你的 GitHub 个人资料中

# 什么是 GitHub？

现在，正如你所知道的 git，那么第二个术语也很简单

其背后的基本思想是 Github 只不过是一个存储库托管服务

读完这篇文章后，你可能会问，什么是存储库托管服务？

那么，伙计们，存储库托管服务就像一个在线数据库系统，你可以跟踪你的项目，并且可以保存你本地系统上的所有项目

但是，这里有一个问题，这项服务不是免费的，现在这里有另一个问题

关键是，如果你想用它来创建你的个人资料，你也可以免费使用这项服务

这意味着，你只需登录 Github 的官方网站，就可以创建你的 GitHub 个人资料

因为，当你在 GitHub 上创建你的个人资料，然后在 GitHub 上上传你的项目时，你就可以把你的 GitHub 个人资料添加到你的简历中，就像我之前告诉你的那样

这在招聘人员面前产生了非常好的影响，利用这一点，他们也可以看到你已经完成的项目

使用 GitHub，你可以与他人或你的朋友分享你的作品或程序，这样他们就可以对其进行编辑，并可以随心所欲地进行修改

您创建的分支将帮助其他人将他们的代码写入新文件，而不会覆盖其他人的代码

GitHub 还有一个好处是，当你创建分支时，无论你对程序文件做了什么改变，这些改变都不会反映在主文件中

这意味着，你将创建另一个文件，它是分支的副本文件，然后你和你的朋友将处理这些副本，而不是主文件

如果你想把它作为主文件，或者你想把所有的修改都反映到主文件中，那么你可以使用 push 或 pull 函数

所以，伙计们，Git 和 Github 是不同的术语，但却是相同的，因为许多人认为这两者是相同的，做的是相同的事情

但不是这样的，git 是一个版本控制系统，它会跟踪你所有的工作，或者你正在做的源代码项目

Github 只不过是一个基于云的托管服务，帮助你管理你创建的所有存储库

如果您不知道术语存储库或我们在本文中讨论过的任何其他术语，请不要担心

因为在接下来的文章中，我们将会看到所有这些术语，这样你就会了解它们

使用它，您可以轻松地将您在本地系统上创建的所有项目推送到 GitHub 存储库中

为此，您需要遵循不同的命令，以便将所有这些项目添加到您的 GitHub 配置文件中
