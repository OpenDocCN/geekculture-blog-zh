# Java 的推出和起源

詹姆斯·高斯林和太阳微系统公司在 1996 年创造了 Java 作为一种高级编程语言。Java 的 C/C++语法编写简单，但功能强大，足以为现代应用程序提供功能。

Java 起源背后的原则是程序可以在软件开发者选择的计算机上编写，或者可能是他们唯一可用的计算机。

然后，专家们将在那台机器上编写的程序编译成一种中间形式，这种形式可以在任何装有“Java 运行时”的设备上运行。

当用户运行 Java 程序时，目标计算机上的运行时将读取这个中间字节码形式的程序，并将其转换成本地代码，然后实际运行。

这个两步过程意味着 Java 程序员可以避开操作系统和硬件的差异，否则这些差异会使代码不兼容。

程序员可以针对任何平台，只要 Java 运行时可用于期望的目标计算机的配置。

这种广泛平台的一个结果是 Java 是当今最流行的编程语言之一。

Java 很容易学，但是真正的挑战来自于库(Java API)和依赖链，它们可能很复杂。

当移植到较新版本的 Java 时，API 也带来了挑战。这也是为什么很多用 Java 写的企业应用还在用 Java 版本 8，而最新版本是 Java 版本 17 的原因之一。

# Object Pascal 和 Java 都是面向对象的

尽管 Object Pascal 不是严格的面向对象，但它具有所有必要的面向对象编程特性，如继承、封装和多态。Java 是完全面向对象的。

Java 试图解决 C++中的问题，并取得了很大的成功。使用 Object Pascal，您可以从面向对象和过程化编程这两种编程范式中获得最佳效果。

面向对象编程使用对象和类而不是函数来表示现实世界或抽象元素。使用对象可以使代码更容易重用和维护。

一个子类可以继承另一个祖先类的数据和方法，因此您可以避免重复相同的代码来实现子类共有的共享功能。

# Object Pascal 和 Java 有什么不同？

# 目标 Pascal 的快速编译程序

对象 Pascal 编译器——尤其是 Delphi——通常非常快。Object Pascal 直接将代码编译成机器语言，而 Java 使用中间标记化的字节码，需要像 Dalvik Virtual Machine (DVM)或 JVM 这样的解释器来执行。

与 Java 不同，Object Pascal 在执行时不需要转换，也不需要运行时的存在，因为程序通常是独立的，很少或没有额外的依赖性。这要求它成为商业应用和游戏等复杂软件应用的理想选择。

# 与对象 Pascal 相比，Java 是冗长的

Java 严重依赖于具有冗长语法的类、接口和泛型，这使其成为重复的“样板”或脚手架代码来描述和实例化它所使用的对象。

Object Pascal 易于阅读、书写和学习。

它设法避免属性的重复代码，但是总体上对这种语法修饰的需求保持在最低限度。
