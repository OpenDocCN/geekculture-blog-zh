产品方面

看现实生活作业，详细解答

通常，面试产品管理职位的公司会要求应聘者完成一项任务。在这个系列中，我将展示一些最有前途的希腊公司所使用的真实作业的详细回答。

理想情况下，我试图涵盖以下所有要点，但我可能会根据任务范围有所不同。

  1. 设定背景:明确设定，了解公司(SWOT分析、使命、战略、约束、风险)。对市场、公司及其战略有一个高层次的心理模型是非常重要的，这样才能确保努力方向正确。
  2. 构思并区分优先顺序:列出几个选项并区分优先顺序。对于后者，我更喜欢使用DHM模型的扩展版本。
  3. 定义MVP: 强调MVP的关键特征以及我们应该测试的风险更高的假设。将假设与衡量标准列表结合在一起，以此来判断测试是否成功。想想当MVP失败时会发生什么。
  4. 超越MVP :作为创建MVP的一部分，我们需要为它创建一个路线图(优先考虑最好的特性，而不仅仅是最容易的)。我们还应该将MVP路线图与更大的产品路线图结合起来(如果在任务的上下文中知道的话)。

注1 :考虑到大部分考生会从互联网搜索获得自己的信息，可以肯定的是有些假设可能不准确。没有人会因此而惩罚你——请做出必要的假设(关于你发现的任何数据差距、业务限制、策略等)。).主要目标是突出你的思考过程。

注2 :考虑到本系列的所有答案都是基于真实的面试任务，我不得不强调你不应该复制提供的答案并作为自己的答案提交。首先，这是不道德的，我明确反对。也不要忘记，有问题的公司已经看到了答案，他们应该很容易发现任何抄袭。最后但同样重要的是，他们可以简单地改变问题:-)

本系列的目的是表达一些关于产品管理的要点，最重要的是，学习(所以请分享您的意见)。

介绍够了，让我们切入正题。

案例研究1: 蓝色背景

Blueground的业务是从业主那里租赁房屋，对其进行翻新和装修，然后以更高的价格将其转租一个月以上。Blueground已经获得了超过7700万美元的资金，目前在全球管理着大约4300处房产。

他们布置的作业的要点是:

第一步:提出帮助Blueground的想法，并解释你选择它们的原因。
第二步:定义MVP，如何评价其成功。
步骤3 :描述MVP如何发展，给出路线图。

正如你所看到的，这是故意简单和广泛。换句话说，这是展示你思维方式的最佳场所。整个响应分为4个部分:

  • 第一部分:设置地面。
  • 第二部分:构思和优先排序。
  • 第3部分:定义MVP。
  • 第4部分:超越MVP的思考。

案例研究2: 学习世界

LearnWorlds是学习领域的国际顶级参与者之一，“凭借其 无代码、白标、一体化平台，为教育工作者创建、主持、营销和销售高度互动和引人入胜的在线课程。“LearnWorlds最近获得了3200万美元的资金用于扩大规模。

分配给产品经理职位候选人的任务可以总结为:

第一步
A .列举异步(即非直播)视频作为学习工具的几个缺点。提出关键绩效指标来评估你的产品的表现。

第2步
提出一些关于让视频更具社会性的改进，并描述一些用户故事。

第三步
创建原型。

虽然在这种情况下，LearnWorlds会询问具体的事情，并且没有给我为Blueground创建的四部分答案留出空间，但我们仍然可以丰富我们的答案并展示我们的基本原理。
