# 每个初级 UI/UX 设计师都会犯哪些错误

> 原文：<https://medium.com/geekculture/3-mistakes-as-junior-ui-ux-designer-5d1532392a17?source=collection_archive---------13----------------------->

在一家初创公司做了一年的 UI/UX 设计师后，我明白了当你作为初级设计师开始构建东西时，你就开始犯错了。

它可以是任何事情，例如没有做适当的研究就开始项目，或者没有与其他设计师交流。

在这篇文章中，我将分享我过去犯的错误，我相信这些错误会帮助你成为一名成功的 UX/UX 设计师。

# 第一个错误:不主动做事

在我作为初级设计师的最初几天，我常常等待资深人士告诉我需要做什么。因此，我开始与我的经理以及其他设计师沟通。

如果有人没有告诉你需要做什么，我会建议你开始着手做一些事情，开始提出新的想法。

# 第二个错误:没有做适当的研究

作为一名初级设计师，你必须有设计想法，你所做的是，开始在你的设计中实施它们，不做适当的研究，也不遵循 [UX 过程](/ux-planet/the-evolution-of-ux-process-methodology-47f52557178b)。

我当时不知道自己在做什么。我以前觉得好的设计是完成了用户的需求和想要，其实不是。在开始任何项目之前，我强烈建议你做适当的研究。这会节省很多时间。

# 第三个错误:沟通和时间

在我们的设计世界里，等级制度是最重要的，截止日期通常是最重要的因素。

作为一名初级设计师，我曾经为一项任务工作几个小时甚至几天。然而，当我意识到它不会很快结束时，这种缺乏沟通往往会浪费大量的时间和资源，导致更多的压力和失望。
