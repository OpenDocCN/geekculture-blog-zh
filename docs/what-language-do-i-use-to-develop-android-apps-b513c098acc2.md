自 2008 年推出以来，Android 一直是一个开源操作系统。谷歌在 2005 年收购了 Android，这一战略承诺继续推动其高需求，因为第三方手机制造商可以利用这一平台。由于制造商不需要获得许可或构建自己的操作系统，因此在可预见的未来，他们将继续发布运行 Android 的设备。app 开发也是如此。由于 Android 是开源的，开发者可以灵活地使用多种编程语言和框架。

# 原生还是 Web？

决定用哪种语言来[开发一个安卓应用](https://upstack.co/hire-developers/android)完全取决于你想完成什么。如果你想让你的应用利用 Android 设备的特性，构建一个原生应用是更好的选择。然而，如果你想让你的应用在其他平台上运行，比如苹果的 iOS，一个利用桥接技术的网络应用，比如 PhoneGap，可能会更好地为你服务。

# 原生 Android 应用开发

自从第一批移动设备推出以来，原生应用程序的开发一直在不断发展。尽管 web 开发可能成本更低，上市时间更短，但构建原生应用程序有几个明显的优势。它们通常更安全，提供更好的性能，并允许开发人员完全访问设备功能。

[Android Studio](https://developer.android.com/studio/features) ，Android 应用的官方集成开发环境(IDE)，让开发者能够使用 [Kotlin](https://upstack.co/hire-developers/kotlin) 、 [Java](https://upstack.co/hire-developers/java) 和 [C++](https://upstack.co/hire-developers/c) 构建原生应用。选择最适合你的语言取决于你的经验水平和你想要达到的目标。

## 科特林

Kotlin 是 Android 开发的官方首选选项。它是一种免费、开源、静态类型的语言。像 Java 一样，它也是多平台、通用的，并且被设计为在 Java 虚拟机(JVM)上运行。

Kotlin 是一种流行的 Android 开发语言，因为它提供了几个独特的特性。它可以与 Java 互操作，这意味着在 Java 项目中使用它不会有任何困难。它也更加简洁，所以您可以用更少的代码获得相同的功能。Kotlin 还提供了一些高级特性，比如字符串模板、运算符重载和 Lambda 表达式。它也更容易学习，因为它是静态类型的，不容易出现编译时捕捉到的错误。

## Java 语言(一种计算机语言，尤用于创建网站)

Java 是另一种官方支持的 Android 应用开发编程语言。Sun Microsystems 于 1995 年发布，它比 Kotlin 要老得多，存在的时间也长得多。当 Android 首次推出时，Java 是该平台的首选编程语言。科特林要年轻得多，在过去几年才开始走红。

Java 的许多特性使其成为构建 Android 应用程序的可行选择。它还具有同样的静态类型的优点，使它更健壮，更不容易出错。它支持面向对象的原则(OOP)，使其更具可伸缩性、可扩展性和灵活性。因为它已经存在了二十多年，所以它有一个很大的社区和支持基础，并且也是更安全的编程语言之一。然而，与 Kotlin 相比，它要复杂得多，并且有着陡峭的学习曲线。

## C++

C++是另一种可以用来创建原生 Android 应用的跨平台开发语言。1985 年发布，比 Java 大十岁。即便如此，根据 GitHub 的统计数据，它仍然是十大编程语言之一。像 Kotlin 和 Java 一样，C++是独立于平台的。它也是一种通用语言，在不同技术的各种用例中使用。 [Android Studio 支持 C++](https://developer.android.com/studio/projects/add-native-code) 配合 Java NDK。

# 网络应用

如果你的目标是提供移动友好的内容，把它构建成一个 web 应用而不是一个本地应用是一个可行的选择。一组 [HTML5](https://upstack.co/hire-developers/html5) 、 [CSS](https://upstack.co/hire-developers/css) 和 [JavaScript](https://upstack.co/hire-developers/javascript) 可以为你提供在 Android 设备上渲染应用所需的所有基本功能。构建一个 web 应用程序的主要优势是你有一个跨移动平台的单一代码库。它使得开发和维护更具成本效益。这并不意味着如果你只打算在 Android 上发布，你就应该忽略网络应用。他们确实有他们的优势。Web 应用程序更容易编码，你使用的技术也更简单和用户友好。然而，在为 Android 开发移动应用程序时，它们的缺点可能是一个决定性因素。

# Web 应用程序的缺点

Web 应用程序无法像本地应用程序一样访问设备功能。除非你用 PhoneGap 这样的技术包装应用，否则用户需要使用浏览器来访问你的应用。它们也比原生应用程序更慢，反应更慢。

# 渐进式网络应用

渐进式 Web 应用程序(PWA)是常规网页和本机应用程序的混合体。移动浏览器在过去的十年里取得了长足的进步，现在提供了更先进的功能。PWAs 利用这些功能为用户提供更直观、更具互动性的体验。这些 PWA 功能包括发送推送通知和访问硬件组件，如振动。

# 关注用户体验

最终，你选择的语言应该取决于你想要构建的应用程序的类型，你需要的时间，以及项目的目标。网络应用成本低，开发时间短得多。然而，它们无法提供本机应用程序的速度、性能和用户体验。本机应用程序提供了更多功能，因为它们直接与设备交互。然而，与 web 应用技术相比，这是一条陡峭的学习曲线。

如果你是一名移动应用开发者，你需要同时具备开发本地和网络应用的能力。Kotlin 和 Java 都是 Android 应用程序开发支持的语言。也支持 C++，但是你需要通过 Android Studio 上的 Java NDK 来工作。这些语言都非常适合构建 Android 应用程序。然而，如果您也有 HTML5、CSS 和 JavaScript 技能来创建符合您项目的用户体验目标的应用程序，这将会有所帮助。
