# ABC 公司的方法

让我们考虑一个组织 ABC，它有许多保存不同类型信息的业务应用程序。应用程序以点对点的方式集成，引入新的应用程序已经成为主要的开发工作，因为有许多系统需要集成。由于这个原因，应用程序之间的互操作性已经成为内部和外部利益相关者的一个主要挑战。

ABC 公司的首席技术官决定启动一项新计划，对组织进行数字化转型，并遵循 API 优先战略，以实现跨系统和合作伙伴生态系统的互操作性。作为第一步，他建立了一个由以下角色组成的架构评审委员会

*   商业领袖，
*   企业架构师，以及
*   一些聪明的开发者

确定向消费者和合作伙伴交付最高商业价值的 API，并通过易于访问的门户(市场)公开它们。

该团队已经初步确定了 ABC 公司的业务目标，并根据业务需求调整了 API。与此同时，他们引入了一组度量标准(KPI)来衡量构建 API 的进度和业务方面的使用。他们坚持的另一个主要概念是将每一个 API 都视为一个产品，并给予每一个 API 与公司中任何其他产品相似的突出地位。这家 ABC 公司是一家制造公司，在其整个业务运营中体验到了集成供应链的价值。

# 使用 WSO2 API 集成平台提高基于 KPI 的投资回报率

ABC 公司希望将 API 计划作为一个试点项目开始，并希望在不花费大量资金在专有供应商身上的情况下开始开发工作。由于这个原因，该团队一直在寻找开源技术平台，这些平台可以用来以最低的成本启动项目，同时在开发阶段提供完整的功能集。他们评估了几个开源平台，并决定继续使用 [WSO2 API 管理](https://wso2.com/api-manager/)平台，原因如下。

*   最全面的功能集，包括

—开发者门户

货币化

—云原生支持

—流式 API

*   全面的安全性(OAuth2、JWT、API 密钥、mSSL、基本身份验证)
*   部署选项包括内部部署、云部署、多云部署和混合部署
*   开源版本中的完整功能集
*   灵活性和可扩展性
*   竞争定价模型
*   Forrester Wave 报告中的强大领导地位
*   集成 API 供应链的故事
*   完整的平台，内置集成和安全性

一旦平台选择完成，一个小的开发团队就被分组到一个敏捷团队中，并开始为项目工作。此时用户故事已经定义好了，团队从与业务目标最相关的故事开始。 *WSO2 为希望在初始 POC 阶段与 WSO2 合作的客户提供评估支持。这种支持不需要客户支付任何费用。*

# 第一个用例—通过 API 吸引合作伙伴

第一个用户故事与业务运营直接相关，旨在向向消费者销售产品的合作伙伴经销商公开 API。在此 API 发布之前，合作伙伴必须到 ABC 公司办公室了解最新产品、库存和保修信息的详细信息。借助新的 API，合作伙伴经销商现在可以通过使用 API，使用自己的应用程序立即获得这些信息。有了这个 API，集成到系统中的新合作伙伴的数量在短时间内增加了，合作伙伴的销售额也显著增加，因为他们不需要等到库存到达他们家门口，他们可以立即发送订单。

# 第二个使用案例——与快递员合作进行在线销售

随着该 API 的成功，该团队开始构建新的 API，通过他们的网站在线销售产品，并允许快递公司与该系统集成，以将产品送到家中。这是通过 API 开发人员门户完成的，交付合作伙伴使用 API 来集成销售流程，并获得需要交付的新销售订单的通知。WSO2 API Manager 支持 Websockets API 来构建基于事件的 API，以便在订单准备就绪后立即向快递合作伙伴发送通知。

现在，业务领导层已经确定了 API 平台的价值主张，越来越多的 API 构建在该平台上。CTO 仍然与业务目标保持一致，并希望确保每个 API 都有业务价值，并且价值主张通过 KPI 来衡量。
