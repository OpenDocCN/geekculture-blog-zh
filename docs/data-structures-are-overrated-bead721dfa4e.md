# 数据结构被高估了？

> 原文：<https://medium.com/geekculture/data-structures-are-overrated-bead721dfa4e?source=collection_archive---------14----------------------->

每个开发人员都想知道答案的问题。

![](img/dcc14252581f85d473e7a46b45dfdda2.png)

树、尝试、地图、图表，我们没有计划任何野营旅行(虽然我们很想:)，这里我们谈论的是数据，更具体的数据结构。数据结构是全球软件工程师中最受欢迎和炒作的东西。计算机工程有许多方面，可能是编程语言、网络安全、计算机网络等等，但没有一个主题像数据结构那样被讨论得那么多。

你可能会想到不同的问题，什么是数据结构？这种炒作和流行是为了什么？这真的配得上这样的人气吗？我们将在本文中详细讨论数据结构及其炒作。

# 什么是数据结构？

正如数据结构这个词，本身就说它是数据的结构。数据结构由两个词组合而成。首先是数据，这意味着任何形式的信息、知识或价值集合；其次是结构，这意味着数据的任何形式、形状或收集或存储方式。总之，我们可以说数据结构是我们收集、存储和呈现数据的一种专门方式，以便它可以用于不同的目的。

![](img/38acee6b096ac3f561098b3f39ce0e0d.png)

数据无处不在。您浏览网页、观看视频、使用社交媒体，甚至您刚刚登录系统，数据都会被收集。所有这些数据可能以不同的形式存在，可能以结构化或非结构化的形式存储在不同数据库的一些表格中，或者存储在一些集中的数据湖中，以后可能用于不同的目的。所有这些类型的存储都有一个共同点，那就是使用这些数据或对这些数据执行一些操作时，需要有一些特定的结构。这就是数据结构发挥作用的地方。例如，我们可能希望以某种分层的方式来表示数据，因为我们将使用树数据结构来表示数据。

## 数据结构的类型:

根据数据在数据结构中的存储方式，数据结构有不同的类型。以下是不同类型的数据结构:

1.  线性数据结构:顾名思义，这些数据结构以线性方式存储数据，条目一个接一个。这些包括像数组、链表、栈、队列等数据结构。
2.  非线性数据结构:这些是数据结构，顾名思义，数据是以非线性方式存储的。这些包括像树、图等数据结构。

![](img/d5d07df165127c6747924f9b0de88cdc.png)

Types of Data Structures

每种类型的数据结构都是根据不同的用例构建的。例如，当我们希望存储一个数据列表，以便可以从任何索引中随机访问值时，我们可能希望使用数组，如果我们不想使用索引访问值，并且我们没有大块的连续空间，我们可能会使用链表。

> 你会注意到我在上面的段落中用了“可以”而不是“必须”。这是因为使用任何数据结构都没有硬性的规则，但是每个数据结构都有一些特定的属性，这些属性使得使用特定的数据结构最适合特定的用例。

# 数据结构的重要性？

当你开始从事任何与计算机有关的工作时，数据结构无处不在。我们可以说，‘**有数据的地方，就有数据结构**。随着技术的进步，越来越多的人开始上网并分享数据。Snapchat 用户分享了 527，760 张照片，120 多名专业人士加入了 LinkedIn，用户观看了 4，146，600 个 YouTube 视频，在 Twitter 上发送了 456，000 条推文，Instagram 用户发布了 46，740 张照片所有这一切都发生在一天中仅仅**的一分钟内。**现在由你来想象这将是怎样的数据。这只是大局的一小部分。

![](img/007a0b22d2776c5acaef8041edbce7b4.png)

如果没有数据结构，这一切都是不可能的。当你在网上浏览 Instagram 的故事和卷轴时，你想在每个卷轴后缓冲和加载吗？….不要！！你在互联网上可能有的所有流畅体验，都是因为对数据或间接数据结构的高效利用。这就是数据结构变得越来越重要的原因。随着要处理的数据的规模和数量的增加，数据结构的重要性也增加了。

# 为什么炒作？

到目前为止，我们已经了解了什么是数据结构，数据结构的类型以及为什么它很重要。但是，开发人员大肆宣传数据结构的原因是什么呢？

随着一切都上网，甚至几年前需要人工干预的工作现在也开始自动化。AI/ML 让我们的生活更轻松，公司利润更高。许多大公司正在向他们的技术人才投入大量资金，这些技术人才不仅可以处理如此大规模的数据，还可以有效地利用数据来充分利用数据。处理这一切需要大量的技能。你可以在这个系统的任何部分工作，但是你总是需要的一个共同技能是数据结构的技能。数据结构，并成为与计算机科学相关的一切的支柱。你想想计算机网络，网络安全，像 AI/ML 这样的热门话题，你说出它的名字，数据结构就在那里。

这导致了计算机工程专业的学生想加入任何大公司并在其中工作，他们关注的是其他的一切+数据结构。因为需求和炒作总是相伴而生，所以才有了炒作。即使是公司也希望拥有掌握一些技能+数据结构的工程师，因此大肆宣传。

# 现代采访

随着工业开始走向自动化并在工作中采用计算机系统，对计算机工程师的需求开始急剧增加。每个公司都想雇佣最优秀的人才来帮助公司支持和发展技术。当招聘开发人员时，需要测试特定候选人的各种技能，因为该行业强烈要求一种通用技能，可以作为选择候选人的基础。这就是行业选择数据结构作为测试技能的基础。

![](img/759c087f319838fb90fbf53aae4cb5dd.png)

每个科技公司开始使用数据结构作为选择候选人的基础。这是数据结构开始在计算机工程师中越来越流行的时候。虽然许多公司也采用了数据结构的完整流程来测试候选人，但说实话，数据结构仍然是最重要的参数之一。这导致了计算机工程师心理的变化，数据结构是他们准备工作的中心。

# 竞争性编程

这种情况一直持续到面试，但是数据结构现在已经超越了他们目前对面试的推动，达到了成为工程师竞争的水平。这导致了像“竞争性编程”这样的运动/挑战的产生。

![](img/04921c95927dff7e088ad244af8446d5.png)

开发人员正聚在一起解决每月、每周和每天的挑战。这催生了许多有竞争力的编程网站，如 CodeForces、CodeChef、LeetCode、HackerRank、HackerEarth 等。开发者们从这些网站中脱颖而出。一般来说，人们认为职位越高，获得理想工作的机会就越大。甚至连公司都没有落下。像谷歌这样的公司会定期组织不同的比赛，开发者可以参加比赛来展示他们的技能。

现在，让我们回到当前的问题上来，

# 数据结构被高估了吗？

我的回答是否定的和肯定的。

No**为什么？**我们之前讨论过的数据结构是与计算机相关的一切的核心。作为一名开发者，人们很容易理解它的重要性。你可能写了一行代码，并认为你没有使用过任何数据结构，但事实并非如此，数据结构是任何现代编程语言的基础。现代编程语言充满了内置功能，很难注意/理解数据结构的使用，结果是许多开发人员轻视数据结构，因此他们的代码质量受到影响。人们应该理解数据结构，以便最好地利用编程语言。相信我，数据结构的重要性只会随着数据及其使用的增加而增加。说出来不会有错

> ddevelopers 在数据结构方面做得一样好。

Yes，为什么？毫无疑问，数据结构是重要的，并将继续重要。但所有围绕它的炒作真的不值得。许多开发人员在数据结构方面考虑过多。是的，竞争性编程提高了解决问题的技能，但这也是一个事实，最好的竞争性程序员可能不是最好的数据结构理解者。公司和他们的面试结构只是放大了这种情况。如今，许多开发人员在竞争性编程中落后，尽可能多地解决问题，甚至死记硬背。但是对数据结构的理解不需要。人们应该关注的是发展整体技能和对数据结构的理解。这也导致许多优秀的开发人员错过了不同的机会。

# 结论

在本文中，我们深入研究了数据结构和数据结构的类型。然后我们检查了什么是数据结构的重要性。然后我们看了看数据结构背后的炒作。最后，我们试图回答我们的问题，****数据结构是否被高估了？。**我们可以得出结论，数据结构是计算机科学最重要的部分之一，但我们应该尽量避免对数据结构的过度描述，应该专注于理解数据结构。**
