# 要点

1.  **理解一个新代码库的最大困难-** 不缺少评论。写得不错的代码。不，也不是写它的语言。这些都是事实，但它们都是一个更大问题的例子。你很难理解一个代码库/新项目，因为你不理解代码库背后的上下文。
2.  **好吧…那又怎样-** 要快速理解一个代码库，就要理解其中的原因。为什么会存在？它在解决什么问题？然后继续讨论如何解决这个问题。不是技术细节。只是什么让它有用。在此之后，获得对系统的每个大组件的高层次理解，以及它们如何对期望的最终目标做出贡献。不是具体细节，只是他们存在的目的是什么。
3.  在这之后- 现在你已经整理好了背景步骤。是时候进入第一步了。选择你认为你最有机会理解的部分。对我来说，这将是人工智能/处理系统。根据你的技能，这将是不同的。那就说得具体点。选择这个组件的一个非常小的部分(我喜欢使用管道的结束功能),并很好地理解这个部分。一旦你知道那块是什么和它做什么，从那里开始扩展你的知识。
4.  **如果卡住了怎么办-** 如果你被某个特定的组件/步骤卡住了，跳过它。向前看。把它当成一个黑盒(*当然机器学习的家伙这么说*)。只要理解它把什么作为输入，把什么作为输出。其他的都跳过。没有必要用你的头去撞墙。给你足够的时间，你会学会你必须学会的东西。
