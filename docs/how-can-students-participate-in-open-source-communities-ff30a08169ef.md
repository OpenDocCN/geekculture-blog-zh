# 学生如何参与开源社区？

> 原文：<https://medium.com/geekculture/how-can-students-participate-in-open-source-communities-ff30a08169ef?source=collection_archive---------7----------------------->

![](img/af88e4940c24a329aa5be6b7c8d76d94.png)

对于当今的开发人员来说，拥有一些开源项目或社区的经验是很常见的。事实上，不仅是成年人，学生也应该，而且越来越有可能参与到开源项目中来。

如果您想了解更多关于为什么您应该考虑成为开源项目的一部分，请参考“为什么您应该参与开源社区”。

退一步说，过去的两年充满了挑战。新冠肺炎·疫情在很多方面永远改变了我们。它改变了我们接近和快乐生活的方式，我们的工作方式和我们的社交方式——最终使网络研讨会或在线活动成为新常态。

学生也受到了影响。在线学习意味着他们必须适应吸收课程的方式，还必须适应在这个前所未有的时代寻找第一个职业机会或实习机会。网上实习是学生度过这段艰难时期的一个选择，他们可以充分利用这段时间，获得宝贵的经验。为了连接世界各地的人们，谷歌[ [2](https://summerofcode.withgoogle.com/) ]、Anitab[ [3](https://anitab-org.github.io/open-source-day/) ]和 ISCAS[ [4](https://summer.iscas.ac.cn/#/homepage?lang=en) ]正在为学生举办许多在线开源项目或黑客马拉松。的确，大部分都是免费的，有些甚至相应地为学生和导师提供奖学金和学费。

简而言之，通过加入开源社区，学生将获得以下好处:

-练习他们的技能

-开启实习机会或获得更多实习经验

-与世界各地的同伴和导师建立联系、建立纽带并开展合作

-获得奖学金

> **下一个问题是如何利用这些有吸引力的程序加入开源？**

不同的程序有不同的且通常是特定的规则，尽管基本过程通常是相同的，即:

-选择与相关项目合作的开源项目或社区

-提交学生申请

-联系你的导师

-贡献&编码(你可以用多种方式做出贡献，不仅仅是编码)

等待评估

-收集最终结果

基于这些步骤，下面是我在 GSoC[ [2](https://summerofcode.withgoogle.com/) 、OSD[ [3](https://anitab-org.github.io/open-source-day/) ]和代码之夏[ [4](https://summer.iscas.ac.cn/#/homepage?lang=en) ]的指导经历中得到的一些重要提示，我很乐意与你分享。

*   **选择合适的程序**

正如我在上面提到的，许多组织正在启动具有不同时间表、任务持续时间、资格和奖学金条件的开源项目。你最好为各种各样的项目做一些研究，然后选择一个最适合你的。

*   **提前选好一个项目:**

如果你想要一张优先入场券，千万不要等到正式截止日期才选择你的节目。通常，这些项目的委员会将选择优秀的开源社区作为他们的合作伙伴来提供指导。例如，在成为 GSoC 官方导师之前，我们已经在等待公众申请时创建了许多候选任务。

我们欢迎学生提前发表意见。这使得学生和导师有足够的时间互相了解。也许一个学生不一定是合适的，但他们的主动性会给导师留下深刻印象，他/她会被选中。

*   **简明扼要地写下你的建议:**

这个项目的提案应该是简洁的而不是多余的。要做到这一点，建议你参考导师的任务细节(有些导师会详细描述任务，有些不会)或者直接问导师的期望和关注点是什么。当涉及到任务或任务的目的时，猜测或在一厢情愿的幌子下操作是准备你的提案的最低效的方式。导师和学生都希望事情顺利有效地进行，因此不要羞于向导师寻求帮助。

*   **积极联系导师:**

假设你的提议被接受了，那么下一阶段就是投稿或者开始编码。这是你产生影响的一个重要机会。

你可能是一个编码奇才，但是你会遇到一些你不确定的问题，或者有时候你不知道你的导师对你的工作有什么看法。你还应该考虑到，有些导师一次负责两三个学生，同时忙于他们的生活和工作。话虽如此，但主动联系你的导师问问题或定期汇报你的进展是有益的。如果你等到你的名字出现在你的导师的脑海中，那是行不通的，因为当它出现的时候，可能已经太晚了。

*   **棘手问题:**

虽然有必要与你的导师保持密切联系，但这并不意味着你是一个婴儿。事先考虑一下你会问什么问题，自己做些调查和研究，看看这是不是可以用你的批判性思维或者每个人信任的朋友——谷歌来解决。

如果你做不到这一点，导师可能会认为你缺乏解决问题和分析能力，或者更糟，认为你不愿意通过实践来学习。如果你在做了研究后仍然感到困惑，那么用特定的术语，在你的研究报告的支持下，问你的导师你的问题。这不仅能确保你得到导师的关注，还能让导师明白你确实有些困难，你会得到导师的全心关注。

在指导了无数本地和国际学生之后，这是我可以与你分享的观点。我真诚地鼓励你看看上面提到的有趣和有意义的节目，以丰富你的学术生活，提高你的技能，扩大你的朋友圈。最后但同样重要的是，如果你有兴趣加入开源分布式数据库生态系统，我在这里等你[ [6](https://github.com/apache/shardingsphere) ]。
