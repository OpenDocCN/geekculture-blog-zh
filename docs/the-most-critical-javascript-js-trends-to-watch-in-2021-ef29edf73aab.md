# 打字稿的优点

# 可选静态类型

使用 TypeScript，您可以通过组合现有类型来创建复杂类型。为此，您可以使用联合或泛型。TypeScript 也支持接口。

# 早期错误检测

TypeScript 可以通过突出显示意外行为来减少程序错误的数量。这发生在编译时，而不是像 JavaScript 本身那样在执行过程中。通过这种方式，TypeScript 可以生成更高效的代码。

支持大型解决方案

用于创建大型复杂系统的 JavaScript，现代网络中充斥着这种系统。为了实现这一点，TypeScript 采用了一些面向对象的特性，如接口、泛型和模块。

# 广泛接受

大多数关键的 JavaScript 库和框架都支持 TypeScript。此外，TypeScript 可以编译成可读的、基于标准的 JavaScript。

# |轻松入职

对于熟悉 JavaScript 的人来说，切换到 TypeScript 并不是一个很大的挑战。

这就是为什么根据 Stack Overflow 开发者调查，TypeScript 在 2020 年成为第二受欢迎的语言。JS 的趋势表明，它的受欢迎程度将在 2021 年继续增长。

因此，TypeScript 的广泛接受是当前最有影响力的 JavaScript 趋势之一。然而，它也有其缺点。TypeScript 代码必须首先被编译成常规的 JavaScript 代码，以便由浏览器执行。在处理小项目时，这种延迟可能会很长。所以在这些情况下，JavaScript 还是比较好用的。

# |作为 web 构建块的 Web 组件

没人能预测五年后 JavaScript 框架会有多流行。但是 web 组件可以用在任何流行的框架中，甚至可以用在普通的 JavaScript 中。

这就是为什么这些现代 web 应用程序的构建块也被称为独立于框架或不可知的 web 组件。它们使 web 开发更容易，并减少了创建应用程序的时间和成本。

但是是什么让这个概念在其他 JavaScript 趋势中如此流行，它是如何工作的？它基于以下规范:

*   自定义元素允许开发人员创建具有完全可自定义行为的新自治 HTML 标签。
*   影子 DOM 允许 DOM 子树封装 web 元素。
*   ES 模块决定了如何在其他 JS 文档中包含和重用 JavaScript 模块。
*   HTML 模板通过在运行时实例化标记片段，提供了创建不影响页面加载的标记片段的能力。

当谈到最新的 JavaScript 趋势时，web 组件是大众的最爱。
