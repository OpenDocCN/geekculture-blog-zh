**为什么要用数据库进行机器学习？**

使用来自数据库的正确数据非常重要，这可以最大限度地减少更高效、更快速、更易于操作的机器学习所需的步骤。

数据库被广泛用于操作机器学习有一些原因

1.  ***为简单起见***

使用数据库为团队中的每个人提供了简单性，因为它提供了一些工具来轻松获得所需的数据。

2. ***最小化时间***

数据库确保最小化数据移动和更高的速度，从而节省时间和成本。

3. ***易制作***

在机器学习中使用数据库减少了复杂的部署工作。这使得数据库中的模型更容易部署和操作。

**机器学习用哪个数据库？**

在直接跳到机器学习使用哪个数据库之前，了解和理解不同类型数据库的用途是非常重要的。

数据库有两种类型

1.  本质上是关系或表格的 SQL 数据库。
2.  本质上是非关系或非表格的 NoSQL 数据库。

在机器学习中，我们可以使用任何基于 SQL 或 NoSQL 的数据库。但是，由于各种原因，各种 NoSQL 数据库在行业中被广泛使用。

在机器学习、计算机视觉和大型项目的自然语言处理方面，为什么选择 NoSQL 数据库而不是 MySQL 的一些原因？

1.  ***高度且易于扩展***

SQL 数据库可以存储大量数据，但只能存储在一台机器上，这是 SQL 数据库的最大缺陷。这就是 SQL 数据库也被称为垂直可伸缩数据库的原因，垂直可伸缩数据库意味着一个接一个地添加记录。

而 NoSQL 的数据库恰恰相反。NoSQL 数据库提供了一种存储来自各种机器的数据的替代方法。这就是为什么 NoSQL 数据库也被称为水平可伸缩数据库的原因，水平可伸缩数据库意味着在各种机器中添加记录。

2. ***动态模式或灵活数据模型***

SQL 数据库是关系数据库，因此数据库表的列或属性已经固定。不能添加或删除任何特定记录的属性。该系统限制用户将数据存储在固定的模式中。

而 NoSQL 数据库是无模式的，所以可以在没有任何预定义模式的情况下插入数据。因此，数据格式或模式可以随时更改，而不用担心模式。

3. ***少花钱维护***

SQL 数据库需要训练有素和高度规模化的人力来管理数据库，而 NoSQL 数据库更容易管理。它们提供了各种特性，比如自动修复、更容易的数据分发、没有预定义的数据模型等等，这些特性减少了数据库管理员的调优工作

4. ***交易速度***

SQL 数据库中的事务速度比 NoSQL 数据库慢。在 SQL 数据库中，光标搜索每个条目以查看满足条件的记录，但是在 NoSQL 中，搜索是基于记录的 id，这使得速度更快。

5. ***云计算***

基于云的数据库是不断发展的机器学习世界的解决方案。在机器学习中，数据不仅仅是数据，它是大数据。为了利用它来实现我们的目的，建立一些伟大的东西和大规模的生产，我们使用云计算将数据分散到多个服务器上，这只有使用 NoSQL 数据库才是可能的。SQL 数据库不用于处理大型数据集，而 NoSQL 是专门为处理大型数据集而设计的。

所有这些都是使用 NoSQL 数据库进行大规模生产的优势。

在了解了 NoSQL 数据库的所有这些优点之后，在选择哪个 NoSQL 数据库时可能会有些困惑。在市场上，有各种研究人员使用的 NoSQL 数据库，一些最著名和已知的数据库如下

1.  **MongoDB**
2.  ***阿帕奇卡珊德拉***
3.  ***【MLDB】***

**结论**

总之，让我们明白这个话题。数据库对于机器学习非常重要，因为它提高了操作的简单性，更快地处理数据，并且易于操作。

根据项目(大规模或小规模)，这两种类型的数据库都可以用于机器学习。MySQL(SQL 数据库)通过提供足够多的特性来满足小规模项目的需求。MongoDB(NoSQL 数据库)应用于生产项目并使其大规模运作。
