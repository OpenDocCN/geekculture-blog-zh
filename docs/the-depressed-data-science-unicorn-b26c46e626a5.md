假设我们已经找到了一个。

数据科学领域独一无二的独角兽。那个软硬技能完美匹配的人，你需要最终将数据转化为现金——那个可以独力完成所有事情的特别的人。从具有正确业务影响的数据科学使用案例的构思开始，建立一个强大的数据管道来生产质量有保证的数据功能，提供解决问题的最佳模型，并确保可持续性的性能监控。

抛开利益相关者管理、展示结果和路线图设计等中间任务，让它变得更加简单，让我们也假设(在那个独角兽世界中)有一个现成可用的数据科学技术堆栈。

# 鉴于这些看法和假设，我们又能从这种罕见的美丽生物身上期待什么呢？

在严酷的商业现实中，数据产品的上市时间也决定了数据科学用例的成功，我们期望这只孤独的独角兽具有竞争力吗？

如果是的话，可怜的小东西！但是让我们来看看这个:

这位预示着独角兽开发了一个数据产品的想法，并将其推销给决策者。这个想法得到了批准，每个人都对独角兽概念化的模型洞察力培育的业务关键型营销流程的承诺优化感到兴奋。

作为利益相关者，唯一不适合我们的是独角兽提供的预期上市时间。所以接下来可能会发生三种情况:

*在第一个场景*中，我们可以说:“这个想法太棒了，让我们去做吧，冒着上市时间长的风险”——在最坏的情况下(但并非不可能)，这可能会导致竞争对手赶上拥有相同想法但在更短时间内实施的人。期望中的优势将会消失。

不快乐的独角兽，不快乐的利益相关者。

*第二个*，我们可以说我们会采纳这个想法，但决定采用一种快速的方法，并降低重要的质量保证计划的优先级。但这造成了所谓的 [*隐性技术债务*](https://papers.nips.cc/paper/5656-hidden-technical-debt-in-machine-learning-systems.pdf)*，从长远来看会拖慢我们的速度，希望的优势也就没了。*

*不幸的独角兽，不幸的我们。*

**第三*，我们决定建议的一切都应该由 unicorn 实施，但要在更短的时间内实施。翻译过来就是工作-生活-平衡，这意味着:在时间压力下的巨大工作量会导致一只精疲力竭的独角兽。*

*你见过被烧毁的独角兽吗？它们拒绝进食，失去了生活的乐趣，久而久之成为一匹精神崩溃的普通马。最终，独角兽会离开我们，心甘情愿地加入*被火覆盖的公牛*与海里的其他独角兽团聚。而我们将再也见不到最后的独角兽[](https://en.wikipedia.org/wiki/The_Last_Unicorn)*。**

**在这三种情况下，独角兽除了变得沮丧，别无选择。**

**但也许还有一种情况，对我们和独角兽可能有用。如果我们有一个独特的数据产品想法，在这种情况下，我们的竞争对手会很快开发出一个相同的想法。如果我们的概念如此独特，即使上市时间很长也不会损害这个想法的成功，那么我们可能会有一个浪漫的结局。即使这样，也可能仅够产生一个原型或[最小可行数据产品](/idealo-tech-blog/what-is-minimum-viable-data-product-49269e338d85)。一旦涉及到操作和扩展一切，我们将再次需要一个正常工作的数据团队。加速并保持我们的竞争优势。**

**总之:在罕见的商业环境中，罕见的数据科学独角兽只能生存，而不能繁荣。至少在一段时间内。**

**嗯，现在我也很沮丧。**

# **一群独角兽被称为祝福**

***那么，如果我们最终需要一个完全专业化的团队，为什么还要寻找数据科学领域的独角兽呢？事实上，似乎没有现实的场景可以让独角玩家和我们一起快乐地工作。***

**我不怀疑独角兽的存在——在我的职业生涯中，我见过一些独角兽，但它们都以上述命运之一告终。我相信独角兽并不罕见，因为只有少数存在。他们很稀有，因为他们藏起来了。他们知道如果他们出现会发生什么。我们应该消除他们的恐惧，打开新的视野和视角，让独角兽茁壮成长。**

**我能想象到寻找独角兽的唯一原因是不想让它表现得像独角兽。我想要一只独角兽，因为我知道在领导数据产品团队时，它可以创造奇迹。他们监督整个数据产品开发过程，了解应该首先解决什么，并可以等待下一次开发迭代。我们不应该让他们在不合理的时间内自己实现数据产品的每个方面，而是应该鼓励他们让数据产品团队成员成为他们自己的最佳版本，并适当地交付。它可以组织数据工程师、数据架构师、数据科学家和团队中的所有其他数据专业人员，并允许他们成为专业化的独角兽。考虑到这一点，我们可以建造一群独角兽。**

**人们常说，一群独角兽被称为**福气；**同样，一个完善的专家团队也是一件幸事，不管心爱的独角兽是否存在。当你有机会找到一只数据科学独角兽时，勇敢面对，帮助理解它作为团队成员的重要作用，并支持它成为团队领导者。**

***听起来像个计划！但如果独角兽只是想成为一名数据科学家呢？*嗯，是的。这可能会发生，实际上，这很有可能发生，因为那只独角兽申请成为数据科学家，对吗？**

**在这种情况下，我们应该忘记上面写的一切，停止寻找独角兽，而是聘请一位能够轻松创造人工智能产品的产品经理，他可以领导一个数据专家团队。而且实际上，说实话，这甚至是各方的最佳解决方案。因为产品经理的职责就是组织、协调、挑战和鼓励一个伟大的专家团队去完成他们自己的任务，即使它需要对提到的中间任务负责。**

***等等！什么？* —是的。**

**我喜欢独角兽，但我们不需要它们，因为我们无法提供独角兽可以茁壮成长的童话环境。所以让他们自由吧。当他们准备好进入新的高度和维度时，他们就会出现。毕竟，[一个协调良好的专家团队也是一种福气](https://en.wikipedia.org/wiki/High-performance_teams)，对吗？**

**那为什么还要费心呢？**
