# 关于理智的思考

自从我开始在内容密集型应用程序中使用 Sanity 以来，已经有 4 个月了，我想分享一下我对 headless cms 的想法。在开发数据驱动的应用程序时，我们需要处理很多事情，比如资产管理、动态内容、数据安全性以及对数据的更好控制。理智提供了所有这些。

所以在健全之前，我尝试了 WordPress，在我看来这是一个定制的噩梦。把博客文章推给其他东西的想法对我来说不太好。(顺便说一下，我不是 PHP 的粉丝)但是我发现学习和定制 Sanity 很容易，文档也写得很好。许多平台上都有一个很好的社区，我发现它真的很有帮助。

所以我们使用 Sanity 作为前端的 cms 和 Nextjs。我觉得组合真的很好。他们相处得很好，适当使用 Nextjs 的特性，比如静态生成和服务器端呈现，就有可能优化站点的性能、SEO 和健全的数据带宽。

我认为动态定制整个网站是可能的，只要有理智、良好的想象力和更多的工作。数据建模真的很有趣。

我真的很喜欢资产管道。API 允许经过身份验证的客户端上传和删除资产。它支持几乎所有常见的图像类型和 pdf 等。目前，没有视频支持，我真的很想有这个功能，但至少我不用注册亚马逊 s3 bucket 或 Cloudinary 来上传文件。这些资产是通过全球 CDN 提供的，酷的是它们允许 URL 选项模糊、裁剪或调整大小等，这在提供优化内容方面有很大帮助。

所以总的来说，对于内容驱动的应用程序，健全是我的第一选择。有大量的插件可用于大多数场景，开发定制插件也是可能的。

我做的最有趣的事情是为健全工作室的现场预览。这真的很容易做到，并大大改善了编辑体验。对于任何使用 Sanity 和 Next.js 的人，我推荐使用静态站点生成，因为它将节省大量带宽，除非您希望内容在发布后立即得到反映。当使用静态站点生成时，内容将在下一次构建时更新，但会更快。

我真的认为我没有充分发挥这个平台的潜力，但是第一次，我真的对 cms 平台感到满意，并将继续学习利用 Sanity 提供的所有功能。
