## 和一些能帮助你从其他候选人中脱颖而出的建议

![](img/454478f198d98ef9471d3fde848f3964.png)

几周前，在一个缓慢的工作日修复 bug 的过程中，我收到了公司一位董事的消息。他问我是否愿意和他一起面试一些公司初级职位的候选人。我只花了几秒钟就回复了“我很乐意”，这肯定会是有趣的一天。

老实说，我很好奇这个过程会是怎样的。我只工作了两年，很幸运在短时间内找到了一家让我开心的公司。

但我也经历过糟糕面试的痛苦，经历过随之而来的各种不同的心态。到了年底，它不能告诉我是否知道如何编码了，变得哑口无言。

我读了大量的文章，听了播客，研究了几十个准备中的问题。所有这些都很有帮助，但我意识到这总是一个候选人的视角，或者面试官是一个身居高位的人。所以我想分享我的经历会是一个有趣的想法。

伴随着经验而来的是大量的学习。所以我也会分享一些对你有用的小技巧，让你领先于其他候选人。

# 声名狼藉的简历

在开始前几分钟，我会从候选人那里拿到简历，快速浏览一遍。这就是为什么这一步很重要，应该认真对待。

一份漂亮、简单、干净的简历会让面试官对你印象深刻。有时候参加面试的人并不总是来自人力资源部门(特别是在小公司)。你必须做好准备，这可能不是他们的主要工作，他们只有几分钟的时间来阅读你的简历，获得你的第一印象。这就是为什么保持简短、甜蜜和整洁是很重要的。

有一件事曾经以不好的方式引起了我的注意，那就是简历的**格式。是的，你可能会认为把文档上传到 PDF 或 Word 上很傻，但是相信我，不是的。**

你可能认为现在每个人都在使用 Word，所以这没关系。但是请做好准备:我没有安装 Word。我的工作电脑是一台 MacBook，除了 Pages 之外，我从未安装过任何文本编辑器。我也没有时间，我必须集中精力快速阅读。

所以当我看到一份有*的简历时。docx* 格式，当我打开它的时候，我以为我的眼睛要中风了(我知道这是不可能的，但你知道我的意思)。

有人花了这么大力气制作的完美文档或模板在我的眼前分崩离析了。我得到的是一大堆用红色标记的错别字、多余的空格和其他问题，这让我的阅读变得缓慢而痛苦。这让我记住了这位候选人，但不幸的是，原因是错误的。

所以将来为了安全起见，一旦你完成了，就把它转换成 PDF 格式。不管面试官用什么样的电脑或设备来阅读，它看起来都会很棒。

# 个人项目

作为一个热爱编码的开发人员，当我看到一个 Github 简介的链接时，这是我首先关注的。这是观察任何兼职项目和检查你的大学/训练营项目(如果候选人有的话)的完美方式。

有趣的部分来了:**我们喜欢凌乱的**。你可能会认为一个干净的项目，包含格式完美的代码、注释和大量高效的函数，才是你所需要的。但是你猜怎么着，这是否是一个重要的部分，有时候越乱越好。

不要担心提交时会有类似“我不知道这是如何工作的，但它确实是这样！”或者成吨的树枝。在申请入门级或初级角色时，有些候选人没有版本控制经验。任何表明你对此有所了解的东西都值得展示。

我喜欢挖掘某人的知识库，了解他们一直在做的事情的乐趣。这给了候选人一个讨论他们擅长什么的机会，也让他们有别于那些只用 Github 账户上传项目的人。

# 展示和讲解

你做过什么大学项目吗？你和一些朋友一起做了什么有趣的东西来尝试一些技术吗？你认为“我每天都有这个问题，我想找到一种方法来解决它”，你对此感到自豪吗？秀出来！

这让我大吃一惊。老实说，当一些候选人要求展示他们所做的一些东西时，我很震惊。我找工作的时候怎么没想到呢？

我自己花了这么多月的时间来完成我的大学毕业设计，为此我付出了血汗和泪水。我根本没想到要展示出来！

我在采访中提到了它，但我太尴尬了，人们会看到我不太专业的信息。一个例子是“*克隆存储库，因为我在圣诞节*搞砸了，我删除了我的整个项目。让我告诉你，这不是有趣的一天。

不要担心是否有不工作的部分，试着不要点击它们，没有人会知道😄。展示你一直在做的东西应该让你感到自豪，不管它有多简单。

# 网络摄像头框架

不幸的是，我们仍然处在一个 COVID 的世界里，几乎所有的采访都是远程进行的。而且大部分都是在家里，当你上身穿着花里胡哨的衣服，下身穿着睡裤的时候(我也去过)。

**拜托，拜托，拜托**确保面试官能看到你的脸和嘴。尽你所能适应这种情况，把自己放在一个不让人分心的背景中。确保你的整张脸是完全可见的。

![](img/316a8fa05ba972f9dede155f1cc1d4fa.png)

A perfectly framed webcam does wonders for a better communication

很重要的一点是，如果我和你说话时，我只能看到你的额头和眼睛，那么谈话会变得很片面，会很不舒服。我没有办法判断你是否在微笑，所以这个小提示可以大大改善面试过程。

# 身体语言

你紧张是可以理解的，但是你最不想让面试官认为你很无聊，很累或者你不想在那里。

我有一次面试，在那里我得到了很多“请结束这种折磨”的感觉，这并不令人愉快。这并没有反映整个面试，随着时间的推移，候选人似乎放松了一些，情况变得更好了。我没有让这影响我对这个人的印象，但其他人可能不会这么理解。

我知道这很难实现，但如果你知道这可能发生在你身上，你可以总是诚实地分享你的紧张。面试官不会吃了你，完全正常！

还有，小心叹气。一声叹息可能不引人注意，两声叹息令人担忧，第三声叹息是一个大红旗，它可以表明缺乏兴趣。翻白眼也是一个大禁忌，试着不去想它，即使事情可能不是朝着最好的方向发展。

# 服装服饰

我将保持这一部分的简短，因为我相信你知道这个话题:穿得比你日常的衣服正式一点。穿得多总比穿得少好。

![](img/cecea632c946fcc1f93bce611c2effb7.png)

额外提示:注意面试你的人的穿着。如果你不打算在偏远的地方工作，这会让你对公司的着装有所了解。

我故意穿便装，以表明我们公司没有正式的着装规范。我认为这将有助于候选人了解我们公司文化中的这一主题。

# 问问题

你对这个角色、公司文化或者任何你渴望知道的事情有什么问题吗？去争取一下，问一下。

我还建议在面试结束时，就你在面试过程中的表现征求一些反馈。面试官可能有一些对未来有用的好建议，或者一些你没有注意到的东西。

在我的案例中，我鼓励一个候选人探索一下版本控制世界，因为他没有这方面的经验。另一位面试官推荐了另一位应聘者去查看他简历上的链接，其中一位被重定向到一个错误页面。

# 不要过度紧张

说比做好，但是不要太担心。如果你没有副业可以展示，或者没有任何经验，也没关系。你努力了，你得到了这个，你已经达到了这个点。如果你得到了这份工作或者进入了下一轮，那么恭喜你！🎉

但如果你没有，那就不要惩罚自己。想一想你从这种情况下得到了什么，以及你犯了什么错误，这可能是下次改正的好方法。**把这当成一次学习经历，并不断推动自己走向下一次。**

# 结论！

最后，这对我来说是一次有趣的经历。当你是初级开发人员时，并不是每个人都足够幸运能够在面试中扮演积极的角色。我没有参与做任何最终决定，但提出问题、提供反馈以及与其他面试官讨论是很有趣的。

公司应该更经常地向这样的政策敞开大门。面试总是充满压力，这将有助于人们对这样一个棘手的话题有所了解。
