# 为什么选择客户数据平台？

**客户数据平台****【CDPs】**的承诺是将所有的客户数据统一到一个平台上。他们承诺实现有趣的用例，从典型的报告到高级用户激活场景，例如驱动个性化的 UI 体验和基于客户行为提供优惠。

那里的大多数 CDP 产品都是从其他产品开始，逐渐演变成今天的样子。这是有充分理由的。与客户相关的数据总是很重要，不仅是为了了解客户的行为，也是为了采取行动。此外，与客户相关的数据往往很大，原始，充满了问题和不一致。这些数据应该由没有技术技能的人使用，比如一个成长经理。这些是过去两三年中 CDP 得以存在的部分原因。

# 对数据驱动型企业的需求

如今，从他们搜索的产品，到他们点击或忽略的内容，再到最终的购买，可以真实地捕捉到你的客户的整个旅程。此外，还可以用来自企业每个客户接触点的数据来丰富这一旅程。接触点可以是任何东西，从客户支持和管理系统到营销平台和发票管理产品。

这一丰富的数据集可以为企业提供传统报告之外的更多使用案例，具有无限的可能性。这目前正在推动许多产品的开发，这些产品共同形成了一个完整的数据堆栈或基础架构，企业可以利用它来捕获、分析和生产该数据集。

这个数据堆栈的基础是数据仓库。这项技术在过去几年中发展如此之快，以至于它现在可以支持几乎无限的规模、存储和处理之间的分离、高级分析功能，甚至机器学习。因此**云数据仓库****【CDWs】**风靡一时，成为数据驱动型企业的主要推动者之一。

毫不奇怪，硅谷目前最有价值的初创公司之一是 CDW 供应商[的](https://techcrunch.com/2020/02/09/after-479m-round-on-12-4b-valuation-snowflake-ceo-says-ipo-is-next-step/)[雪花](https://www.snowflake.com/)，它刚刚以 120 亿美元的估值融资 5 亿美元。

# 为什么要在自己的数据仓库上构建自己的客户数据平台？

随着 CDW 技术的发展，现在已经有了构建定制的、可扩展的、高效的 CDP 的技术。但是，为什么有人更愿意构建自己的 CDP，而不是购买和使用现成的供应商解决方案呢？

原因有很多，但我们想把重点放在下面。

# 灵活性

无论现成的供应商解决方案有多成熟，它总是通用的。即使在同一个市场，每个企业的客户都是独一无二的。如何定义一个客户？企业有哪些数据来源？使用什么数据模型？应该如何识别客户？这些只是 CDP 必须解决的部分问题。要回答这些问题，最好的方法就是访问公司在数据仓库或数据湖中收集的所有原始数据。

随着业务的发展，越来越多的人需要与客户数据进行交互。每个人都需要探查数据。这包括使用 Looker 等 BI 解决方案的营销经理，以及使用 Jupiter 笔记本和高级 SQL 的数据科学家。没有人知道将来谁会使用这些数据，但几乎可以肯定的是，这个列表会越来越长。像围绕 CDW 或数据湖建立的数据堆栈这样的解决方案保证了公司拥有灵活的基础设施，允许每个人访问数据。

# 数据隐私和治理

我们正在进入一个时代，越来越多的控制将围绕来自不同行为者的数据展开，包括政府。公司将不得不遵守这些规定，并对他们管理的数据进行高度保护。客户生成的任何信息都将被视为敏感信息。围绕数据堆栈添加越来越多的产品，其中数据必须跨不同的工具复制，这使得隐私和数据治理成为一项棘手的工作。

解决这个问题的最好方法之一是依靠定义良好且行业认可的数据架构。在这里，数据存储在定义良好的存储库中，比如数据湖和数据仓库，并且实施了严格的数据访问和安全性策略。

因此，在您的数据仓库上实现您的客户数据平台也将有助于您公司的数据隐私和治理。

# 费用

我们总是专注于创造新的价值，但降低成本是技术的另一个重要影响。构建基础设施来处理如此大量的数据需要资金。加上 SaaS 的利润，我们正在考虑投资数十万美元来购买和部署第三方 SaaS 解决方案。相反，通过在现有数据仓库上构建客户数据平台，公司可以重用已经采购的云基础设施。

# 但是仅有数据仓库基础设施是不够的

没有数据，公司拥有的数据基础设施是完全无用的。因此，捕获、收集和交付数据的数据堆栈层极其重要。这一层负责以下内容:

*   帮助数据工程师一致、安全地将事件流数据从来源(网站或应用程序)收集到数据仓库中。
*   允许分析师和数据科学家推动上面讨论的高级分析和激活用例。同样，这一层应该提供一致性和数据质量相关的保证。这将使公司的上述数据公民专注于创造价值，而不是弄清楚数据是否应该使用以及如何使用。
