# 我希望我们在训练营学到的东西

> 原文：<https://medium.com/geekculture/things-i-wish-we-learned-in-bootcamp-1a05e4141bac?source=collection_archive---------40----------------------->

这是我的故事，讲述了我觉得训练营让我失望的 5 个方面。

在编程领域呆了 2.5 年后，我决定回到学校向专业人士学习。我曾是一名自由职业者，为中小型企业工作过，担任过网站开发项目经理。我想进入一个更高层次的职业，并决定回到学校接受一些正式的培训，提高自己的水平。

我知道新兵训练营是为了在你参加的短时间内给你尽可能多的基础，我的悲哀不在于核心课程，总的来说，我认为我学到了很多，我觉得这对我是一个好处。工作给了我一个不同的视角，我想我应该列出我希望他们在编码训练营中更多关注的五件事。这对于我所在的学校来说是非常特殊的，甚至对于我所在的群体来说也是如此，你的经历可能会有所不同。

1.  **GitHub/版本控制:**
    当我们和 GitHub 一起工作，并在团队中进行编程时，我们得到的曝光率与在真实世界的团队中工作相比相形见绌。
    GitHub 非常庞大，我不指望我们能全部学会，但是，我希望能多强调一下什么是好的 PR(拉式请求),如何正确地审核 PR，以及如何处理合并冲突。
2.  编程中有一句名言，永远不要花 15 分钟去做那些你可以花一个小时去自动化的事情。学习如何“艰难地”做这件事是有价值的，但我也觉得教会我们自动化如何提高我们的工作会是有益的。创建生成器甚至使用 [GitHub Actions](https://github.com/features/actions) 对程序员来说很大程度上是有益的，对团队来说更是如此。自动化就是你如何把一个开发人员变成一个[“10 倍开发人员”](https://www.techopedia.com/definition/31673/10x-developer#:~:text=A%2010x%20developer%20is%20an,10x%20programmer%20or%2010x%20engineer.)。作为一个新的程序员，在输入每一个按键时会有很多乐趣，但是一个专业人员应该知道他们的工具以及如何利用它们。
3.  **React Hooks:** 我参加的课程是全栈，我们学习了 P.E.R.N 栈，PostgresSQL，Express，React，Node。因为我们的 Javascript 库是 React，所以我认为我们肯定会学习 hooks，并且有点惊讶地发现我们的课程中完全没有 hooks。钩子使得状态管理更加容易，大多数使用 React 的社区都使用 React 钩子。
4.  编程基础:
    任何好的公司都需要一个坚实的基础，虽然我们触及了编程的一些基本元素，但我觉得应该在那个领域有更多的关注，我们的训练营大约有 5 个月，这对一个训练营来说是相当长的，我觉得我们可能在特定语言的概念上花了太多时间，而不是更广泛地应用于编程的概念。
5.  **CSS 框架** :
    在我参加的训练营中，我们用普通的 CSS 做所有的事情。学习 CSS 基础知识是很好的，但是大多数公司和团队都使用这样或那样的 CSS 框架。物化，顺风，引导，随你挑。幸运的是，我在训练营开始前有一些引导，我认为这很有帮助。这不仅可以提高你的工作效率，也是将 CSS 魔力带给所有浏览器的好方法。

**我的外卖**

我认为重要的是要强调我学到了多少东西，以及总的来说这段经历有多棒。然而，正如生活中的所有事情一样，总有改进的余地。在这些集训营和你在一起的时间很短的情况下，他们肯定会尽力而为，这些只是我注意到的在工作中非常重要的几个项目，它们要么根本没有被教授，要么被掩盖了。
