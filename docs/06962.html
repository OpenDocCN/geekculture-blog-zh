使用API来支持应用程序是当今的常见做法——这是应用程序开发的未来。API提供了一个很好的模型，使得数据可以被多屏幕世界访问。如果远程服务API支持您的应用程序，第一个应用程序可能是运行在苹果iOS操作系统上的移动应用程序。

至关重要的是，如果商业模式被证明是错误的，您可以快速响应不断变化的需求，并迭代地为Android手持设备构建另一个应用程序。或者创建一个内置的汽车应用程序或任何其他控制台应用程序(甚至不是控制台)。只要您的应用程序可以通过调用远程API来发送和接收数据，您就可以在自己选择的平台上创建自己选择的用户界面。您可以编写和托管您的API。Ruby、Go或Java等流行语言中的许多框架都支持使用REST等标准架构风格创建API。

为什么是GitHub API？

GitHub API是必不可少的，因为如果您正在构建软件，您可能会使用GitHub来管理您的软件代码。没有GitHub的人也可以使用Git。理解GitHub API也很有帮助，因为它将Git功能覆盖到了网络编程接口中。GitHub API可能是设计最好的API。

它是超媒体API，可以是API客户端适应API变化的成功尝试；这是一个棘手的问题。API版本控制合理。它很全面，并且与大多数Git特性密切相关。它始终如一，组织有序。
