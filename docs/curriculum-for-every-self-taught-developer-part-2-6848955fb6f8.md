# 第五个月:系统设计

*   [系统专家](https://www.algoexpert.io/systems/product) (4 周)
*   [系统设计入门](https://github.com/donnemartin/system-design-primer)(参考文献)

在第五个月，你应该精通编程，并且有能力自己开发一个成熟的 web 应用程序。

然而，这给我们留下了另一个挑战— **扩大规模**。

这就是系统设计发挥作用的地方，你无疑会在每次面试中受到考验。

我们的目标是能够清晰地表达一般的系统设计术语**，而不会太深入地陷入技术细节**

SystemExpert 出色地为您配备了工具箱中的基本工具包。它提供了大量的系统设计示例，我强烈推荐它。

# 第六个月:计算机网络、操作系统、数据库管理

![](img/cf418118be49abff8eb2bb2cf688a1ce.png)

Photo by [Igor Starkov](https://www.pexels.com/@igor-starkov-233202) on [Pexels](https://www.pexels.com/photo/photo-of-people-on-building-under-construction-1117452/)

*   [网络工程——Hussein nae sar](https://www.youtube.com/watch?v=7IS7gigunyI&list=PLQnljOFTspQUBSgBXilKhRMJ1ACqr7pTr)(1 周)
*   [操作系统— Neso 学院](https://www.youtube.com/watch?v=vBURTt97EkA&list=PLBlnK6fEyqRiVhbXDGLXDk_OQAeuVcp2O) (2 周)
*   [数据库管理系统— Neso 学院](https://www.youtube.com/watch?v=6Iu45VZGQDk&list=PLBlnK6fEyqRi_CUQ-FXxgzKQ1dwr_ZJWZ) (1 周)

来自非 CS 背景，你的**冒名顶替综合症**有时会在有人问你问题时发作。

这些归结为一个简单的原因——没有一个坚实的基础。

因此，您将通过涵盖以下主题来构建您的基础知识

*   **联网** — OSI 模型、IP 地址、端口号、DNS、HTTP、TCP、UDP
*   **OS** —调度算法、程序、进程、线程、死锁
*   **数据库管理系统** — SQL，NoSQL，缓存，ER 图，数据库设计

目标是充分理解它们中的每一个，以便从整体上了解事物是如何工作的。

# 第 7 个月:面向对象，设计模式，坚实的原则

*   面向对象编程(1 周)
*   [设计模式](https://www.youtube.com/watch?v=v9ejT8FO-7I&list=PLrhzvIcii6GNjpARdnO4ueTUAVR9eMBpc) (1 周)
*   [坚实的原则](/backticks-tildes/the-s-o-l-i-d-principles-in-pictures-b34ce2f1e898) (2 天)

编程不仅仅是将需求转化成代码，更是一门艺术。

编写**可读的、可重用的**和**高度可伸缩的代码**是每个程序员的梦想，这是你和我将花费一生来掌握的技能。

以上三个话题，是你迈向千里之旅的第一步。

尽管它们通常不会在面试中被测试，但作为初级开发人员，它们会让你受益匪浅。

# 第 8 个月:Leetcode

![](img/f0f9131e2569b6d68ea6da95b2c49354.png)

Photo by [Tembela Bohle](https://www.pexels.com/@tbphotography) on [Pexels](https://www.pexels.com/photo/photo-of-man-running-during-daytime-2803160/)

*   [算法专家 160 个问题](https://www.algoexpert.io/questions) (2 周)
*   [LeetCode 百大点赞问题](https://leetcode.com/problemset/all/?listId=79h8rn6&page=1) (2 周)
*   [LeetCode 顶级面试问题](https://leetcode.com/problemset/all/?page=1&listId=wpwgkgt) (1 周)

祝贺你到达最后一英里！在最后一个月，你将会竭尽全力，最后一次成功。

一次标准的软件工程师面试不可避免地会伴随着**现场编码问题**，这也是大多数噩梦出现的地方。

> 就像其他技能一样，熟能生巧，对于编码来说尤其如此。

集中精力完成上面所有的问题，

*   大声思考和**解释你的代码**就像有人在面试你一样
*   以彻底的时间和空间复杂性分析结束每个问题

把这一点运用到每个问题中，你会很容易给每个面试你的人留下深刻印象。
