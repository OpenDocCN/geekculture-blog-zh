1.考试概述

雪花的 SnowPro核心认证考试指南对考试进行了概述，提供了与主要主题、考试后勤和推荐培训相关的信息。其中主题(和权重)如下:

  • 账户和证券(10-15%)
  • 虚拟仓库(15–20%)
  • 数据移动(11–20%)
  • 绩效管理(5-10%)
  • 雪花概述和架构(25–30%)
  • 存储和保护(10–15%)

2.目标应试者

《认证考试指南》建议结合实践经验和教师指导的培训来准备考试。我认为你应该有至少六个月(更像是一年)的实践经验。更重要的是，接触常见问题，例如与以下方面相关的考虑事项:

  • 访问控制—例如，创建访问特定对象所需的自定义角色层次结构和权限
  • 数据加载/卸载—例如，从云提供商安全加载/卸载数据的选项
  • 仓库设计—仓库的数量、大小、多集群？横向扩展还是纵向扩展？

甚至只是接触更多的日常事物，比如:

  • 雪花中物化视图的局限性
  • 命令行工具SnowSQL的使用
  • 使用雪花任务的目的和限制

4.我的建议是——创建一个试用账户，并开始构建示例！

我在上面提到，在参加考试之前，我有大约一年的使用雪花的经验。我认为这使我成为了一个很好的考试候选人，因为仍然有多个我以前很少接触的主题领域(在接下来的主题“我很少接触的主题”中提到)。

但是由于之前的经验，我在Snowflake中直接遇到了许多作者在实践论文中经常提出的问题。例如:

  • ‘用户可以将一个资源监控器分配给多个雪花仓库’—我有第一手经验知道这个不是真的
  • ‘一个雪花任务可以执行多个SQL命令’——我再次亲眼看到这个是不可能的
  • “如果对底层数据进行了更改，雪花中的物化视图不需要执行手动刷新” —这是事实，因为雪花管理物化视图
  • ‘存储集成对象用于生成凭证，凭证用于认证从云提供商到雪花的通信。此外，当存储集成对象改变时，任何依赖的雪花阶段都需要刷新'——我再次遇到了这个问题

我可以举出许多类似的例子。不过，不管怎样，我的建议是试着构建《进阶系列》和雪花文档中描述的代码的工作模型。

5.我以前很少接触的话题

尽管以前有过实践经验，也遇到过我在建议中强调的一些事情，但有几个话题我以前很少或根本没有接触过，也没有接触过。这些话题在考试中(和练习试卷中)出现过，所以我建议你熟悉这些:

  • 连续数据保护:例如时间旅行、故障安全、零拷贝克隆
  • 账户使用信息:特别是与使用雪花数据库模式相关的信息:ACCOUNT_USAGE & INFORMATION_SCHEMA

例如，在哪里可以找到客户使用了多少存储的信息？以及到哪里去寻找信息以获得关于哪些对象已经被使用的审计。

  • 安全原则:例如数据加密、联合认证和单点登录
  • 聚类键&聚类表
  • 微分区&数据聚类

例如理解默认情况下查询是如何被聚类的，并且替代地，使用查询简档来理解雪花查询的微分区结构。

  • 数据共享和数据库复制

以及围绕更多边缘情况数据加载对象的主题，特别是:

  • 雪花流(以及围绕变更跟踪实现的细节)
  • SnowPipe REST API
  • 雪花卡夫卡连接器
  • 以及SnowPipe参数和功能，例如 $PIPE_STATUS

无论如何，我希望这些信息对其他人有用。如果你有任何问题，请联系我！
