# 格林威治标准时间

GMT 代表格林威治标准时间，我们大多数人都听说过这个术语。解释一下格林威治是伦敦的一个镇，当时被认为是零经度。只是为了刷新事实，纬度和经度属于用来标识地球上任何位置的坐标系统。所以无论你去哪个地方，都有某种坐标帮助我们识别它。你可以通过这个网站找到你所在位置的坐标[https://www.latlong.net/](https://www.latlong.net/)，不要和任何人分享，因为他们会找到你。

由于格林威治被认为是零经度，任何远离它的地方都有一个加/减几个小时的时间，这取决于该地方最先看到太阳的时间。

我们中的许多人可能会好奇，既然地球几乎是圆的(除了那些地球是平的人)，为什么我们都认为零经度是从格林威治通过皇家天文台的。

首先，要弄清楚的是没有实际的线通过，它只是一条假想的线。经过我的研究，因为大英帝国在某个时候拥有统治地位，这给了他们做这个决定的超级大国，每个人都必须同意。我希望现在很清楚了，虽然它们也是其他理论，让我们相信这一点，因为它是广泛可用和一致同意的。

# 时区

另一个需要理解的关键逻辑是时区的需要。如前所述，有必要为所有这些商业、法律和其他问题制定一个标准。供你参考，整个地球被分成 24 个不同的时区。

![](img/96ba633be5a39674b3401322b224eac6.png)

Source: Wikipedia.

时区也不是任何实际的区域，而是两个连续经度之间的假想区域。理想情况下，一个时区与相邻时区相差一个小时。但是时间的差异甚至可能更大或更小，因为各国都被赋予了维持其地理边界一致性的杠杆作用。

![](img/dcf03860d37a6df8b7866a266c139a88.png)

An image specifically showcasing India. Source Wikipedia.

让我们以印度为例，从技术上讲它有两个区域，但从西到东作为一个标准区域。所以稍微调整一下，印度就把比本初子午线多+5:30 小时作为自己的时区。虽然实际上，它与本初子午线的距离是+5:00 或+6:00。这有助于整个国家归入同一区域。

所以时区是一个非常重要的概念，每个人都应该理解。试想一下，格林威治西部的任何东西的时区都比格林威治时间短，反之亦然。

另一个有趣的显而易见的事实是，不同国家的时区可能是相同的，因为它是两条被称为经度的垂直线之间的差异，这就是它可以一次覆盖许多国家的原因。

# 协调世界时

UTC 代表协调世界时，它是格林威治时间的继承者。UTC 于 1970 年采用，并于 1972 年实施，在此之前它被称为 GMT。

对 UTC 没有什么不同的理解，因为它与我们在 GMT 中描述的时区相同。所有类似的地面被认为是在协调世界时的情况下，本初子午线通过格林威治，被认为是零经度，并用于计算其他地点的时区。

格林威治以东的每个地区都被认为是正时区，反之亦然。这背后的实际原因是地球从西向东移动，地球的东部比其他地区更容易看到阳光。

# 世

Epoch 也称为 Unix 时间 Unix 时间戳。自 1970 年 1 月以来，时间不过是几秒钟而已。

各种编程语言都有一些函数可以给你 UNIX 时间戳。纪元通常是存储日期的一种简单方法。因为按照用户的要求，我们可以改变格式的最终时间与用户想要看到的。

当您使用各种微服务时，以不同格式存储时间可能会导致问题。例如，一个微服务以 UTC 格式存储时间，而另一个以 IST 格式存储时间，此时时区差异可能会导致问题。

每当你试图构建一个特定的项目时，就要确定一些关于你将遵循的时区的指导方针，否则从长远来看，这可能会引起一个大问题。
