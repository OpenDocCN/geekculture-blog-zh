当然可以！下面是一个PyTorch程序，演示了如何训练一个简单的卷积神经网络来对猫和狗的图像进行分类:

import torch
import torch.nn as nn
import torch.optim as optim
import torchvision
import torchvision.transforms as transforms

# Set device
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

# Load the training and test datasets
train_dataset = torchvision.datasets.ImageFolder(root='train', transform=transforms.ToTensor())
test_dataset = torchvision.datasets.ImageFolder(root='test', transform=transforms.ToTensor())

# Create data loaders
train_loader = torch.utils.data.DataLoader(train_dataset, batch_size=64, shuffle=True)
test_loader = torch.utils.data.DataLoader(test_dataset, batch_size=64, shuffle=False)

# Define the model
class ConvolutionalNeuralNetwork(nn.Module):
    def __init__(self):
        super(ConvolutionalNeuralNetwork, self).__init__()
        self.conv1 = nn.Conv2d(3, 6, 5)
        self.pool = nn.MaxPool2d(2, 2)
        self.conv2 = nn.Conv2d(6, 16, 5)
        self.fc1 = nn.Linear(16 * 53 * 53, 120)
        self.fc2 = nn.Linear(120, 84)
        self.fc3 = nn.Linear(84, 2)

    def forward(self, x):
        x = self.pool(torch.relu(self.conv1(x)))
        x = self.pool(torch.relu(self.conv2(x)))
        x = x.view(-1, 16 * 53 * 53)
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        x = self.fc3(x)
        return x

# Create an instance of the model and move it to the device
model = ConvolutionalNeuralNetwork().to(device)

# Define a loss function and optimizer
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(model.parameters(), lr=0.001, momentum=0.9)

# Train the model
for epoch in range(5):
    running_loss = 0.0
    for i, data in enumerate(train_loader, 0):
        # Get the inputs and labels
        inputs, labels = data[0].to(device), data[1].to(device)

        # Zero the parameter gradients
        optimizer.zero_grad()

        # Forward pass, backward pass, and optimize
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

为什么人工智能还不够(将来也不会)

第一，代码不行。但是即使它有效，编程也不仅仅是产生代码。

在编程中，我们需要:

  • 与客户交谈并收集需求
  • 设计解决方案架构
  • 准备数据
  • 测试代码
  • 朝着某个目标迭代，通常是多个目标
  • 优化质量、速度、内存等。
  • 决定走或放弃某些道路

我要说的是，如果没有其他人的合作，这些步骤通常是无法完成的，尤其是客户、提供或创建数据的人、测试人员、管理员和资助者。

为人类编程从根本上来说是一种以人为中心的活动，即使你的过程包括独自坐在地下室一年。

是的，我们可以针对特定的领域和使用案例实现部分自动化。但这些都是例外。

人工智能观念的转变

使用机器学习进行编程并不是一个新的话题。它在科学界已经存在很长时间了，自21世纪初以来一直在进行密集的研究。只要看看谷歌学术上列出的关于基于搜索的软件工程的文章就知道了，这是一个活跃的领域。它从一开始就使用人工智能技术，例如基因编程。

你编码的时候会在网上搜索解决方案吗？好吧，那你是在用人工智能，因为连谷歌搜索都用人工智能。谷歌只是PageRank算法的时代早已过去。搜索引擎的一部分是一个名为RankBrain的神经网络，它可以进行一些过滤。规范化是搜索的另一个ML组成部分。

更一般地说，人工智能的流行定义会随着时间而变化。

20年前的人工智能现在被认为是理所当然的，仅仅是背景。

开发这些东西的过程被非技术人员视为只是工程。再补充一个例子，在20世纪60年代甚至连专家系统，也就是将专家知识编码为“if-then-else”逻辑，也被认为是AI。这些时代已经过去很久了。

我想说的是，我们已经生活在AI时代了。不管我们喜欢与否，ChatGPT和Github Copilot只是对我们以前拥有的人工智能工具的改进。

谷歌搜索引擎已经做了很多为我们找到编程问题解决方案所需的人工智能工作。

可以说，通过列出网站，也就是文档、许可信息等等，谷歌已经做得更好了。

版权问题

免责声明:我不是律师，这些只是我的担忧。我并不建议在法律上支持或反对使用ChatGPT输出。如果你需要实际的法律建议，去别处看看。

我想进一步阐述我在ChatGPT例子中提到的版权问题。代码由语言模型生成，该语言模型由具有不兼容许可的输入(其他人的代码)训练而成。我怎么能这样假设呢？因为它是在网络上训练的，而且几个开源许可，例如GPL，与专有许可是不兼容的，不应该一起使用。这可能是个问题。

即使在网上找不到具体的ChatGPT输出代码，我们甚至被允许使用这样一个模型的结果吗？输出不是由一个人创造的，他显然有权受到他人的启发，而是由一台从别人的数据中训练出来的机器创造的，那么输入的作者对输出有一些权利吗？

此外，如果它是由OpenAI这样的公司创建的，假设他们自己没有产生任何训练数据，那么 it 对输出有任何权利吗？如果他们做了呢？此外，当涉及到版权时，输出的具体内容会在多大程度上产生影响？

这是一个热门话题，无论哪种方式都有争议。出售这个工具产生的任何大量代码我都不会感到舒服，但我不是律师，不相信我的话。

那么像ChatGPT这样的技术对程序员有用吗？

是的。

我想象实际使用ChatGPT这样的工具是为了学习。

你不能为一个问题创建一个有效的解决方案，或者调试你的代码，但是你可以感觉到如何使用一种特定的技术来解决这个问题，比如C++、Rust、Java或者其他人的Pytorch。如果问题足够小，代码甚至可以工作。

裁决

像这样的工具将许多人独立完成的可重复任务自动化。他们不能为一个真正的新问题产生合理的结果。

正如吉多·范·罗苏姆最近在回答莱克斯·弗里德曼关于编程工作是否会受到像Github Copilot 这样的工具威胁的问题时所说，

当你问的问题是“请提醒我我是如何做到这一点的”(…)时，这样的工具总是最好的，它就像一个伟大的助手，但决定你希望代码做什么的创造性工作完全是你的。

我真诚地希望我们能得到更多这样的工具来帮助我们更快地实现目标。但是我不认为我们会很快退休。
