在印度尼西亚从事这一领域大约 3 年后，我可以分享我对作为一名初学数据科学家或当你刚刚开始作为他们中的一员工作时所期待的几件事情的一些想法。

我根据我的个人观点和一些信息、反馈以及在其他公司(初创公司和企业)以相同“头衔”工作的同事的故事创建了这篇文章。

希望你能从这里找到有见地的信息，享受！

当你开始你的数据科学家之旅时，特别是在印度尼西亚，你会遇到一些情况。

# 1.他们雇用你只是因为趋势

听过这个行话吗？

> “数据是新的石油”
> 
> “工业 4.0”
> 
> “大数据转型”

毫无疑问，许多公司和组织开始寻找实现这些目标的方法。策略之一当然是“雇佣一名数据科学家”。

它可以是好的，也可以是坏的。作为一名数据人员，需求增加对我们来说是件好事，

不好的一面是有时他们只是雇佣你，仅此而已。对于数据科学团队将如何运作，如何支持决策，没有明确的计划，更糟糕的是，甚至没有一个团队。你将作为“数据看门人”在 IT 部门工作

# 2.角色和责任不明确

数据科学家，数据分析师，数据工程师，ML 工程师，AI 工程师，AI 研究员，研究科学家，ML 开发者，BI 分析师，等等！！！

以上是我经常在工作岗位上看到的头衔。每个公司可能对每个角色都有自己的定义，你有可能会被“一股脑儿”塞过去。有些角色可能已经在他们的工作描述中很好地解释了，而有些角色还不清楚。

当你已经被一家公司聘用，比如说作为一名数据科学家。你的角色和责任可能(很可能)会不断变化(和增长)。

假设你正在做一个项目来创建一个客户流失预测。结果是没有数据管道来处理用户动作事件。因此，您被要求从头开始创建它(您在这里的行为就像一个数据工程师)。

在另一个美好的日子里，他们可能会问你关于“哪种产品更适合每个客户概况”的分析报告，而你仍然在处理你的预测 API 中的某个生产 bug(你充当分析师和工程师)。

这可能会导致非常疲劳和紧张的工作环境，并对项目的产出产生负面影响。

# 3.没有可用的数据

是的，这种情况经常发生。没有要探索的数据，所以你将不得不设计收集什么数据。它可以只停留在设计模式上，但也可能要求你实现它。

如果存储成本是公司关心的问题，那么情况将变得棘手。他们可能会要求你只收集那些“能让项目运作起来”的数据，而我们仍然不知道我们会得到多少数据，分布情况如何，等等。

# 4.没有经理/主管

在我看来，有两种情况非常相似:

1.  可能没有专门的经理/主管/导师直接和你一起工作。所以你将直接向上级汇报。
2.  第二种情况是，有一个经理作为你的直接主管，但是他没有数据科学背景(或者很少)。

在这两种情况下，作为初级数据科学家，我们将面临沟通困境。

*   "我应该简单解释一下还是详细介绍一下？"
*   “这个倡议很复杂，我该怎么办？”
*   “不知道这种做法是不是最好的做法，应该去哪里求教？”

从积极方面来说，这可以成为学习如何与高层沟通的一种方式(对于数据科学家来说，这是一项很好的技能)。相反，如果那些高层非常忙，没有时间交流，我们可能会产生低质量的工作，甚至更糟的是我们什么也没有产生

# 5.数据科学家==魔术师

在 IT 员工相关领域有一些流行的笑话，尤其是在印度尼西亚。“你可以修理智能手机，设置打印机，解决无线问题，入侵 Instagram 帐户，以及任何事情，因为你是一名 IT 人员”

嗯，在数据相关领域，也是这样的！！例如:

*   让我们用人工智能建立一个智能考勤系统，你是一个数据科学家，所以这是你的第一个项目。
*   我们想建立一个人计数器系统利用我们的闭路电视使用人工智能，我认为你可以作为一个数据科学家在这方面的工作。
*   请为我们的用户建立一个类似 Instagram 的 feed，使用人工智能个性化
*   还有很多，你可以用自己的经历来填充:)

# 6.你开始想“我的深度学习技能什么时候会被用到？”

这是你在公司的第二年，在一个由 5 人组成的数据科学团队中担任数据科学家。各自从事不同的项目，支持不同的利益相关者。

在一个星期天的早上，你会思考过去两年你都做了些什么…

*   手动数据标记
*   为运输团队创建 excel 报表
*   构建 data studio/tableau 仪表板
*   再次创建 excel 报表以进行定价模拟
*   创建和维护计划查询
*   再次手动标记
*   再次修复查询，等等…

然后你意识到，“没有建模，没有训练，没有深度学习”。我是一名数据科学家，为什么我不生产任何模型？我的深度学习技能什么时候会用到？诸如此类。

当你已经处于这个阶段时，恭喜你现在已经有了成为一名优秀数据科学家的坚实基础。

为什么？因为一大步就是要知道数据科学并不总是使用深度学习，机器学习模型等。

主要目的是利用你拥有的数据给出价值。通过创造性地提出见解来解决问题，并从中做出决策。理解问题，理解难点，并使用数据科学方法为它们创建解决方案。

# 结束语

成为一名数据科学家并不是一件简单的任务，可以在为期 2-3 个月的训练营中完成。永远学习需要经验、耐心和成长心态。

此外，即使你已经掌握了足够的技能，也并不意味着它会在你目前所在的公司/组织中工作得很好。这是一个很大的领域，需要所有利益相关方的协作，才能使用数据科学方法实现伟大的成就。
