程序员只知道从 **StackOverflow** 复制粘贴；这是一个笑话，不是吗？这很有趣，就像说所有作家都是从字典里复制粘贴一样。这是真的，但同时这也不是*的全部*真相。在这篇文章中，我想澄清一下什么是*stack overflow，什么是*不是*，以及*如何以及何时*使用它来帮助您在编程项目中更快地前进。*

# *什么不是？*

*让我们先解决这个问题；StackOverflow 并不是您问题的最终、精确的解决方案。在一些罕见的情况下，您可以在 StackOverflow 中找到一段代码，您只需复制并粘贴它，它就会完成您需要的确切工作，但这是极不可能的，而且这不应该是您使用它的目标。*

*在 StackOverflow 中找到准确的解决方案就像在**字典**中的一个单词用法示例中找到你的小说的准确的开头句子一样。It *可以*发生，但可能性极小，也不是某个人找词定义的正常意图。*

# *那是什么呢？*

*StackOverflow 是一个**众包代码字典用法示例**。仅此而已。当你这样想的时候，你会突然明白为什么每个人都使用它。而且，*如何使用* 变得更加清晰。*

*要知道这意味着什么，让我们都记住字典有三个重要部分:**单词**、**定义**和**用法示例**，后者是 StackOverflow 的主要用途。所以，让我们在编程世界里把它们分开:*

## *单词和定义*

*这是你的 UI 元素、类、函数等等…以及对**它们是什么**和**它们做什么**的描述。这通常是您从代码文档中至少可以学到的。*

*这也是我个人认为作为程序员最应该知道的。如果你只能从你多年的编程经验中记住一件事，我会说应该是单词和事物的定义，而不是如何使用这些东西。*

*这里有一个例子:在 Android 开发中，用于高效显示一长串项目的组件被称为**recycle view**，它使用所谓的**适配器**来显示项目。在 iOS 中，你可以使用 **UITableView** 或 **UICollectionView** 以及那些使用**委托**和**数据源**来显示项目的应用。*

*寻找如何使用一个 **RecyclerView** 比寻找在 Android 中使用什么来有效地显示一长串项目要容易得多，这将我们带到 StackOverflow 的主要用法。*

## *用法示例*

*与人类语言词典不同，用法的编程示例依赖于许多无法在单页中总结的东西；这就是为什么代码文档经常不能覆盖所有用例的原因。*

*例如，一个**回收视图**可以与一个*单类型*或*多类型*的项目一起使用，其中一些项目可以是*可点击的*、*可滑动的*、*可展开的*、*可移除的*或*可插入的*，所有这些项目都可以有*图像*、*文本输入*最重要的是，这些项目可以从本地、*从后端服务器*获取，或者*将数据发送到后端服务器*，这只是冰山一角。当您开始考虑上述所有因素的复杂组合时，事情可能会变得棘手，这就是 StackOverflow 的用武之地。*

*StackOverflow 是一个很好的资源，它允许您在*特定上下文*中寻找*特定组件*的*特定用法*。简单的用法可以在代码文档中找到，但是一旦你的代码变得更加复杂，你就需要开始寻找其他人对与你的更接近的问题的解决方案。*

*说了这么多，让我们快速总结一下。以下是解决编码问题的步骤:*

1.  *直接进入文档。如果你的问题足够简单(假设你是编程新手)，那么很可能你会当场找到答案。*
2.  *如果你找不到答案，至少试着了解一下解决这个问题需要什么样的“词语”。*
3.  *试着通过自己修改这些词语来实现一个解决方案，并尽可能地接近这个解决方案。*
4.  *在 StackOverflow(或者谷歌，反正它会带你去 StackOverflow)中搜索你没有做的事情。*
5.  *你可能会发现许多问题和许多答案，这些问题和答案会包含你所需要的一些东西，或者至少会启发你找到正确的解决方案，而且很可能你只需要稍微修改一下你的解决方案。*
6.  *如果你找到了你要找的东西，确保**在你的代码库中以评论链接的形式给予**应有的肯定。*
7.  *如果你得到一个没有人谈论的解决方案的灵感**，通过写下它来额外努力回馈社区**。*
8.  *如果你还是失败了，就睡一觉，然后冲洗并重复。*

# *最后的想法*

*StackOverflow 已经成为任何软件项目不可或缺的一部分，不管你有多有经验。之所以写这篇文章，是因为我觉得知道*什么时候*和*如何*使用 StackOverflow 非常重要，尤其是如果你是一个新人的话！*

*记住上面的 8 个步骤，尤其是最后一步，你会对你的项目有更顺利的体验。在我 10 多年的职业生涯中，他们从未让我失望过，我希望他们也不会让你失望。*
