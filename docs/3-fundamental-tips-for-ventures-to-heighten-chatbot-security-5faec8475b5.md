# 何时提高聊天机器人的安全性

数据保护涉及静态和传输中的信息。

# 提高聊天机器人安全性的技巧

无论您是企业、开发者还是当前的机器人用户，在聊天机器人安全方面，都应该考虑以下这些技巧来保持领先。

# 1.跨越渠道并不总是无缝的

聊天机器人技术的一个关键优势是在收集执行任务所需的信息时能够跨频道。但是，随着所有这些交叉和私人信息的交换，信息被泄露的可能性大大增加。聊天机器人生态系统中的每个人都需要考虑可以访问的每个渠道。

虽然可以创建一个 bot 来通过私人通道保护信息，但在公共通道(如脸书)中共享的数据会受到该通道的安全敏感性的影响。私人渠道很好。公共频道不安全。

# 2.爱上加密

我们知道，机器人为用户、开发者和公司提供了更快的速度、更大的灵活性和更大的便利，这是有条件的。如果你没有适当的安全措施，比如机器人加密存储的用户信息，恶意攻击是可能的。

![](img/22023c3c457ace4fe2d1259fea4648c5.png)

Photo by [Markus Spiske](https://unsplash.com/@markusspiske?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

企业了解这一问题，并采用跨通道加密来保护静态和动态数据。你的电子邮件和短信是加密的，任何其他交换敏感信息的服务也是加密的。

当谈到可以访问个人信息的机器人时，对保护数据和通信感兴趣的开发人员将只授予机器人访问加密通道的权限。然而，对于公共机器人来说，这是一个不同的故事，你无法控制这些平台，他们加密什么，不加密什么。一不小心，就有可能是一场巨大的网络安全风暴。

那么你如何阻止这种情况呢？端到端加密。

这将阻止除发件人和收件人之外的任何人看到该信息。这方面的一个很好的例子是 WhatsApp，当你向另一个用户发送消息时，会使用他们的端到端加密，只有你和那个人可以阅读、查看或收听你发送的内容，甚至 WhatsApp 都无法访问它。

说到 WhatsApp，最近他们对隐私政策做了一些修改，然而，他们仍然声称你的信息是加密的，不会被他们的员工或第三方获取。
