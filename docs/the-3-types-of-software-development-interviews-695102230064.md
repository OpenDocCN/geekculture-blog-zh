最近，我发现自己在申请新的软件工程工作时，处于一个既兴奋又害怕的位置。

我说令人兴奋是因为一个人不禁对新的技术工作、新的城市、新的同事、新的薪水所带来的可能性充满希望和兴奋🤑。我们都听说过那些拥有惊人福利的公司，比如弹性工作时间、免费食物和办公室中央的桌上足球桌。但最重要的是(至少对我来说),我们都期待与现代技术和前沿的敏捷实践一起工作——因为这通常反映了一种有趣的、现代的工作文化。

这个过程中唯一的问题是可怕的技术面试**。任何参加过软件开发人员面试的人都知道，这是一次痛苦的经历。您需要记住框架的语法，回顾您的 HTML、CSS 和 SQL，回顾承诺与可观察到的不同之处，并且经常进行令人恐惧的现场编码面试。**

**作为一个刚刚经历过这一过程并在最近的历史中在十几个不同的地方采访过的人，我觉得我有足够的条件给你一些我所看到的信息。软件开发人员面试并不只有一种。根据我的经验，有三种截然不同的类型。**

## **编码面试(压力水平:高)**

**第一种是每个人都告诉你要准备的，编码白板面试。这是你在线或亲自完成一个或一组问题的地方。在线版本通常是第一阶段的测试，目的是确保你具备进入下一阶段面试的一般能力和经验。**

**这种类型的面试很常见，但幸运的是，它正慢慢变得不那么规范。人们普遍认为，这种类型的面试更多的是选择没有表现焦虑的人，而不一定是选择最有能力的开发人员或最适合组织文化的人。因此，它开始被视为不是为一个职位选择个人的最有效的方式。(顺便说一句，你不喜欢这种类型的面试已经开始变得有些普遍了。有时，你的招聘人员实际上可以为你牵线搭桥，给你找一个其他类型的人**

**不幸的是，除了尽可能多的学习并希望你被问到一个你学过的问题之外，你没什么可以为这些面试做准备的。根据公司的声望和职位提供的薪水，面试会变得非常技术性和困难。然而，反之亦然。**

> **我讨厌不涉及我们作为 web 开发人员遇到的日常代码类型的编码面试，如果我被要求编写一个多年未见的复杂算法，我通常会对这个角色失去兴趣。**

## **提问式面试(压力水平:中等)**

**我经常看到的第二种类型的面试是口头提问式面试，有时这是你会得到的唯一一种面试类型，根据我的经验，这通常是你在申请咨询职位或使用 Angular 的职位时得到的面试类型(我不知道为什么，但我已经为 Angular 和 Spring Boot 类型的职位进行过几次这种类型的面试)。**

**这种类型的面试比第一种压力要小，因为通常不会出现你完全被难住或者回答不出大多数问题的情况(假设你已经研究过相关的框架或技术)**

**在我看来，这次面试是最容易准备的，你只需要研究公司将要使用的框架。例如，如果是一个使用 Angular 的职位，你应该花时间复习 Angular 上的速成课程并做笔记，不要觉得有义务编写项目代码，而是尽可能多地尝试和记忆框架，加快视频速度，因为你不会跟着编写代码。你可能会被问到的更难的问题是，你如何设置路线守卫？或者你如何利用延迟加载？**

## **文化契合度面试(压力水平:低)**

**如果你足够幸运，你能遇到的第三种也是最后一种面试也是最好的一种。这是一次面试，除了与工程经理闲聊和谈论你过去所做的工作之外，没有其他内容。是的，信不信由你，这有时是面试过程的全部——有时和几个经理见面交谈。这是迄今为止最容易遇到的面试类型，但也可能是最难伪装或扯淡的，因为你必须能够谈论你已经完成的项目，很快就会很清楚你不知道什么。这次面试的目的是了解你这个人，他们希望看到你充满激情、诚实，并且有学习和成长的动力。**

**幸运的是，这种类型的面试正在兴起！你当然应该了解自己的工作，并有几年的工作经验，但如果你知道，而且你有平易近人的性格，你可以放松下来，坦诚相待。告诉他们你做了什么，你有什么兴趣，你可能在做什么兼职项目。谈论你的 GitHub，谈论你最喜欢的技术组合(确保与公司的技术一致),并尽可能多地提问！如果你很安静，不善于交谈，你会开始被问到问题，而且不能保证这些问题会很容易。为了给人留下最好的印象，最好把讨论引向你觉得舒服的话题。**

> **我工作的前两个地方都有这样的面试，它们是我工作过的最好的公司。文化契合是为一个角色寻找优秀候选人的最重要的部分。**
