如何后端→

让我们讨论一下如何创建前端，什么是REST API，以及为什么它更有意义。现在这只是我的思考过程，我不敢说它是完美的，但它对我很有效。

任何开发的第一步都是理解问题陈述。这个事实是最关键的任务。很好地理解需求可以让你的生活更容易，或者当你试图弄清楚你到底想要什么样的行为时，它可以让你对着键盘哭泣(我从许多糟糕的经历中学到了这一点，所以从我的错误中吸取教训吧，人们..绝对清楚需要做什么)。问一些让你的游戏计划更清晰的问题。不仅仅是对客户，也是对你自己..就像客户告诉你他们的应用程序需要从用户那里收集数据一样，你应该问问自己，你所知道的安全、高效地收集数据的最佳方式是什么，并尝试获得更多关于正在收集哪些数据、访问这些数据的频率或目标数据量的信息。现在，这可能看起来非常简单和“常识”的事情，但在现实中，它可以改变你使用的数据库或你处理数据的方式。

现在你已经清楚了你的问题陈述，让我们来谈谈休息

休息→

很久很久以前，在一个叫做developer的社区中，他们用来创建web应用程序，也就是所谓的“单片”，因为他们过去常常在一个大文件中告诉应用程序要做什么..(想象一下调试这样的系统有多恐怖)。然后出现了一些聪明的人，他们认为我们为什么需要这样做，我们可以将它分成可以在不同端点提供的小服务，因此从灰烬中诞生了微服务的概念。

总之，REST APIs帮助开发人员创建一个更易于管理的代码库，您可以在其中为特定请求提供应用程序数据。让我详细说明一下..

让我们以一个基本的电子商务web应用程序为例。现在我们想让一个用户登录。现在，我们可以将表单数据(form = = log in form)发送到一个端点，在那里我们将函数(我喜欢称之为handler，因为它处理东西)映射到该端点(或链接),它提取表单数据并查询数据库以检查用户的存在和有效性。如果用户在场，它将告诉应用程序登录成功，可以继续了。如果失败，它可以向应用程序发送一条错误消息，甚至是一个带有消息的代码，可以用来告诉用户输入的信息是错误的。现在，由于这是通过将一个功能映射到一个端点来完成的，我们可以将这些与用户有关的功能分组到一个地方，这样就更容易管理了。

我稍后会展示它的样子。REST APIs有一个叫做动词的东西，它告诉我们预期的请求类型。Get和Post是最常见的，您可以认为get-requests的目的是从后端获取一些信息(如电子商务web应用程序的产品列表)。当您希望将数据发送到后端时，会发送一个post请求(例如，在前面的示例中，登录/登录将是一个post请求，因为我们希望将表单数据发送回后端，以检查用户的有效性，并且不会泄露用户在表单中输入的信息)。其他有趣的动词还有 put，delete，patch … 等。如果好奇的话，你可以阅读更多关于类型以及有哪些动词的信息，但get，post，delete，patch，put是我所经历的最常见的(大多数时候是按这个顺序)。

在这篇文章中，我会选择SQL作为数据库，因为mongoDB教程太常见了，有时关系数据库比非关系数据库更有效。现在，选择数据库是一个非常重要的决定，因为您需要最有效地存储和检索数据。我可以链接一个关于后端之战的很好的视频，作为何时使用哪个数据库的参考。你也可以同时使用这两种类型。(看看这个→【https://www.youtube.com/watch?v=rRoy6I4gKWU T2】)🤔也许将来我也会写另一篇关于数据库的博客。

现在我们对REST有了一些了解(你总是可以得到)，让我们来谈谈编码。现在有很多选择来编写后端代码，但是最流行也是我最喜欢的是javaScript和python。

开始开发流程→

我们将使用npm来安装用于创建后端服务器的软件包..快速JS模块。

为了方便和简洁，我们将使用VSCode，但是您可以选择您最喜欢的编辑器继续。因此，为项目创建一个文件夹，并在编辑器中打开它。要开始开发，运行npm init并回答询问的信息(或者只需按enter一串时间来保留所有默认值)。这将创建一系列文件，为节点项目做好准备。

npm install -g express --save安装模块。这是安装包的标准方式。我们将在整个博客中安装很多包，所以你会很需要它。一旦安装了包，并将其上传到git，(您需要将node_modules文件夹添加到gitignore，因为上传到git会很大)。当你在一台新机器上再次拉这个项目时，你可以运行npm install来安装所有的包。

现在我们已经准备好了一个项目，我们需要创建一个 server.js 文件来为我们的服务器编写逻辑。启动服务器时，我们将运行这个文件在指定的端口启动服务器(将在代码解释部分解释)
