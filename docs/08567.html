Java和Object Pascal是怎么开始的？

在20世纪70年代，瑞士的尼古拉斯·沃斯开发了一种用于教授结构化编程技术和概念的新语言，称为Pascal。

Pascal在大学里被广泛用于教授计算机科学。今天可用的商业对象Pascal版本始于1995年的Delphi集成IDE工具链。

Java，一种新的面向对象语言，是由詹姆斯·高斯林和太阳微系统公司于1996年开发的，专门用于他们独特品牌的工作站“JavaStation”

目标是提供一种独立于平台的“编写一次，在任何地方运行”的语言来产生可以在任何机器上工作的程序，提高软件在不同计算机之间的可移植性。

这两种语言在今天得到了非常积极的发展和广泛的使用。两者都可以追溯到计算机制造商、操作系统和架构激烈争夺主导权的时候。

对象Pascal和Delphi的起源

Object Pascal是苹果公司在1986年开发的Pascal的一个版本的演变。安德斯·海尔斯伯格开发了Object Pascal的扩展，作为Turbo Pascal的增强。

1994年和1995年，Borland在这些早期面向对象的Pascal原则的基础上，进一步增强和扩展它们，产生了Delphi语言。Delphi版本于1995年2月14日首次亮相。

在整篇文章中，我们交替使用术语Object Pascal和Delphi来有效地表示同一事物。

Java的推出和起源

詹姆斯·高斯林和太阳微系统公司在1996年创造了Java作为一种高级编程语言。Java的C/C++语法编写简单，但功能强大，足以为现代应用程序提供功能。

Java起源背后的原则是程序可以在软件开发者选择的计算机上编写，或者可能是他们唯一可用的计算机。

然后，专家们将在那台机器上编写的程序编译成一种中间形式，这种形式可以在任何装有“Java运行时”的设备上运行。

当用户运行Java程序时，目标计算机上的运行时将读取这个中间字节码形式的程序，并将其转换成本地代码，然后实际运行。

这个两步过程意味着Java程序员可以避开操作系统和硬件的差异，否则这些差异会使代码不兼容。

程序员可以针对任何平台，只要Java运行时可用于期望的目标计算机的配置。

这种广泛平台的一个结果是Java是当今最流行的编程语言之一。

Java很容易学，但是真正的挑战来自于库(Java API)和依赖链，它们可能很复杂。

当移植到较新版本的Java时，API也带来了挑战。这也是为什么很多用Java写的企业应用还在用Java版本8，而最新版本是Java版本17的原因之一。

Object Pascal和Java都是面向对象的

尽管Object Pascal不是严格的面向对象，但它具有所有必要的面向对象编程特性，如继承、封装和多态。Java是完全面向对象的。

Java试图解决C++中的问题，并取得了很大的成功。使用Object Pascal，您可以从面向对象和过程化编程这两种编程范式中获得最佳效果。

面向对象编程使用对象和类而不是函数来表示现实世界或抽象元素。使用对象可以使代码更容易重用和维护。

一个子类可以继承另一个祖先类的数据和方法，因此您可以避免重复相同的代码来实现子类共有的共享功能。

Object Pascal和Java都非常适合企业团队

Pascal和Java非常适合在多台机器上开发大型程序的大型开发团队。

Java可以在任何有Java运行时的操作系统或硬件上运行，无需重新编译。它非常适合需要共享多种不同机器和操作系统类型的团队。

Pascal使用结构化循环和简单的语法来减少编码错误，处理大型程序时不会变慢。

两者都将程序代码块分解成不同的物理代码文件，比如Pascal的单元或名称空间。

只有那些工作在完全相同的程序区域的人需要完全同步和解决他们不同的变更。较小的文件将更容易在大型团队中保持同步。

Object Pascal和Java有什么不同？

Java和Pascal在编译代码、执行任务和内存管理(即垃圾收集)方面有很大的不同。

以下是Java和Object Pascal之间的一些显著差异。

目标Pascal的快速编译程序

对象Pascal编译器——尤其是Delphi——通常非常快。Object Pascal直接将代码编译成机器语言，而Java使用中间标记化的字节码，需要像Dalvik Virtual Machine (DVM)或JVM这样的解释器来执行。

与Java不同，Object Pascal在执行时不需要转换，也不需要运行时的存在，因为程序通常是独立的，很少或没有额外的依赖性。这要求它成为商业应用和游戏等复杂软件应用的理想选择。

与对象Pascal相比，Java是冗长的

Java严重依赖于具有冗长语法的类、接口和泛型，这使其成为重复的“样板”或脚手架代码来描述和实例化它所使用的对象。

Object Pascal易于阅读、书写和学习。

它设法避免属性的重复代码，但是总体上对这种语法修饰的需求保持在最低限度。

Object Pascal在Delphi中有一个专用环境，而在Java中有一个ide环境

Java存在于不同的环境中，有多种ide，如Netbeans、IntelliJ IDEA和Eclipse，其中许多都被设计成通用代码编辑环境。配置或重新配置工作工具链可能非常耗时。

另一方面，Object Pascal占主导地位的集成工具链环境是RAD Studio Delphi，它是专门为该语言设计的，包含大量的功能，直接面向那些编写Object Pascal代码的开发人员。集成的工具链可以显著提高生产率。

开发人员还可以在常规文本和代码编辑器(如Visual Studio代码)中编辑对象Pascal代码。

Delphi有一个可视化的设计器，提高了生产率，增强了软件团队的协作，并使用可视化的检查代码简化了项目管理。

虽然有些Java IDEs有可视化设计器，但Delphi one更全面，它专注于为Delphi程序设计和创建屏幕的细节。
