# 编码问题——如何解决？

> 原文：<https://medium.com/geekculture/coding-problems-how-to-approach-them-119357daf7ca?source=collection_archive---------16----------------------->

![](img/819818626e33991924b598969bc1019e.png)

我目前正在寻找我的第一份编码工作，作为旅程的一部分，我遇到了解决编码问题。不幸的是，在训练营期间，我没有意识到我应该为此做好准备。后来我了解到，编码问题是技术面试的一个重要部分，我需要为此做好准备。

我在回顾编码挑战时遇到的第一个资源是 [LeetCode](https://leetcode.com/problemset/all/) 。这是一个很好的资源，它给了你一组按类别、难度等划分的问题。它还为您提供了一个环境来编码您的解决方案，测试它们，您可以用不同的编码语言来完成。所以我开始复习那些被标记为容易的。LeetCode 还提供了提示、讨论，甚至解释了一些问题的解决方案，这是一件很棒的事情。因为当我开始时，甚至很难理解如何接近他们。

我遇到了 [PEDAC 方法](https://launchschool.com/blog/the-two-layer-problem#:~:text=the%20PEDAC%20process.-,PEDAC,is%20about%20coding%20or%20syntax.)，它对我理解如何解决编码挑战有很大的帮助。你可以使用上面的链接阅读 launch school 的详细解释，但我想分享一些我从学习这个方法中获得的收获。

PEDAC 代表:

1.  **P** 问题
2.  例例
3.  **数据结构**
4.  一种算法
5.  **C** 颂歌

一旦你这样分解，每个问题看起来都比你第一次看的时候简单。

# 问题

老实说，当我阅读一些编码问题时，我花了一些时间来理解需要什么。首先，我建议试着用你自己的话重新表述一个问题。确保您清楚地了解两个要素:

1.  要求
2.  输入和输出(BTW LeetCode 以一种清晰的方式提供了这些信息)

# 例子

当你理解了你的问题，一旦你有了你的算法或代码，有一些场景来测试你的结果是很棒的。再一次，LeetCode 很棒！在 LeetCode 中，已经为您准备好了样本，如果您愿意，还可以添加更多样本。但是当你在技术面试中解决问题时，你应该能够问面试官:“如果我的输入如下，结果会是什么？”所以你提出输入，然后从面试官那里得到输出。

# 数据结构

根据手头的问题，您应该能够确定可以使用哪种数据结构来解决问题。存储将输入转换为输出的数据的更有效的方法是什么？到目前为止，我见过的最常见的数据结构类型是[数组](https://en.wikipedia.org/wiki/Array_data_structure)和[链表](https://en.wikipedia.org/wiki/Linked_list)。

# 算法

这是方法中最重要的部分，你应该写伪代码。您写下将输入数据转换为最终输出的解决方案的每一步。记住，在面试中，面试官想知道你是如何思考和处理问题的。所以这是展示你思路的绝佳时机。当算法准备好了，你应该使用你的例子来确保你得到预期的结果。

# 密码

一旦你有了伪代码，并且每一步都被记录下来，你就应该把它转换成代码。你可以使用任何你喜欢的编码语言，如果你不确定的话，你可以谷歌一下语法。

但是只要你的算法是正确的，你总是能找出如何编码它。

现在，您的示例应该也能与您的编码解决方案一起工作。

这只是解决编码问题的方法之一的概述。最近，我与不同的人进行了几次交谈，并听到了关于一本书的很好的东西——《[破解编码面试](http://www.crackingthecodinginterview.com/)》。我刚拿到这本书，将开始阅读。以后，我会尝试对这本书和其他解决编码问题的方法给出一些见解。
