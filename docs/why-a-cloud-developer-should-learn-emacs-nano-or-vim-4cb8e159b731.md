# 为什么云开发人员应该学习 Emacs、Nano 或 Vim

> 原文：<https://medium.com/geekculture/why-a-cloud-developer-should-learn-emacs-nano-or-vim-4cb8e159b731?source=collection_archive---------46----------------------->

## 在这个花里胡哨的 ide 都可以免费获得的时代，为什么老编辑们仍然令人敬畏。

现在有许多很酷的 ide 可供开发人员使用。命令行编辑器落后了。然而，开发人员应该学习其中的一种。在某些情况下，当您需要远程登录到没有运行 X-server 的机器或服务器时，这些编辑器会派上用场。

这些编辑器是您日常开发活动的绝佳选择还有许多其他原因。

## 免费并安装在每个系统上

几乎大多数 Linux 发行版上都安装了至少一个这样的编辑器。我不记得曾经登录一个系统却不能使用 Vi 编辑器。Nano 或 Emacs，在 Linux 上默认情况下可能可用，也可能不可用。但是它们很容易安装。

## 不需要 X-Server

这些编辑器在 GUI 和命令行上都可用。你不需要运行 X-Server 来编辑代码。

很多时候，云开发人员需要登录到服务器来检查日志和调试。您不会找到通常用于开发的 ide。这些编辑器派上了用场。

## 不需要鼠标/触控板

Vi、Nano 和 Emacs 都有自己的快捷键，可以做任何事情，比如移动光标、查找和替换等等。有了这些快捷方式，代码可以写得更快，对我个人来说，这些是最方便的。

## 众多插件

这些编辑器有各种插件来适应你的编码习惯、语言、颜色主题等。这些都是高度可扩展的。

更现代的 ide 也有插件来支持 VI 风格的编辑。VI 编辑器的所有键盘快捷键都可以与 Visual Studio 等 ide 一起使用。
