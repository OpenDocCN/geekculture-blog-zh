# 这个想法

因为我创作我的收藏是为了乐趣，而不是为了金钱上的收益，所以我可以想出一些愚蠢的想法，只是从这个项目中获得乐趣。我的第一个收藏(失败了)只有一些鸡尾酒像素艺术图片。但是它让我对这个项目中的每一件事都想了很多。

我的第二个系列，[Pixel pizza](https://www.pixel-pizzas.com/)，从“每个人都喜欢披萨”这个想法开始。我仍然支持这一理念。我联系了一个和我很亲近的人来创作像素艺术，我们玩得很开心。我们有最愚蠢的想法,从 10000 件极度罕见的烤焦的比萨饼到谈论艺术和与艺术斗争。

玩了一会儿并讨论了系列的最终目标后，我不得不继续手头的下一个任务，网站，最终图像，智能合同，以及在 Medium 上的一点点营销。

# 图像和元数据

像素艺术由几个层组成，这些层共同创建了一个独特的图像。因此，当我们准备好所有单独的层时，我开始使用一个相当简单的 Python 程序来创建最终的图像，以及这些图像的元数据。

简而言之，元数据是存储在网上的一条信息。在我们的例子中，比萨饼有哪些配料，等等。我一起生成图像和元数据，因为这样做几乎不需要更多的努力，并且在以后需要元数据时节省时间。

接下来，元数据必须包含到图像本身的 IPFS 链接。为此，我使用 Pinata 的 API 将我的图像上传到 IPFS，一个分散的存储网络，并将链接放入元数据 JSON 文件中。

最后，元数据需要上传到 IPFS 的一个文件夹中。我去掉了元数据文件的文件扩展名，因此它们被称为:“0”、“1”等等。我再次上传它们，用 Pinata。这将给出我的 baseURI，即每个令牌或令牌 URI 的链接的第一部分。

tokenURI 只是将 tokenId 附加到 baseURI。因此“mysite.io/ipfs/”将变成“mysite.io/ipfs/123”，并链接到正确的数据。

# 智能合约和区块链

智能合约在我的收藏中相当简单。我使用 OpenZeppelin 主要是为了方便编程，我添加了适当的功能来为每个令牌获得少量补偿。值得庆幸的是，经过几堂智能合同的速成课之后，这并不是很难实现。

在对区块链科技和智能合约进行研究后，我得出结论:以太坊很贵。但是因为 OpenSea 也支持多边形，所以我用了多边形。我在以太坊学到的所有知识也适用于此，至少就智能合约而言。

由于我做这个项目不是为了获取冷酷的利润，我可以很容易地使用 Polygon，否定它比以太坊更不友好的事实。花了大约 50 美分，我就可以部署这个合同。

# 网站

互联网上的每个网站都使用 HTML、CSS 和 JavaScript。至少，大多数是这样。但是对于区块链应用程序，在大多数情况下，您还需要连接 Web3 或 MetaMask。由于 Web3 和 MetaMask 的出色工作，这在 JavaScript 中很容易实现。这都是客户端的。

所以我用我最喜欢的技术和库从头开始创建了一个外观简单的网站。我添加了到 Web3 的连接，并添加了一个连接你的钱包的按钮和一个铸造令牌的按钮。

我测试，测试，再测试一些，然后我准备好了。我将合同部署到 Polygon Mainnet，并部署了我的网站。

最后，我写了一篇关于 NFT 收藏的文章。
