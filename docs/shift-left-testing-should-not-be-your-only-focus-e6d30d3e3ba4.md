# 什么是左移测试？

如果你不熟悉狗屎左派的概念，我可以用四个字来总结:

> "尽早并经常测试."

现在'*早测试，勤测试'*背后有更多的深度和意义，所以请继续关注我。在传统的测试实践中，测试发生在软件开发的最后阶段是迫在眉睫的。所以引入左移意味着测试发生在过程的早期阶段。

打乱执行顺序的基本原理是完全有道理的。通过在早期阶段引入测试，我们应该能够更早地识别任何值得注意的问题，并采取行动。

当我们将测试提升到几个阶段时，我们会引入一些额外的问题。例如:

*   谁应该在这个阶段进行测试:QA 还是开发人员？
*   开发后我们还测试吗？
*   开发者是否应该先测试再 QA？

## 你应该把注意力放在哪里？

所介绍的问题并不复杂，但却切题。与其回答这些问题，我想提出的问题是:“你应该把你的注意力放在哪里？”

在传统的测试实践中，有一堵无形的墙将开发和测试分成两个独立的筒仓。仅仅这种分歧就带来了一系列不同的问题。如果你感兴趣的话，我在另一篇关于[软件测试如何发展的文章](https://qatoddy.medium.com/software-testing-is-evolving-heres-how-d435307060f9)中谈到了它们。

在早期阶段进行测试是有益的，但是我建议你将注意力放在: ***平衡低效测试实践的优化上。***

# 听起来需要更多的工作

左移听起来并不吸引人，这个名字给人一种工作被转移到左边的感觉。那些不熟悉这个概念的人可能已经对接受它感到有点冷淡了。

术语 Shift-Left 是由拉里·史密斯在 2001 年创造的，它包含了测试本身需要更早地发生，更早地识别软件问题。

## 你应该把注意力放在哪里？

如前所述，无形的墙仍然存在于传统的测试实践中，左移也不例外。

QA 工程师是测试专家。探索如何与开发人员分享知识，让他们成为自信的测试人员。 ***关注注入测试作为开发的核心部分*** 开始拆除无形的墙。
