云计算因其诸多优势和便利性而成为消费者和企业的热门选择。然而，每个系统都有其优缺点，云计算也不例外。

这里有一些云计算的缺点，可能会导致一些问题，以及一些简单的最佳实践，以帮助减轻这些潜在的问题。

# 安全性和隐私

大多数云提供商设计他们的服务，以确保他们遵守最佳的安全标准和行业认证。然而，尽管如此，当我们在第三方硬件上存储数据时，总是存在风险。云服务提供商有望管理和保护您存储数据的底层硬件基础设施，但您需要权衡可能的违规风险。

## 保护

始终确保正确实施由您负责管理的安全功能。这包括用户访问管理、加密、多因素身份认证等。

# 浪费的成本

虽然云计算通常会带来成本节约的好处，但如果使用不当，它实际上会让您付出更多的成本。过度供应或忘记终止您的资源，如实例或数据库，可能会导致您在月底招致惊人的巨额账单。我个人以前就经历过这种情况，因为忘记关闭亚马逊网络服务中的一个 RDS 实例而招致 500 多美元的账单。

## 保护

请确保不要过度调配您的资源，并利用自动扩展来自动化该流程。确保您的自动扩展策略在不需要资源时也可以选择缩减。创建警报，帮助您识别未终止的资源，并了解如何利用您的云提供商的成本分析工具，如 AWS 预算、Azure 成本管理+计费等。

# 停工期

当利用属于云提供商的基础设施时，总是存在服务中断的风险。2017 年，AWS 发生了一次停电，给上市公司造成了 1.5 亿美元的损失。2019 年 6 月和 7 月，Cloudflare、亚马逊、Shopify、谷歌和 Reddit 等许多公司都受到了宕机的影响。停机和减速对企业极其不利，尤其是那些拥有平台、软件或技术作为其产品的企业，即 SaaS 公司。

## 保护

设计具有高可用性的服务，利用多 AZ、多区域甚至可能是多云计算部署。这有助于减少中断导致整个应用程序瘫痪的可能性。然而，可用性越高，成本也越高，所以要明智地选择。

# 顽固

虽然云计算增加了大多数领域的灵活性，但也降低了某些领域的灵活性。客户对云中服务的功能和执行的控制通常很少。此外，一些工作负载可能不允许在云中运行。这取决于云提供商的 EULA 和管理政策。

## 保护

选择云提供商时，请确保检查他们的 EULAs 和管理政策。请务必考虑您的云提供商可以为您提供的支持量。够不够？最后，在选择服务时，确保检查您可以设置哪些配置，这样您就不会受到服务的限制。
