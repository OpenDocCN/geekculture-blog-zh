## WordPress 值得学习吗？

## 学习如何正确使用 WordPress

所以，最近我收到了很多人的问题，关于他们是否应该把 WordPress 作为他们唯一的技术。正确回答这个问题真的很难。主要原因是，有些人仅仅依靠 WordPress 谋生。其他开发者说真正的开发者不使用 WordPress，不管那是什么意思。今天，我从自己的经历中来回答这个问题。从那时起，你将能够做出一个明智的决定，是将它作为一个工具放在你的腰带下，还是完全忘记它。

首先，WordPress 开发者有两种。一个使用 PHP 核心技术在 WordPress 中开发主题、插件和任何自定义内容的人。另一种利用这些主题和插件为他们的客户设计一个看起来不错的网站。两者都没有错，拖油瓶和滴管都有市场。

当我得到这些问题的时候，很多事情在我脑海中闪过。这些人大多是初学者。我不能指望他们现在就有很多工具。问题是我不能说是，我甚至不会建议任何人把 WordPress 作为他们唯一的技术。这不是因为它没有市场，而是因为你在处理不同的项目时不会灵活。我一直问自己，为什么人们会去追求他们讨厌的技术？我的意思是，抱怨它的开发人员在某些时候仍然会使用它。

无论如何…

起初，WordPress 是为博客作者而建的，他们中的大多数都是非技术人员。与使用其他技术相比，建立一个简单的 WordPress 网站过去和现在都更加容易和便宜。那么，主要问题是什么？为什么有很多开发者讨厌 WordPress？老实说，我认为讨厌 WordPress 的人不知道 WordPress 有多强大。

对 WordPress 的一些主要抱怨是速度，可移植性，刚性，重量，频繁更新，糟糕的 SEO，糟糕的安全性，仅举几例。看看这些投诉，我认为 WordPress 没有任何问题。坚持住，让我解释。我能想到一些方法来避免每一个抱怨，我会解决它们，使用插件不是解决方案的一部分。

我同意 WordPress 没有很多现成的功能，你需要很多插件来增加额外的功能。等等，我不认为你知道。看，既然开发者对此抱怨最多，我看不出他们为什么不能把他们需要的特性编码到 WordPress 本身中。没有法律禁止定制 WordPress。我只是觉得抱怨它死板的开发者简直就是懒。大多数非技术人员可以在他们的网站上添加一些插件，如果他们是为自己做的话。

我在 WordPress 中拥有大量插件的主要问题是可维护性、安全性、大小、SEO、可移植性和速度。你意识到什么了吗？WordPress 作为一个 CMS 本身，没有任何外部添加，是非常可维护的，安全的，轻便的，快速的。往里面添加东西会引发人们抱怨的所有问题。

举个例子。那里的大部分人都是免费工作的，因为这被认为是一场竞争，以获得一些经验。由于报酬不高，大多数人最终选择了简单的路线，选择空插件和许多其他不再受支持的免费插件，只是为了确保他们能为那个吸血的客户拼凑出一些东西。

现在，我并不是说每个用 WordPress 建立的网站都是不好的，只是有时候开发者滥用这个强大的 CMS 搬起石头砸了自己的脚。使用 WordPress 作为拖放工具进行开发是大多数自由职业者的谋生方式，这并没有错，因为客户无论如何都会得到他们所支付的东西。另一方面，它在某种程度上破坏了你的声誉。如果客户不断向你提出关于你建立的网站的问题，你的批准率就会下降，客户把你推荐给他们朋友的机会也会下降。

我可以继续讲述 WordPress 让你陷入困境的不同方式，但是我想我已经花了一段时间讲述背景信息了。让我来介绍一下你使用 WordPress 的不同错误方式，以及如何避免它们并获得更好的体验。

# 1.刚性和便携性

WordPress 很死板。这是大多数互联网得出的结论。但问题是，视角很重要。如果一个非技术人员试图使用 WordPress 并抱怨它太死板，这是可以接受的。然而，我不能在道德上原谅这样说的开发商。WordPress 首先是一个 CMS。仅此一点就能让它如你所愿地灵活。你可以在它上面建造很多东西。

你认为你不能得到很多现成的功能吗？把它们编码进去。除非你只是想走捷径，否则使用 WordPress 需要对 PHP 和 MySQL 有很好的理解。如果你没有上述技术的经验，想使用一个定制的前端，使用 WordPress 作为一个无头 CMS。您将能够多次更改您的前端，因为您只是对您的网站的前端进行更改。

# 2.持续更新

同样，如果你是一个抱怨这一点的开发者，简单地做没有插件和需要不断更新的主题。编写自己的代码。这就是为什么你是一个开发者。把一个主题或插件拆开，把定制的功能放入 WordPress 本身的核心功能中。这样，你就可以在插件区拥有你想要的一切。这将需要更长的时间，但这是值得的。问题解决了。

# 3.速度慢

这甚至不应该是抱怨。我见过很多加载时间少于 3 秒的网站。这种抱怨主要来自于那些不愿意自己多做点事来打理网站的人。基本的日常任务，如缩小图像尺寸和单独使用合适的图像格式，可以让你速度很快。其他提高速度的任务包括缩小 HTML、CSS 和 JS，从 wp-includes 目录中移除阻塞渲染的资源，移除未使用的脚本等等。这些是人们不太关注的事情，然而他们却抱怨这些事情。

你不能指望只是快速拼凑一些东西，并准备好去没有确保良好的网站维护实践的每一个方面都涵盖在内。此外，如果这对您很重要，并且您熟悉其他技术，您可以将 CMS 与前端分离。您可以在前端使用任何您认为快速的技术，并使用 CMS 附带的 Restful API 甚至 GraphQL 从 CMS 获取数据。您甚至不必在同一台服务器上托管它们。

# 4.可怜的搜索引擎优化

在开发中，SEO 是你通常与优化一起做的最后一件事。现在，如果你是一个没有 SEO 经验的开发者，我建议你让其他人来处理这方面。我知道有些插件可以保证 SEO 的高分，但是由于这不是一篇快速解决问题的文章，我有一个更好的方法。

在测试网站性能时，我敢打赌你会使用 Google lighthouse、Gtmetrix、Pingdom tools 等工具。你只需要一个灯塔。当你测试你的网站时，google lighthouse 会给你的网站评分，并向你展示不同的问题。除此之外，它还指导你如何解决上述问题。你可能无法解决所有的问题，但我相信如果你处理了大部分，你的分数会在绿色区域。

# 5.安全性差

说到安全，我知道大多数人通常会通过添加某种安全插件来解决这个问题。我想让你知道，插件只是可以编码到 WordPress 本身的功能。这样，你就不会时不时地遇到插件失败的问题。你将完全控制你的网站。你可以更上一层楼。使用 WordPress 作为一个无头 CMS 使它更加安全，因为如果你的前端使用另一种技术，没有人会知道如何访问登录区。

# 6.重量

大多数用 WordPress 制作的网站通常很笨重，因为当你把主题和插件安装到你的网站时会带来额外的膨胀。解决方案是使用更少的插件或者根本不用，走自定义路线。我知道这并不容易，也不会很快，但是它会让你对 WordPress 有更好的体验。

# 结论

现在，关于你是否应该把 WordPress 作为你工具箱中唯一的技术的问题。不要！学习 WordPress。非常擅长。至少学习一个前端框架，并擅长它。了解如何正确利用基础知识——html 5、CSS3 和普通 JS。学习至少两个数据库——一个关系数据库和另一个非关系数据库。为什么我要你学这些？他们可以一起工作！这就是技术的美妙之处。你将有很多选择来处理不同的项目。

在你的项目中使用 WordPress 是一个非常个人化的决定。在选择要走的路线之前，有很多事情要考虑。这些因素包括预算、时间、性能等等。作为一个开发者，你应该给你的客户一些好的建议。在开始一个项目之前，和你的客户仔细检查每一个选项。告诉他们使用主题和插件会便宜又快速，但是需要大量的维护。让他们选择自定义主题。仔细考虑每个选项的利弊。告诉他们无头选项，看看他们对此有何反应。

主要因素永远是钱。如果客户愿意以你的价格与你见面，你必须给他们最好的质量。你必须在他们的交付物上见到他们。好的客户总是会做对他们的业务最有利的事情，你也应该为你的客户这样做。

我希望这篇文章能让你对 WordPress 有所了解。在一个优秀的开发者手中，它的确是一个强大的工具。不要给它遮阴。改变你使用它的方式。
