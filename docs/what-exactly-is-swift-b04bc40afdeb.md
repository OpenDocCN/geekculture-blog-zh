# SWIFT 到底是什么？

> 原文：<https://medium.com/geekculture/what-exactly-is-swift-b04bc40afdeb?source=collection_archive---------11----------------------->

![](img/29dc3b1dbd3c1f788cb7c255d2de594d.png)

你曾经把钱转到国外吗？如果是，您一定听说过 SWIFT 代码。SWIFT 不是一种支付服务。它只是银行使用的一种非常安全的信息服务。

SWIFT 的全称是**环球银行金融电信协会。**当我们从一个国家向另一个国家转账时，您的银行会询问收款人的银行详细信息和**该银行的 SWIFT 代码**。

银行如何使用这个代码把钱转到另一个银行账户？让我们借助一个例子来看看。

例如，A 在 A 银行有一个银行账户，他想向 B 银行的 B 人汇款 100 美元。当 A 人转账时，他们将提供 B 银行的 SWIFT 代码。然后，SWIFT 将向 B 银行发送一个安全的报文通信，包括所有必要的信息。

当银行 B 收到此消息时，支付转账将开始。让我们看看转移将如何进行。

当银行之间有直接连接时，SWIFT 就会工作，这意味着**A 银行将在 B 银行**拥有账户，而**B 银行将在 A 银行**拥有账户。

> 银行 A 将从 A 的账户中扣除 100 美元。
> 
> A 银行将把这 100 美元贷记到 B 银行的账户上。
> 
> B 银行将从 A 银行在他们银行的账户中扣除 100 美元。
> 
> 银行 B 将把 100 美元(减去他们的费用)记入银行 B 的账户。

因此，要使 SWIFT 发挥作用，银行之间应该有直接的联系？不一定。当银行 A 开始发送交易报文时，SWIFT 将确定成功汇款报文的路径。

假设如果 A 银行与 B 银行没有直接联系(意味着 A 银行没有 B 银行的账户，B 银行也是如此)，那么 SWIFT 将决定该交易的路径。

假设有一家中介银行**银行 C** 可以用来促成这笔转账。意味着 A 银行在他们银行已经把 C 银行入账了。C 银行已经在他们银行把 B 银行入账，B 银行在他们银行把 C 银行入账。

> 银行 A 将从 A 的账户中扣除 100 美元
> 
> A 银行将把 100 美元贷记到 C 银行在他们银行的账户上
> 
> C 银行将从 A 银行在他们银行的账户中扣除 100 美元
> 
> C 银行将把 100 美元(减去他们的手续费)记入 B 银行在他们银行的账户
> 
> B 银行将从 C 银行在他们银行的账户中扣除该金额
> 
> 银行 B 将把剩余金额(减去他们的费用)记入 B 人账户的贷方。

因此，如果你看到，如果在一笔交易中有许多中间银行，那么这将增加交易费用，因为每个银行都将收取他们的费用，交易将需要时间来完成。

这是关于 SWIFT 的高级信息，在这个过程中还有许多其他事情。但是你现在知道什么是迅捷了吧？
