通过媒体备份导出媒体文章并在你的Gatsby网站上重新发布的详细指南。

动机

在盖茨比的网站上有一个很好的插件生态系统，其中几个解决了将你的媒体帖子放到你的盖茨比网站上的功能。然而，几乎没有一个允许你从你的媒介账户中复制整个帖子列表。

我们将分析现有的解决方案，并且我们将引入一个新插件来有效地做到这一点。

现有方法

通常，你会从 Medium的开发者门户开始获取一些信息。这将引导您进入 GitHub 上的API文档。

最终，你会发现Medium支持 RSS提要。

从上面的页面中，你甚至可以找到一个Medium支持的 WebSub(原PubSubHubbub)API——http://medium.superfeedr.com/。

还有一个非官方的媒介API—https://medium.com/@eugenehauptmann?format=json&限制=100

这里有一个示例以 JSONP 格式输出。和另一个Python实现https://github.com/enginebai/PyMedium

另一种方法是使用类似于木偶师的东西来抓取Medium作者写的帖子。

最后一个，也是本文的重点— 从介质中导出您的所有数据。

总结一下，现有的方法如下:

  1. 官方媒体API—https://github.com/Medium/medium-api-docs
  2. 官方媒体RSS Feed—https://help . Medium . com/HC/en-us/articles/214874118-Using-RSS-feeds-of-profiles-publications-and-topics
  3. 官方媒体web sub API—http://medium.superfeedr.com/
  4. 非官方媒体JSONP API(https://github.com/enginebai/PyMedium)
  5. 手动废弃页面
  6. 下载您的媒体数据—https://help . Medium . com/HC/en-us/articles/115004745787—下载您的信息

现有插件的优缺点

不幸的是，上述大多数方法都有缺陷。原因如下:

  1. 官媒的API不让你拉Posts只能创建他们。
  2. RSS API只返回最近的帖子，并且只给你每个帖子的要点，而不是完整的副本。
  3. WebSub API也给你新添加的帖子的更新，而不是以前发布的。
  4. 废弃会与网络应用防火墙(WAF) 遇到同样的障碍。
  5. 非官方的JSONP API比官方的API要广泛得多，但它被放在了 Cloudflare WAF 之后，而废弃WAF之后的网站是一件痛苦的事情。
    Ping我们如果你需要帮助设置你的云基础设施和WAFs 。
  6. 下载你的介质备份解决了获取所有帖子副本的主要问题，但是这是半手工的。

2021年媒介API及其状态

Medium API团队在 2015 开了个好头，向所有人开放了他们的第三方应用程序流程。

不幸的是，他们在2019年年初停止允许的新应用，没有事先宣布。那些幸运地事先注册的人仍然可以继续使用他们的应用程序和官方媒体API，这些应用程序和API只对创建帖子有好处，而不是下载它们。

获得媒体帖子备份的解决方法

构建新插件以导入媒体备份

Gatsby是一个高速网站的现代网络框架。—https://github.com/gatsbyjs/gatsby

在过去的4年里，我们一直在我们的公司使用GatsbyJS，它最终成为我们培训初级工程师的默认框架。

Gatsby有很棒的文档和非常详细的教程，教你如何从头开始创建一个新的插件。你可以找到我们在 GitHub 上构建的插件的源代码。

大多数业务逻辑调用位于gatsby-node.js中，入口点在onPreInit中。Gatsby的架构允许你在构建期间通过各种插件运行所有必要的业务逻辑。

为了让这个插件工作，我们需要一些依赖关系:

  • 解压缩器 —一个流库，用于访问zip存档中的文件。
  • Turndown —一个将HTML转换成 Markdown 的工具。
  • Cheerio —一个类似HTML选择器和解析器的JQuery。
