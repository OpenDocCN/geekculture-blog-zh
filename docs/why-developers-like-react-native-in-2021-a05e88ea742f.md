# 为什么开发者喜欢 React Native 2022

> 原文：<https://medium.com/geekculture/why-developers-like-react-native-in-2021-a05e88ea742f?source=collection_archive---------3----------------------->

![](img/90a9cabc42bed393b108e9c3aad953b7.png)

在过去的三年里，我一直在开发 [React Native](https://reactnative.dev/) 应用，在这篇文章中，我将解释为什么开发者更喜欢 React Native 以及为什么它在 2021 年仍然相关。你应该对这篇文章有所保留，因为这是我根据自己的经验得出的观点。其他框架和替代方案也是可用的，但是它们超出了本文的范围。

2016 年我刚开始写 React 原生应用的时候和现在是完全不同的环境。一切都是新鲜的，事物处于不断变化的状态，有许多问题。然而，React Native 近年来取得了显著进步，它现在为创建跨平台项目提供了最佳的开发人员体验之一。

# 1.JavaScript 中的代码

事实上，您的整个项目都是用 JavaScript 编写的，这是 React Native 最吸引人的地方。对于一个开发商来说，这是一个很强的卖点。几年来，JavaScript 一直是使用最广泛、最流行的编程语言之一。这意味着如果你懂 JavaScript，你就能很快学会 React Native。Web 开发人员可以使用他们现有的 JavaScript 技能来创建移动应用程序。

# 2.使用反应

React 原生使用 React 创建原生移动应用，顾名思义。这是采用 React Native 进行移动开发的第二大好处。React 仍然是创建前端应用最广泛的 JavaScript 库。如果你已经是一名 React 开发人员或者有一个 React 开发团队，那么选择 React Native 来设计移动应用程序是很自然的。在这一点上，学习曲线非常小。这也意味着 React Native 包含了 React 的所有最新特性。

# 3.由脸书支持

尽管 React Native 在 2015 年开源，但脸书继续支持它。一组开发人员正在努力改进 React Native。去年，React Native 团队发布了 [Hermes Engine](https://reactnative.dev/docs/hermes) ，重新架构了 React Native 的关键基础部分。Hermes 是 Android 的开源 JavaScript 引擎，专门运行 React 原生应用。这显著提高了 React Native 在 Android 上的速度。

鉴于高度的社区信任，这些举措都来自脸书团队。脸书的许多产品也使用 React Native。React Native 用于编码脸书应用程序中的市场，以及其他功能。

# 4.卓越的开发人员体验

这些年来，React Native 的开发人员体验数量激增。以下是 React 的一些原生工具和特性，它们帮助我获得了美妙的开发体验:

## 反应本机调试器

React Native Debugger 是一款非常棒的 React Native 应用独立调试工具。它还自带 React Inspector 和 Redux Dev 工具。这个工具提供了比标准 Chrome 调试更好的调试体验。如果你在构建 React 原生应用，我强烈推荐。

## 世博会

[Expo](https://expo.io/) 对 React Native 的改进和对开发者的可用性有很大的帮助。Expo 允许您创建 React 原生应用程序，而无需接触任何原生代码。这是一个 React 原生包装器，让您不必担心原生代码或 Xcode 和 Android Studio 等工具就可以构建应用程序。这对于那些从未在移动设备上工作过的人来说非常理想。Expo 提供了许多 API 来帮助您开发项目。

Expo 是创建 React 原生应用程序的最快方法，同时还能提供出色的开发体验。

## 快速刷新

[快速刷新](https://reactnative.dev/docs/fast-refresh)是 React 的原生特性，当 React 组件更新时，它会给你近乎即时的反馈。默认情况下它是启用的，但是您可以在 React Native developer 菜单中通过切换“启用快速刷新”来关闭它如果启用了快速刷新，大多数编辑应该在一两秒钟内可见。这大大加快了开发过程。

# 5.一次编码—构建 iOS 和 Android 原生应用

React Native 的跨平台能力是该框架的主要卖点。

要用 React Native 制作移动应用，你不需要了解 Objective-C、Swift、Java 或 Kotlin。你可以使用 Javascript 和 JSX 开发一个跨平台的程序。根据我的经验，几乎所有的代码都是在 iOS 和 Android 之间共享的。通过适度的调整来改进两个平台上的最终产品。太棒了，不是吗？要跨平台支持同一个软件，你不需要有多个团队或代码库。

相反，你有一个团队和代码库，可以在 iOS 和 Android 上工作。这对小型企业来说是一个重大的胜利，也是一个金钱和时间的节省者。

此外，您创建的应用是原生应用，而不是在线应用。React Native 中的核心小部件完全是本地组件，为用户提供无缝体验。与它的一些创建网页浏览量的竞争对手相比，这有很大的不同。

# 6.优秀社区

React Native 在过去几年里越来越受欢迎，以至于许多开发人员每天都在努力改进它。开源的 [React Native GitHub](https://github.com/react-native-community) 项目有近 2000 名活跃贡献者。对于一个开源项目来说，这是一个巨大的数量。React Native 是 React 生态系统不可或缺的一部分。除非主题是高度专业化的 React Native，否则大多数查询通常可以由一般的 React 社区来回答。

还有 [React 本地社区](https://github.com/react-native-community) repo，它包含 70 多个存储库。这些包含许多重要的项目和工具，这些项目和工具没有包含在基本的 React 本机 repo 中，但对您的 React 本机应用程序有益。

另一个有很多关于 React Native 的资料和疑问的地方是[栈溢出](https://stackoverflow.com/questions/tagged/react-native)。

必须提到 Expo ，它是 React 本地生态系统的主要参与者。如果你想用 React Native 获得最好的开发体验，我会向你推荐 Expo。这消除了在您的终端上任何本地编码的需要。

社区作为一个整体是巨大的，并在不断增长。之前已经解决了很多困难，在你的开发经历中，你不会是在多此一举。
