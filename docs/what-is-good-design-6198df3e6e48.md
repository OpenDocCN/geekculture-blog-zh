# 什么是好的设计？

> 原文：<https://medium.com/geekculture/what-is-good-design-6198df3e6e48?source=collection_archive---------20----------------------->

![](img/418805936726ce478768d72e1aa5f1c1.png)

作为一名有艺术背景的技术人员，我期待并欣赏功能与令人愉悦的审美相结合。因此，当我看到一个设计糟糕的网站或应用程序时，我自然会退缩。最近，我开始钻研吸引人的设计的心理学。我想知道为什么我觉得缺乏具体的视觉效果，而且完全令人不快。

例如，我不喜欢去看当地的艺术展。由于设计糟糕的传单，我感到很疲倦。直觉告诉我这不值得，但我不顾自己的第一想法还是去了。然而，它把我带到了组织者的网站。我只想知道谁在画廊后面。相反，我无意中发现了我见过的最糟糕的网站之一。从重复的标签到不和谐的配色方案，这个网站还有很多需要改进的地方。

当我们想到好的设计时，我们倾向于坚持视觉，但是好的设计也需要功能。当创建一个吸引人的和功能性的用户界面/UX 时，有多种方法可以遵循。但是首先，我们必须注意 UI 和 UX 之间的明显区别，同时理解它们是如何交叉的。用户界面体现了应用程序的美感。布局、视觉、品牌、输入和输出硬件都在这里。用户体验就是用户的体验。它包括最终用户与产品的全部互动。这意味着用户角色(由研究推动)、旅程和整体结构设计是重要因素。这两个非常不同的概念相互依赖，产生一个非常有效和愉快的设计。

利用行动的七个阶段(Don Norman，日常用品的设计)是一个很好的起点。通过形成一个目标和跨越执行的鸿沟，计划，指定，执行。然后我们就可以跨越评价、比较、解释和感知的鸿沟。最后，我们填补了我们的终点线和现实世界之间的差距。

现在，我们回到之前的糟糕设计。该网站最大的缺陷之一是缺乏凝聚力和明显的错误。当然，当一个人想到一个吸引人的网站时，首先想到的是布局。结构是否清晰、合理、易于使用？解决这个问题的一个方法是遵循 Jakob Neilsen 的 10 个可用性启发法。

1.系统状态的可见性

2.系统与现实世界的匹配

3.用户控制和自由

4.一致性和标准

5.错误预防

6.认可而不是回忆

7.使用的灵活性和效率

8.美学和极简设计

9.帮助用户识别、诊断和恢复错误

10.帮助和文档

这些提示有助于设计师的旅程从开始到结束。例如，采用试探法 8 和 4，想想一个伟大的网站/应用程序与一个不那么伟大的网站/应用程序。你可以立刻想到是什么让这个网站/应用成功或不成功。成功的网站可能在整个 UI 中使用相同的配色方案和形状，以及简洁的设计方法。没有多少明显的错误，比如上面的例子有一个重复的动作按钮。该设计是流动的，这意味着它利用响应或液体文本来防止用户因 F 扫描模式而错过关键组件。

用户友好的设计几乎总是利用试探法 5 和 2。错误是自然的；因此，添加有助于防止错误的约束对于良好的设计至关重要。人们喜欢“熟悉的”，让你的设计尽可能贴近现实生活会给用户一种舒适感。例如，激发这篇文章的网站在他们的“关于我们”中使用了抽象的语言。它没有给读者一个清晰的关于他们业务的视图，这只会增加网站整体混乱的主题。

随着网站/应用成为日常生活中商业和交流的重要组成部分，设计可以成就你，也可以毁灭你。如果用户觉得 UI 太麻烦、混乱或不和谐，就少了一个人在使用你的产品。更进一步，对你的产品/品牌的用户界面/UX 有一个清晰的认识可能会被用来保护资源。Web Apps America 的联合创始人布莱恩·迪格斯和首席运营官谈到了用户界面和 UX 设计的联姻。“当你谈论筹集资金时，能够向潜在投资者销售产品的心理是，你有信心向他们推销你的想法。拥有一个强大的用户界面/UX 会给你这种自信，消除产品的模糊性。”这个概念也嫁回了用户；如果你的用户界面/UX 不利于向公众推销你的品牌，你就有失去利润和客户维系的风险。

在当今的数字时代，建立一个没有代码的网站或应用程序比以往任何时候都容易。然而，虽然这给了创作者一个负担得起的和不费力的方式进入市场，你失去了生产高质量的用户界面/UX 的能力。这些工具对于为你的品牌打下基础来说是极好的，但最终总会需要彻底检查。我并不反对使用这些工具，但是它们会限制你的吸引力。

我们是否需要在每个项目中使用所有不同的指导方针；没有。但是，把它们记在心里不会有什么坏处。
