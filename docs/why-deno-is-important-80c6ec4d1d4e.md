# 瑞安·达尔

2009 年 Node.js 的创始人，他的目标是编程[事件驱动的](/geekculture/create-a-basic-server-with-node-js-and-event-driven-programming-really-basic-3337bcece53e) HTTP 服务器，他在 2012 年做到了，我们都喜欢 Node.js，我们像我一样使用它，我们启动我们的包 JSON，我们安装包，我们创建好的项目(有时)等等。

# 但是现在发生了什么呢？

或者我们可以看到 2018 年他发布了一个演示文稿，标题是“我对 Node.js 感到遗憾的 10 件事”。视频在 [YouTube](https://youtu.be/M3BM9TB-8yA) 上，或者在下面观看，我感到震惊。

他很诚实，他告诉我们 Node.js 有很多 bug 问题，为什么他对此感到遗憾，Node.js 的主要问题是安全，linter 可以完全访问你的计算机，另一个是大问题，node_modules 包增加了很多，如果你有多个项目，他们也有 node_modules，等等，这就是为什么这个图像是遗憾的原因！

# 我们可以寻求宽恕

![](img/c5b90d247f4f94169aeeb2c173b2d90f.png)

Example Deno

他解释了为什么 Deno 在某些方面会更好，比如:安全性，因为你可以管理特权，你可以在没有系统写访问的情况下运行脚本，简化模块系统，你只需导入 URL，就这样，减少 node_modules 仇恨的问题，Ryan 喜欢 TypeScript，这意味着我们可以使用 TypeScript 好消息！瑞安很喜欢

# 结论

这使我对 Node.js 的看法发生了一点变化，因为 Node.js 很好，但并不伟大，首先，我真的很兴奋学习 Node.js，但现在有另一个更好的东西可以检验，它们是高级框架，其次，我喜欢学习，这可能是我的下一个挑战，但现在我正在自己的 Node.js 项目中工作，这真的很棒！
