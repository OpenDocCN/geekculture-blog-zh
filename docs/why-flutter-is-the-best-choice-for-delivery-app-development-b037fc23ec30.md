# 跨平台应用开发的兴起

说到交付应用程序开发，iOS 和 Android 是最受欢迎的两个平台。这些平台需要不同类型的代码用于应用程序开发。这种为不同平台开发两种不同应用的场景对客户和移动应用开发公司来说都是一个挑战。

跨平台应用程序开发是创建可以在各种平台上运行的移动应用程序的技术。在这种类型的开发中，应用程序是利用单一的编程语言创建的。这在商业方面更有效率和效果。

一些流行的跨平台开发框架包括微软的 Xamarin、脸书的 React Native、Adobe 的 PhoneGap、Ionic、Cordova 等等。这些框架各有优缺点，并提供各种功能。

# 创业者为什么要考虑跨平台的移动 app？

当谈到移动应用程序开发时，企业和初创企业的所有者通常会从本地和跨平台开发中进行选择。

原生应用程序是专门为一个平台设计的，比如 Android 或 iOS，而跨平台应用程序是为同时在两个平台上运行而设计的。

为两个主要平台开发一个应用程序有很多优势——Android 和 iOS

## 缩短开发时间

跨平台应用程序开发有助于开发人员在单个代码库上编写和工作，而不是创建两个不同版本的应用程序，这样可以节省时间和精力，并更快地开发应用程序。

## 缩短测试时间

由于开发单个应用程序，质量分析师专家需要测试单个应用程序的性能，而不是测试具有各种设备和操作系统的两个不同平台。

## 降低开发成本

通过减少开发时间、工作量和测试应用的时间，交付应用的开发成本会随着分配给开发资源的时间的减少而降低。

## 更短的发射时间

因为交付应用程序的开发比原生应用程序的开发过程花费的时间少得多，所以企业家可以更快地将应用程序推向市场，并获得先发优势。

谷歌不想被不断扩大的移动行业和持续的技术进步所遗忘，于是创建了 Flutter。Flutter 是谷歌最新的跨平台框架，用于开发 Android 和 iOS 应用。

Flutter 于 2018 年底由谷歌发布，此后因其在跨平台应用开发方面的可扩展性和熟练程度而获得了世界范围的好评。现在，让我们更深入地了解一下 Flutter，它的优点和缺点，以及为什么 Flutter 现在如此重要。

# 什么是颤振

Flutter 是一个开源的 UI 框架和软件开发工具包，旨在加速和简化 UI 和应用程序开发过程。Flutter 的发布具有早期跨平台开发框架所缺乏的特性。Flutter 是由谷歌开发的开源平台，允许使用单一代码库开发原生 Android 和 iOS 应用程序。它不是一个框架，而是一个完整的 SDK(软件开发工具包)，包括开发跨平台移动应用程序所需的一切。

Flutter 是唯一一个不需要 JavaScript Bridge 就能提供反应视图的跨平台框架。谷歌开发人员在使其广泛可用之前一直在努力工作。他们从事以下工作:

*   Windows 应用程序开发支持
*   支持大量 Firebase APIs
*   增强的文档
*   国际化
*   聊天、广告和在线视频都受支持。
*   Visual Studio 和 Android Studio 工具
*   错误修复
*   所有类型的开发人员都可以访问它

既然我们可以清楚地证明 Flutter 在所有其他跨平台框架上的优势，那么是时候研究一下 Flutter 在企业家市场上是一个前锋的原因了。
