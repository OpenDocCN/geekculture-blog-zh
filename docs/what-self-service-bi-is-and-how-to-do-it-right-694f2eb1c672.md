# 什么是自助 BI

# 自助 BI 定义

自助式商务智能通常被定义为一种商务智能形式，它使用简单易用的商务智能工具，允许不精通技术的业务用户(销售、财务、营销或人力资源)直接访问数据并自行探索。

自助式商业智能不同于传统商业智能，传统商业智能是由 IT 或商业智能部门作为一个集中的功能拥有的。在传统方法中，是这些团队负责一切。他们准备所需的数据，存储和保护数据，构建数据模型，创建查询，并在收集最终用户的需求后为他们构建可视化。

自助式 BI 的理念与数据民主化密切相关，数据民主化的重点是让组织中的每个人都能访问和使用数据。最终目的是在组织层面产生更多的洞察力，并推动更好的业务决策。

# 自助 BI 的主要优势

*   **更快的洞察时间** —将控制权转移给最终用户意味着跳过传统 BI 流程中耗时的阶段。在自助式 BI 中，最终用户不必等待数天甚至数周，直到他们的报告在通过诱导和批准后最终上线。当他们意识到需要更多的视觉效果时，他们也不必处理繁琐的变更请求管理过程。这是因为他们可以即时删减、调整和添加数据，以揭示重要的趋势、模式或异常。
*   **提高运营效率** —通过为业务用户提供全面的领域知识，使他们能够临时执行自己的数据分析，自助 BI 可以产生更高质量的见解，同时将 IT 或 BI 团队从处理与数据相关的日常任务中解放出来。相反，这些团队可以专注于更困难的问题，如建立数据管道，在正确的时间将净化和转换的数据传输到正确的目的地，以及维护重要的数据治理流程。
*   **降低成本** —除了优化 IT 和 BI 功能以节省时间和成本之外，许多自助式 BI 采用者更进一步。他们用执行[高级数据分析](https://itrexgroup.com/services/data-analytics/)的知识和工具武装主题专家。换句话说，他们培养了公民数据科学家，这些科学家知道如何生成对商业至关重要的 ML 驱动的预测。随着数据科学人才的价格越来越高，这种投资可能是数据驱动型公司能够做出的最佳投资之一。

# 自助 BI 工具的核心功能

为了实现上述自助 BI 的强大优势，自助 BI 工具应具备以下基本特性:

*   **数据连接器**支持自助 BI 工具与数据库、CRM、ERP、营销分析、财务软件以及其他内部和云系统集成，以最有效的方式满足分析需求。
*   **强大的报告功能**从具有可定制设置的书本质量的预设报告到临时的深入查看，同时允许用户计划分发或将结果划分为不同受众的子集。
*   **直观的拖放式或点击式界面**，允许用户选择数据字段和视觉效果，并将其拖放到报告画布中，以供浏览和讲述。
*   **数据可视化模板**简化了基于用户偏好和需求创建仪表板的流程。

许多组织通过在[数据科学](https://itrexgroup.com/services/data-science-consulting/)和[机器学习](https://itrexgroup.com/services/machine-learning-development/)中增加功能，将他们的自助 BI 提升到一个新的水平。增强分析平台使用户能够发现更多数据，评估无特征数据集，并创建假设情景。通过这种方式，企业可以尽快对不断变化的需求做出反应，实现最大的灵活性。

# 为什么组织在自助式商务智能方面会失败

**1。不切实际的期望**

一个刚刚开始向新手用户扔数据的组织正面临着报告质量差的严重风险。如果这些具有不同资质的用户在没有首先学习报告的基础知识的情况下，最终获得了没有被误解的数据，那将是非常幸运的。

例如，一个快乐的用户创建了关于历史时期总销售额的第一个报告，可能最终得到的是平均数而不是总和，对各种度量的默认聚合一无所知。或者相反，他们可以提交夸大的数字。还存在数据不一致的风险，这可能会影响加权平均值，因为它们需要以不同的粒度级别显示。

更进一步，非超级用户可能对支持他们最初信念的偶然分析感到满意。未受过训练的用户不一定会意识到确认或挑选偏见陷阱，尤其是在解释某种模式的压力下。

**2。举报混乱**

自助 BI 并不意味着零 IT 参与。让用户在没有 IT 治理的情况下摆弄数据通常会导致报告混乱。

如果没有治理，可能会有来自不同用户的冗余报告，这些用户在孤岛中工作并交付相同的分析，或者来自不同用户的报告分析相同的指标，但使用不同的过滤器，因此交付相互冲突的结果。不同部门的报告可以依赖不同的数量、价值或时间命名约定，或者使用相同的术语，但不一定使用相同的定义。同一个数据库的多个版本，数据库中从未修复的错误，创建只使用一次的对象……这样的例子不胜枚举。

在自助服务的世界中，治理不是一个数据驱动的组织可以抵制的东西。无论一家公司多么希望让用户自由进行他们自己的分析，它仍然需要参与进来，以保持高数据质量和一致性。

**3。缺乏采用**

事实是，不是每个人都喜欢努力工作。大多数商业用户只是想要一个简单的仪表板，给他们数字。然而，有价值的见解往往存在于比简单的业务绩效分析更深的层次。

另一个可能阻碍高效自助 BI 的心理因素是对变革的抵制。对于许多组织来说，在自助 BI 之旅的早期阶段，经常会看到沮丧的业务用户回到 BI 或 It 部门来请求报告，就像他们在过去的美好时光中所做的那样。旧的方法更安全。

不友好的自助式 BI 环境设置也可能是一个问题。对于 IT 或 BI 团队来说，收集和提炼结果似乎是一个易于使用的工具，但对于一个没有技术技能的普通用户来说，它可能具有大量令人沮丧的功能。枢纽表格和电子表格可能很枯燥，但是用户[在遇到困难的时候会很快回到它们上面](https://itrexgroup.com/blog/factors-behind-data-analytics-cost/)。

# ITRex 关于如何成功实施自助 BI 的 10 个技巧

以下是从 ITRex 为小型企业和大型公司构建高效自助 BI 工具的经验中获得的基本要点，其中包括拥有 300 万商业用户的全球领先零售商(点击此处阅读更多 about‌此项目信息):

*   **设置您的自助 BI 策略**。您首先需要定义您希望通过自助 BI 实现的目标，无论是减少延迟报告还是提供组织范围内的数据访问。自助服务对不同的人来说意味着什么，所以你应该清楚你的项目。尽早了解实现的规模、用户的类型、他们的技术熟练程度以及您对可交付成果的期望也很重要。
*   **让所有利益相关者参与整个项目**。您应该考虑您的利益相关者在数据中寻找什么，以及他们与数据相关的成功指标是什么。采访他们，收集他们的功能、可用性、用户体验和其他信息。然后在迭代过程中不断向他们寻求反馈。除了确保您构建了一个相关的自助式 BI 工具之外，您还将让您的利益相关者有一种主人翁感，并提高他们的参与度。
*   **让 IT 部门参与**。这也是必不可少的。您的 IT 拥有关于您的数据环境、现有数据源、适当的数据治理控制和数据访问管理的所有信息。他们将帮助您选择或构建自助 BI 解决方案，该解决方案在用户访问和新数据源集成方面易于维护、监控和管理。
*   **建立健全的治理**。自助式 BI 治理包括以下内容:

-数据治理政策和程序，确保您的数据一致、完整、完整、准确和最新。在这里，您需要[开发一个更广泛的数据管理策略](https://itrexgroup.com/blog/data-management-strategy-benefits-principles-steps/)，并采用主数据和元数据管理方面的领先实践作为其中的一部分

-管理业务指标，在您的自助 BI 环境中统一定义这些指标，并排除任何偏差

-管理报告，以设定质量验证程序

-数据安全性，定义谁可以访问您的自助 BI 中的哪些数据，并建立数据血统

*   **选择合适的工具**。没有放之四海而皆准的策略。您的用户有不同的需求和技能，您的工具应该准确地迎合这些需求和技能。您可能需要在灵活性和复杂性之间取得平衡，以允许您的用户在保持自立的同时提出新的问题。定制的自助式 BI 解决方案将使这一目标更容易实现。
*   **建立单一的真相来源**。单一事实来源作为解决方案架构的一部分实施，以支持基于相同数据的决策。为此，公司建立了一个[数据仓库](https://itrexgroup.com/services/data-warehouse-consulting/)或另一种中央存储库，提供来自多个来源的所有数据的 360 度视图，并使数据访问、分析、丰富和保护更加简单和高效。值得投资。
*   **教育用户**。对最终用户来说，三种类型的培训计划是必须的:1 .数据分析和可视化，2)连接数据和建立数据模型的基础，3)持续的点对点培训。
*   **建立社区**。如果你建立一个卓越中心，或者在 Slack 或团队上有一个专家社区，这将会有很大帮助，这样你的最终用户就知道去哪里填补知识空白。
*   **考虑在业务部门中嵌入 BI 专家**。它们将增加没有分析背景的用户对数据的访问，并提供更高质量报告所需的监督，从而有助于推动参与。
*   **开始小**。选择一个有限的环境来启动您的自助 BI 项目，并使用敏捷方法从那里开始构建。这样，您可以在扩大规模之前尽早解决问题。
