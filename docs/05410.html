寻找符合您需求的正确机器翻译型号

上面你会注意到，Helsinki-NLP/opus-mt是所有型号通用的。只是其余部分各不相同。这意味着所有的模型名称都具有以下格式:Helsinki-NLP/opus-mt-{ src }-{ tgt }，其中src和TGT占位符包含语言代码。所以，对于英语到印地语的文本翻译，模型名称应该是Helsinki-NLP/opus-mt-en-hi。

赫尔辛基-NLP/opus-mt-en-hi简介

该模型在 OPUS 数据集上进行训练。这个开放的平行是来自网络的翻译文本的集合。它还包括维基百科、维基资源、维基图书、维基新闻和维基百科网页的翻译。这个 GitHub页面将提供从wiki网页下载源文本和目标文本的链接。并且所需的预处理步骤还包括使用句子片段库对文本进行分词。我不会深入讨论他们的架构和其他细节。但是，如果你有兴趣了解更多，那么在这个 GitHub repo 中，你可以深入挖掘，找到你所有的答案。
