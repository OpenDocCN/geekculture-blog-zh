# 需要知道的重要概念:管道

要创建机器学习模型，理解管道的概念很重要。流水线是机器学习过程自动化的一种方式。我们可以在一个管道中完成几个连续的步骤。然后，您将只需要运行一次管道，而不必单独重新运行您的步骤。下面是一个关于管道的短片，如果你想知道更多的细节。

Remember to like, comment and subscribe :)

那么如何使用管道呢？简单，自己造。真的就那么简单。不容易，但就是这么简单。建立自己的管道。使用模型搜索、数据扩充、数据清理、预处理和评估的独特组合。在那里你有自己的管道。这听起来可能不是特别令人印象深刻，但仅此而已。如果你有一个适合你的组合，人们会投资进去。

所以你如何找到你的工作组合。以下是一些对我有用的方法:

# 研究

我从事研究的方法之一就是为一些咨询工作找到最好的技术，我叫丁。这不仅仅是找模特。我遇到了许多伟大的想法/概念，如基于排列的重要性、TSNE、贝叶斯优化和随机扩增。当您开始设计自己的管道时，所有这些都很重要。

那么，你该怎么做呢？如果你没有时间或兴趣去挖掘你所在领域/利基的最新动态，那就跟随创造者吧。跟随像我这样的人，Yannich Kilcher，两分钟论文，亨利 AI 实验室等。追随多个创作者的好处是，它会让你接触到许多想法和实现，超越你固定的领域。例如，两分钟论文有一系列关于教机器人走路的强化学习的视频。这让我了解了一个我并不涉足的领域，并将其中的一些知识运用到我自己的工作中。

# 反复试验

接下来是这个过程中不性感的部分。你需要做大量的尝试和错误。你会发现最奇怪的事情。以著名的图书馆熊猫图为例。它用于 EDA(探索性数据分析)。你知道吗，它一完成报告，就会关闭所有的人工智能。这不是任何人都知道背后的原因。我是怎么知道的，我在用 Sweetviz 做工具时注意到了这一点。你可以在这里看到我提出的问题[，如果你想解决它的话。](https://github.com/fbdesignpro/sweetviz/issues/56)

试错法还会让你对代码和过程有一个大致的了解，让你能够构建和扩展这个工具，因为它会产生更多的兴趣。

# Beta 测试+客户反馈

这将非常重要。透过表面现象看问题是很重要的。在我用 Clientell 构建的 AI 的最初版本中，我们只是生成报告。这些报告确实很全面，但是我们把解释留给了用户。这是因为我们可以让他们选择如何根据用户认为最好的度量性能(我们有很多)和模型学习来确定最佳模型。反馈非常积极，因为与我分享工具的人都是数据分析师和机器学习专家。他们很欣赏生成报告的便捷性。他们欣赏管道和流程。

![](img/58802295b7403c1c342959527d22c80e.png)

Remember, different demographics= different Must have Features

接下来是难题。企业用户一点也不领情。他们不在乎这些报告。他们一直问我，“我怎么知道什么是最好的模型？”。我总是回答，“那要看你的需求，看报告，打那个电话。”这次谈话让各方都非常沮丧。最后我听了 CEO 的话，建了一个推荐模型。对于回归，我们选择最低的 MSE。对于分类来说，这是最高的精确度。几乎没花什么时间。**但是人们的兴趣一飞冲天。**

从不同的客户群体中获得反馈将有助于你获得反馈。记住，人们会看重不同的东西。这意味着不同的功能将受到重视/至关重要。Clientell 的技术用户仍然可以根据所有报告做出自己的决定。商业用户可以使用我们的建议。我得到报酬。工具获得大量投资。各方皆大欢喜。
