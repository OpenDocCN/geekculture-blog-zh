虽然RudderStack在客户数据堆栈中有许多用途，但大多数开发人员和数据工程师的主要用例是从数据角度消除困扰营销项目的痛点(就像我们在这里概述的)。开发人员可以更轻松地从数据层管理基础设施，并通过他们想要的工具获得他们想要的营销数据，而不是被迫使用为营销人员构建的工具，费力地构建内部工具，或处理总是有问题的Google Tag Manager。

这里概述了工程团队使用RudderStack从数据层支持营销的具体方式。

步骤1:整合和标准化第一方数据收集

当涉及到第一方数据时，营销面临的一个核心挑战是事实的不同版本。在比较两种不同的营销/销售工具时，症状往往会显现出来。你可能听说过“Google Analytics中的数据与HubSpot中的数据不符”，或者被问到“为什么Klaviyo、Salesforce和我们的Looker报告之间有这么多差异？”

数据差异几乎总是会发生，因为各种工具使用不同的技术机制来收集数据，并且每个工具的定义和格式都略有不同。回到上面的例子，Google Analytics和HubSpot都需要你运行它们的JavaScript代码片段，它们收集浏览行为的方式也不同。Klaviyo和Salesforce具有原生同步功能，但自定义字段、列表成员资格和其他数据点不能很好地转换。Looker数据来自仓库，其中的数据可能已被分析团队清理干净。随着堆栈中工具数量的持续增长，这个问题变得更加严重。

解决方案是将追踪整合到一个单一来源，并标准化数据收集。

这个问题的本质要求在源头进行标准化和简化，这正是RudderStack的SDK所支持的。有了RudderStack，您只需跟踪一次，我们简单的JSON有效负载遵循标准化的模式。

这里有一个例子。假设你是一家电子商务公司，营销团队使用优惠券让人们注册他们的每周交易简讯。营销部门希望在新闻通讯注册活动中添加额外的数据。具体来说，他们想知道促使注册的优惠券的名称。通常，这将是一个巨大的痛苦，需要为每个跟踪注册的目的地(Google Analytics，Klaviyo，Salesforce等)编写稍微不同的代码。)…这还没有考虑更新广告平台的转化率(我们将在下面讨论)。

在RudderStack中，您只需检测一个注册事件并识别新用户。您可以发送名为“时事通讯_注册”的. track呼叫。要添加优惠券代码，您只需获取值并将其添加到track call的属性中，这将向每个营销工具发送相同的标准化有效负载(下面将详细介绍集成)。您还可以识别用户，以便在营销、电子邮件和CRM工具中创建联系人记录。

对于开发人员来说，这意味着大大简化了第一方跟踪工具的工作流程，更重要的是，简化了正在收集的数据的标准化。这也意味着控制——在谷歌标签管理器中不再有惊喜标签……事实上，你可以完全取消谷歌标签管理器(哈利路亚！).最重要的是，模式可以在您的代码库中进行版本控制，因此您可以确切地知道正在收集什么数据。

对于您的营销团队来说，这意味着跨工具的数据一致性，以及当他们向您和工程团队请求数据时更快的周转时间。

步骤2:从等式中移除积分工作

即使您将数据收集标准化，您仍然需要将它发送给市场营销使用的所有工具。通常这意味着开发人员必须做大量的定制API工作来发送标准化的有效载荷。

RudderStack消除了等式中繁琐的积分工作。您仪器的标准化事件可以自动发送到您堆栈中的任何目的地，包括营销工具。没错——不需要集成工作。让我们回到上面的“时事通讯_注册”和用户识别示例。

当identify调用运行时，RudderStack将在所有适用的下游工具中创建用户记录。在这种情况下，用户将在Klaviyo、Salesforce和您在RudderStack中作为目的地添加的任何其他下游工具中创建。如果您将您的数据仓库添加为目的地，我们还将更新用户表，以便您的Looker报告具有最新的数据。

“时事通讯_注册”。跟踪呼叫是一个行为事件，并不适用于每个目的地。在这种情况下，营销部门希望将事件发送到Klaviyo(以便他们可以将这些注册划分到一个列表中)和Google Analytics(以便他们可以分析转换漏斗)。同样，RudderStack将自动用事件信息填充Kalviyo和Google Analytics。

步骤3:简化ad平台的客户端和服务器端转换管理

对于开发者来说，唯一比与营销的云工具进行定制集成更糟糕的事情是处理将转换发送到付费广告平台的巨大混乱。

在脸书、谷歌、Pinterest、必应等网站上进行专门的转化活动。需要持续关注营销运行实验，这意味着大量定制代码或痛苦的故障排除和谷歌标签管理器中复杂的排序。呸。

在RudderStack中，您的现有事件可以作为转换直接发送到广告平台，无论是客户端还是服务器端。所以，让我们说，营销想要创造一个再销售受众，为每个订阅时事通讯的人提供广告。要做到这一点，您只需将您的广告平台添加为目的地，然后将您现有的“简讯_注册”活动映射到平台中相应的转化(例如，对于脸书，您可以将“简讯_注册”映射到潜在转化)。

步骤4:使用转换动态定制和修复目标集成

无论你如何简化你的数据流向市场营销工具，市场营销的需求会随着他们在实验中的学习而改变。这意味着不可避免地，数据有效载荷将需要更新，并且需求将改变，即使是在逐个目的地的基础上。

例如，假设营销团队希望将`most_recent_coupon'值推送到Salesforce，但他们创建的自定义字段是“Latest_Coupon_Code__c”。您可以在RudderStack转换中使用简单的JavaScript为Salesforce目标定制有效负载，而不必向原始事件添加另一个键。在这种情况下，您可以将“most_recent_coupon键重命名为“Latest_Coupon_Code__c”。

因为转换是用JavaScript编写的，所以没有限制，我们的客户已经找到了数百种用途，从定制到即时修复。

export function transformEvent(event) {
if (event.event == 'most_recent_coupon') {
event.event = 'Latest_Coupon_Code__c';
}
if (!event.properties.sale_item) {
event.properties.sale_item = false
}
return event;
}
