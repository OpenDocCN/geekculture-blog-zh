新手开发者通常会有很多问题。如何接近学习过程？从哪种编程语言开始学习？其中一个问题是选择前端还是后端？

其实还有更多的领域:移动开发、数据工程、机器学习等。然而，可以肯定地说，移动开发更多的是前端，而数据工程是后端。

在这篇文章中，我想强调两个方向的利弊，以便新人对它有一个更清晰的了解，并在未来减少遗憾。

人们通常建议从前端开始，因为它有即时反馈。当您编写应用程序时，您会立即在屏幕上看到变化。相反，你可以在后台写很多代码，但是没有现成的 UI，就不容易接触到产品。而且产品越大，其架构越复杂，所以实际上，在一小块接口的背后，可以隐藏几十个服务之间的调用，使用不同的通信协议。

前端开发人员很少与其他团队合作。即使在大型项目中，作为一项规则，他们也要与团队中的后端开发人员就 API 和设计人员进行交流，仅此而已。而后端人员经常不得不与其他团队的同事讨论架构的一般方面。

对于许多人来说，诱人的一点是，前端工程师没有被请来解决客户的问题。事实是，大多数情况下，后端工程师能够识别 UI 端的问题，但后端发生的事情对于前端工程师来说是一片黑暗的森林。因此，通常会有 1-2 名后端人员和/或一名团队领导负责此类电话。

尽管 UI 看起来很简单，但强大的前端开发人员必须了解不断变化的相当广泛的堆栈。每天都有关于新 javascript 框架的东西，这与事实相差不远。此外，在不降低成绩的情况下，在高级职位上切换到后端是相当具有挑战性的。但另一方面，转行做移动开发就容易多了(感谢 Flutter 和 React Native)。然而，有许多 node.js 后端/中间件(例如，使用 Nest.js ),在那里切换要容易得多。

从后端开始的问题是需要大量的初始知识。你可以从 HTML/CSS 开始学习前端，已经找到这个职位的工作，然后扩展你在现代 javascript 框架方面的专业知识。在后端，即使是初级职位，你也已经需要能够处理数据库、迁移、文件系统，对其中一个框架有相当的了解，更不用说算法了。

两者各有利弊。我个人的选择是后端，因为一旦我享受了一个经过深思熟虑的架构的美丽，就不可能用它来为不同类型的浏览器进行样式编辑。
