## 如何保证 API 的安全

![](img/d46566f272dd3dd7b38396fa124d4785.png)

Photo by [Nahel Abdul Hadi](https://unsplash.com/@nahelabdlhadi?utm_source=medium&utm_medium=referral) on [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

API 安全性是一个日益严重的问题。在这篇文章中，我将向您介绍 API 面临的主要安全威胁，以及如何保护您的 API 的最佳实践。API 安全是一个广泛的领域，需要考虑许多漏洞、威胁和保护措施。API 面临着与软件和网络应用相同的威胁。鉴于它们有一组独特的漏洞，API 面临一组独特的威胁。我将所有这些归结为您应该防范的六个主要安全威胁。这些是基本的软件错误、注入威胁、服务、可用性威胁、用户身份威胁、内部威胁以及垃圾邮件和滥用。

# 1.软件缺陷

第一个 API 安全威胁是您的基本软件缺陷。防范基本的软件错误是一个组织定期测试和评审的问题，这对你的成功至关重要。首先，确保您的组织拥有良好的软件工程。这意味着使用标准化的编码指导方针和制定代码评审。通过在您的环境中重现安全问题、修复它们并快速将其投入生产，确保新代码得到定期测试。如果您还没有实施安全审查，请实施安全审查。这样做可以确保一个 API 面临的安全问题不会在另一个 API 中再次出现。您还可以考虑参与第三方渗透测试，从外部的角度真实地了解您的系统、数据和 API 有多脆弱。

# 2.注射威胁

API 还面临注入攻击威胁，如 SQL 注入、远程代码执行或跨站脚本。当有人向易受攻击的软件程序中注入恶意代码时，就会发生注入攻击。幸运的是，由于 API 有开放的 API 规范，有明确的输入和输出，使用一致的输入验证很容易防范这些威胁。像 Apigee 这样的 API 管理平台提供了许多策略，允许您轻松地部署输入验证。

# 3.服务可用性

服务可用性威胁也是组织非常关心的问题。这些攻击通过向您的网络发送大量请求来攻击 API，从而导致拒绝服务问题。为了防范这些攻击，您可以利用容量度量防御工具，如速率限制和配额策略。将这些工具与前端负载平衡器等专门处理 DDoS 保护的其他云工具结合使用，是一种值得遵循的最佳实践。

# 4.用户身份威胁

虽然许多人认为良好的客户体验是在购买过程中支持和指导客户，但良好的客户体验也意味着在每时每刻保护用户及其身份。为了防止用户身份威胁，您将希望使用一个 API 网关，该网关允许您集成 OAuth、JSON web 令牌、签名、API 密钥和更多用于验证和确保身份验证和授权的防御机制。有了 API 管理，您不需要为了安全性而牺牲用户满意度。构建和部署开发人员门户是一种很好的方式，可以让您的 API 消费者获得自助服务体验，从而了解您所公开的 API 并请求他们所需的访问权限。像 Apigee 这样的工具提供 API 监控和分析仪表板。有了这样的仪表板，您可以获得 API 程序健康和安全的整体视图。

# 5.内部威胁

内部威胁是企业安全面临的一个日益严重的问题。它提醒我们，安全漏洞不仅来自外部，也来自内部。让我用谷歌云作为例子。由于组织中有如此多的员工和承包商可以访问敏感信息，防御内部威胁的第一步是将您的 API 管理系统和 Google 云控制台与您的企业单点登录系统(SSO)集成。您还希望集中和结构化您的变更管理过程，以便开发人员能够清楚地看到您的软件基础设施发生的任何变更。它包括了解谁在进行变更，有哪些变更，谁批准了这些变更，以及进行这些变更的必要性。

# 6.垃圾邮件和滥用

让我们稍微讨论一下垃圾邮件和滥用。相当数量的 API 流量来自参与凭证填充、暴力破解和内容抓取的自动化参与者。要记住的最重要的事情是，你不能保护你的 API 免受你看不见的威胁。通过 Apigee 这样的 API 管理平台建立 API 流量可见性。有了这样一个工具，你将能够观察和利用良好的合法交通。你也将能够识别和阻止坏演员。

总之，API 的几个安全威胁从基本的软件错误到垃圾邮件和滥用。API 管理是您的安全架构的重要组成部分，它将帮助您花更少的时间来应对安全问题，而将更多的时间用于创新。在当今日益数字化的世界中，评估改进机会并根据感知到的风险确定安全工作的优先级是开展业务的重要组成部分。让您的安全防御变得直观、智能和可靠。保持安全，让您的企业保持活力。
