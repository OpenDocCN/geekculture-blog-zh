在过去参加了200多次与前端开发相关的面试后，我总结了应聘者在面试中最常犯的需要避免的错误。明天你可能会参加下一次前端开发人员面试，阅读这篇文章肯定会帮助你克服所有的问题。写这篇文章主要是考虑到虚拟面试。

如果你厌倦了阅读这篇文章，你可以在这里看我的15分钟视频，或者你也可以继续阅读我的文章。

错误1:杂项

让我们先涵盖所有杂项错误。

  1. 没有合适的备用电池。
  2. 没有备用互联网。
  3. 身边没有纸和笔。
  4. 没有水瓶/能量饮料
  5. 面试前至少2分钟不参加会议。

避免以上所有错误。

错误二:试图提问太快

不要急于提问，很多时候，应聘者甚至不会等到面试官完全解释完问题。突然他们会开始，卡在中间。请避免这种情况。

错误3:没有问正确的问题

就给定的问题向面试官提出正确/相关的问题。比方说，面试官会要求你用JavaScript创建一个秒表。以下是相关问题的示例。

  1. 所有计时器都包含什么？例如:秒、分、小时等。
  2. 是造型重要还是功能重要？
  3. 单元测试有必要吗？
  4. 定时器应该何时结束等。

错误4:提交时间更短

面试官通常会为他们要问的每个问题规定一个时间。如果他们要求你在15分钟内实现上面的计时器，但你不确定是否能在15分钟内完成，但你需要20分钟。然后随意对面试官说。他们会尽量从另一个问题中挤出时间，交给这个。如果你不明确地问，他们自己永远也不会知道。

错误五:没有先写出症结

假设我要求您在React访谈中使用redux实现一个简单的计数器。大多数从建立递增/递减按钮和点击按钮时触发功能开始。你需要明白，面试官想知道你的重复技能。所以，首先请尝试写它，然后开始建立一个用户界面。即使你没能创建一个漂亮的用户界面，如果他们的重要性在于功能性，那么你就可以取消面试。
