## 初级开发人员犯的 8 个最大和最常见的错误

在我们进入正题之前，我想祝贺你——你已经走上了成功的道路。这需要奉献精神、学习意愿、专注和自律。

我试图以我的方式概述和系统化初级开发世界中最常见和最大的错误。谁知道呢，也许我能让你在未来的道路上免于失望和沮丧。

1.  初级开发人员通常倾向于低估终极基本技能；
2.  有时他们缺乏正确的动机；
3.  他们倾向于过分依赖他们的高级开发人员；
4.  相反，他们倾向于更少地依赖年长者；
5.  他们希望很快就能在酷项目上签约；
6.  有时他们对初级、中级和高级角色的真正含义缺乏正确的理解；
7.  永无止境的*数量超过质量*问题；
8.  他们全心全意地相信各种来源，没有任何批判性的方法；

现在我想分享一些我对我的每一个列表的想法。如果他们与你不一致，请随意跳过一些。

## 初级开发人员通常倾向于低估基本技能

有时我会为我的 WordPress 代理处的职位安排面试——我喜欢为团队寻找新成员！在讨论 web 开发部门的未来工作职位时，我分配了一个示例任务。当然，我的目标是检查对方在这方面的技能和知识。

你看过烹饪节目《厨艺大师》吗？我们经常看到最后的前十名参赛者在有史以来最基本的任务中失败——切洋葱、煮鸡蛋，你懂的。我惊奇地发现，当涉及到复杂和高级的 web 开发技能时，拥有出色和令人印象深刻的简历的人有时会在执行最简单的基于 HTML 的任务时失败。

**我的建议:**永远不要低估基本面的力量。当谈到追求网页开发的职业生涯时，确保你完全受过良好的教育，熟悉最基本的知识。

## 有时他们缺乏正确的动机

听着，我知道程序，我不会评判你——IT 行业以其总工资、在任何地方工作的可能性、灵活的工作时间等而闻名。

但关键是——在“赚大钱”之前，一个人必须获得专业技能、正确的工作态度和执行项目的正确愿景。如果你**的目标是掌握这门手艺**，那么财务上的提升也就顺理成章了。但首先——努力工作。

我的建议是:不要仅仅为了钱而进入 IT 世界。很有可能你迟早会感到精疲力尽。相反，试着专注于学习和你知识的有机扩展。

## 他们倾向于过分依赖他们的高级开发人员

web 开发和 IT 行业高度支持共同项目、合作以及在需要时寻求帮助的想法。这是我个人认为我的职业领域如此伟大的最好原因之一。

但是，当谈到三年级学生时，我经常看到他们对挖掘足够的信息、资源、案例研究、项目样本缺乏态度。通常，他们会带着问题直接去找他们的上司，期望他们给出解决方案。这种方法减缓了低年级学生在该领域的发展，同时也增加了高年级学生已经堆积如山的任务。

**我的建议:**尽一切办法——和学长讨论问题。但是首先，确保你已经收集了所有可能有用的数据和信息。不要把注意力放在所有你做不到的事情上，而是尽可能把注意力放在所有你能正确做的事情上。然后向你的上司提出这个问题。

## 相反，他们倾向于更少地依赖年长者

在我们的行业中，我们都知道趋势是不断变化的——事实上，第二天我们都会对一些全新的软件添加感到惊讶。Web 开发确实是快节奏和动态的——构建网站的人必须始终掌握最新的技术和软件。

话虽如此，我还是想对一些低年级学生说几句——是的，我知道你在研究和灵活性方面遥遥领先，但我只想提醒你，你可以从年长的程序员那里学到很多东西。当然，他们的方法可能与你的大不相同，但在我们这个领域，真正的财富是从不同的角度和视角来看待项目。

**我的建议:**有时候去找源头。和在这个领域有丰富投资组合和多年经验的人讨论一个网站开发的话题，不管这个话题有多“时髦”。

## 他们希望立即签署“酷项目”

当然，我们都想从事下一个酷的、现代的、懂技术的、新潮的项目，但是也许暂时不要着急也不是一个坏主意，尤其是如果你刚刚步入 it 世界的话。为了让你成为一个重大而令人兴奋的新项目的一部分，你**首先必须完成并积累一些知识和经验**。

不要忘记，权力越大，责任越大。最热门和最酷的项目通常是由年长者领导的，因为他们有能力处理如此艰巨的任务。在瞄准一个在编码的第一个月就有数百万购买量的电子商务网站之前，要三思而行。

我的建议是:相信过程，不要匆忙。在过渡到更大的项目之前，把你的时间和精力投入到自我教育、积累经验和各种小项目上。

## 有时他们对初级、中级和高级角色的真正含义缺乏正确的理解

正如我们已经说过的，IT 行业的节奏非常快——这种动态允许昨天的新手成为明天的领导者，仅仅是因为越来越多的项目可能需要管理。

当然，为了让你正确认识你未来的工作，对你来说，充分了解大三、大四学生应该做什么是很重要的:

*   初级开发人员通常被处理最基本和最普通的任务(但仍然是非常重要的任务，因为它们是基本的)；
*   总的来说，中级开发人员也在处理任务，但是主要的区别是他们应该更有资格和更自主——他们很少缺少监督；
*   高级开发人员经常管理整个项目，他们与 UI/UX 设计师密切合作，他们完全依靠自己开发应用和网站；

## 永无止境的“数量重于质量”问题

让我们从稍微不同的角度来讨论这个问题。

如今许多初级开发人员试图通过**掌握**比如所有编程语言来给人留下深刻印象。我很欣赏这种努力工作的态度，但是让我们现实一点——很有可能到最后，你不会在其中任何一项上特别出色。

我也经常看到完全相反的情况——大三学生完全专注于一项 web 开发资产，完全忽略了浩瀚的知识和能力海洋。

我的建议是:在最好的情况下，我会推荐聪明的学习和工作方式。让我们把这两种方法结合起来——我认为对大三学生来说，掌握可以一起派上用场的技能是最好的。例如，如果你热衷于使用 WordPress，一个很好的方法是结合 PHP 和 JavaScript。之后，向 React.js 的过渡会更加平稳和容易。

## 他们全心全意地相信各种消息来源，没有任何批判的态度

最后，我想提醒你，网络上有大量的信息资源。你能做的最好的事情是安全地选择其中最可靠和最需要的。不要相信你读到的任何东西，总是要反复检查。
