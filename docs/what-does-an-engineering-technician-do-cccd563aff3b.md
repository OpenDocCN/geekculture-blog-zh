鉴于这一事实，对软件应用程序和软件专业人员的需求增加就不足为奇了，其中，工程技术人员起着至关重要的作用。

在本帖中，我们将更详细地探讨他们的角色和职责，并涵盖企业在让他们加入团队时应该牢记的最佳实践。

# 了解计算机工程领域

为了理解工程技术人员的角色、职责和日常活动，我们必须首先了解他们工作领域的各个层面。

计算机工程是结合了计算机科学和电气工程的知识和技能的工程领域。它要求将数学、工程和科学中的技术概念应用到软件工程中。

计算机工程的多学科性质包括与计算机系统和一般技术设备的设计和开发有关的一切。

工程技术人员扮演着独特的角色。他们的目的是充当工程团队和其他技术团队之间的联络人，无论是制造团队还是质量保证团队等。

他们的职责包括进行工程设计审查，与供应商协调以确保符合规格，并确保计算机硬件符合环境条件。技术人员还与工程师联系，以便解决与正在开发的软件应用程序相关的问题。

让我们更仔细地看一下专业人士担任这一角色所要完成的主要任务。

# 工程技术人员的日常工作

工程技术人员需要对计算机硬件进行分析、测试和故障排除，以便工程团队实施工程师的设计。虽然这听起来可能是一项单调乏味的任务，但它是必要的，因为工程技术人员必须确保符合热限制等规范，并确保计算机硬件符合环境条件。

以下是工程技术人员经常进行的活动列表:

*   对计算机硬件进行必要的测试和分析，以确保符合工程设计。
*   协助工程师开发、设计或创建计算机组件，如电路、芯片组和软件程序等。
*   通过分析来自可用来源的数据，为新的计算机系统制定计划。
*   构建和测试原型以验证和改进设计。
*   在生产环境中进行质量控制活动。
*   完整的技术文档、计算机维修和计算机集成制造。

这样，他们可以确保在产品推出后，产品设计过程中使用的所有机器仍能正常工作。

# 工程技术人员的重要性

如果没有计算机工程技术人员，软件开发工程师将无法专注于设计新产品或为测试目的创建原型的任务，而是不得不花时间排除机器故障。这种情况不仅会导致项目交付的延迟，还会增加预算严重超支的可能性。

虽然我们人类可以逃脱我们的错误，但在计算机科学中，最轻微的技术故障或不准确都可能危及技术设备的成功。毕竟用户期望值高，不喜欢不能正常使用应用的时候。

工程技术人员在预防应用问题方面发挥着至关重要的作用。

因此，他们在工程团队中发挥着不可或缺的作用。

# 工程技术人员的背景

工程技术人员通常与软件开发人员有相似的学术背景。他们中的大多数人拥有电子工程技术、计算机科学、计算机工程技术或相关领域的大专文凭。

虽然有工程技术人员在类似领域拥有学士学位，但他们是少数。这是因为本科学历不是工程技术人员的典型要求。然而，那些选择在学士水平上继续本科学习的人因其更丰富的学术知识而脱颖而出。

在经验方面，工程技术人员有各种工作背景。他们中的一些人有在计算机项目上直接与工程团队合作的经验，而其他人可能只有他们从学校获得的理论知识。

有趣的是，一些计算机工程技术人员实际上是从类似的角色转变而来的，例如机械工程技术人员、工业工程技术人员或电子工程技术人员。这些都构成了同一个“工程”术语的一部分，其中包括各种相互关联的学科，如土木工程或环境工程等。

# 区分技术人员和工程师

虽然很明显，工程技术人员通常在团队开发中扮演更具支持性的角色，但有时人们可能会认为他们的角色与工程团队的角色重叠。

为了理解工程技术人员与工程师的不同之处，考虑以下几点很重要:

*   资格:通常，工程技术人员没有像工程师那样严格、长期和广泛的学术背景或培训。他们可能接受过特定类型软件开发语言的培训，但不能接管任何工程任务。
*   **完成的任务**:对于工程技术人员来说，与使用软件相比，他们与计算机或其他物理设备的接触更密切，这种情况并不少见。
*   **预期角色**:工程技术人员受雇于需要解决一组特定工程任务的公司和组织。他们可能会被需要他们在工程软件应用方面的专业知识的团体以合同的形式雇佣，但只是临时性的。
*   职责:工程技术人员的技能通常在做出影响其工作结果的决策时较少承担责任。相比之下，软件工程师更倾向于了解工程过程，例如，在 Scrum 方法下，必须做出影响项目如何发展的决策。

# 计算机工程技术人员能为你做什么

下面是它们如何在软件开发项目的各个阶段提供帮助的分类:

*   计算机工程技术人员通常与其他工程师一起工作，并为您的项目提供一个成本更低的替代方案，而不是雇佣额外的软件工程师。
*   他们可以帮助完成一些更常规的任务，比如测试代码或者更新文档来构建和验证原型。
*   他们可以对工程决策提供有价值的第二意见，他们可能比软件开发新手更有经验。当您处理复杂或微妙的设计问题，需要额外的视角时，这尤其有用。
*   此外，工程技术人员可以成为您的首选专业人员，为您带来不同的工程学科。这对于跨学科的项目尤其有用。

# 雇佣一名计算机工程技术人员

当一名计算机工程技术人员入职时，你应该特别注意你的项目需求。

首先，工程技术人员的学术背景应该是你感兴趣的领域。有拥有计算机工程学位的工程技术人员，也有没有的。同样，也有那些拥有计算机工程副学士学位的人与那些学习同样课程但只有学士学位的人。一般来说，专业人士越有资格，他们对你的公司就越有价值。

此外，在组建您的团队时，技术人员的工作经验也很重要。例如，一个拥有多年软件开发经验的技术人员在解决敏捷项目中的问题时，可能和拥有更传统的 CS 技能的人一样得心应手。

对于许多招聘人员来说，这似乎是最符合逻辑的做法。如前所述，技术人员通常的平均工资是每小时 20 美元左右。大多数使用这些平台的自由职业者愿意接受较低的工资，因为他们生活在工资和生活成本较低的国家。

# 雇佣自由职业者的困难

工程技术人员的角色是您团队成功的基础。把你项目的命运和生命线交给自由职业者是极其危险的。

首先，自由职业平台和求职网站是骗子的天堂。事实上，这些平台上充斥着服务提供商，他们的技能、声称的专业知识或宣传的资格从未得到他们工作的平台或第三方的验证。这些候选人没有接受任何评估，以衡量他们是否适合提供他们所提供的服务。

因此，许多骗子利用自由职业者平台缺乏安全或审查程序的优势，试图在他们没有技能提供服务时与企业建立合作关系。

可悲的是，数百家企业已经成为这些非法专业人士的受害者。当他们意识到损害时，损害已经发生了，通常会对企业支出和预算造成严重破坏。

总而言之，这种类型的环境使招聘人员成为主要代理人，他们必须处理入职事宜，并确保他们雇用的候选人的合法性，经常要努力破译伪造的文件等。

也就是说，有时候，你可能会发现一个自由职业者，他拥有成为一名计算机工程师的技能和知识。然而，在大多数情况下，他们在利基市场的能力往往与他们对适当的项目管理过程的跟踪记录形成强烈对比。

这包括按时提交活动，并在整个产品生命周期中促进问责制和透明度。

简而言之，接受求职公告板和自由职业者的问题风险太大，不值得抄近路。
