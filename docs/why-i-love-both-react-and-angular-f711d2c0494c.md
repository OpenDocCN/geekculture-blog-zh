我喜欢与 React 和 Angular 一起工作的原因，以及为什么 Vue 不是一个选项。

# **JavaScript 框架和我使用 React 的经验**

与其他领域和语言不同，Javascript 框架并不是垄断性的，它们大量存在，并且以不同的形式存在于不同的用例中。

但是主要的三个是反应，角度和 Vue。我的职业生涯始于 React Dev，尽管 jsx 对一些人来说很难，但我喜欢它的简单性。我曾经使用过几乎所有不同类型的模块，这些模块可能是 webapp 的一部分，我发现 React 足以满足我的需求。

有时我不得不使用不同的第三方模块，这对我来说很好，给了我使用不同模块的经验，以及如何为应用程序选择正确的模块。

每当我遇到困难时，我喜欢的事情是做出反应，并在网上找到一个实际上对我有用的解决方案，因为该库提供了 API 稳定性。

**几年前推出的钩子重新点燃了人们的兴趣，去掉了许多不必要的样板文件，直接带你进入商业逻辑。**

**你也可以将你的定制实现提取到一个单独的文件中。**

使用 React for me 时必须有库 React Redux for large apps React Router Dom 和 React-hook forms。

# **使用角度**

使用 Angular 是完全不同的，因为它生成的文件数量太多了，引导一个模块可能会让一些人感到不知所措，但是一旦你明白你并不需要所有的东西，但是当你需要它的时候，它就在那里。

**虽然 Rxjs 是一个很难掌握的库，但我喜欢的 Angular、内置 HTTP 客户端、管道、AuthGuards、与 observables 一起工作的反应式表单的特性感觉非常强大和直观。**

Angular 提供的功能非常好用，如果你知道自己在做什么，它会给你很大的力量。

**这里有一个警告，如果你几年前在网上找到类似 Angular 2 的解决方案，你不确定这些突破性的变化今年会不会对你有用**

# **为什么我不喜欢 Vue，尽管它很特别**

Vue 真的很喜欢开发社区，并受到全球许多开发人员的赞赏，但与它一起工作，我感觉它是 Angular 和 React 组合的缩小版，这可能是一些人正在寻找的，但对我来说，它感觉不完整。这样一来，我所在的地区在**机会数量上就显得相当乏善可陈**。对我来说，要么你使用 React，它几乎囊括了所有不同层次的应用程序，要么你就全力以赴。除此之外，反应提供的 SSR 和 SSG 框架的数量，如 Next.js 和 Gatsby，在 Vue 中也有，但没有达到标准，或者如果足够好，也没有社区支持它。

当我从事某项工作时，主要的障碍是发现该领域和社区中的机会数量，我寻求的是作为一名技术主管，让更多的人加入我的团队并让他们尽快跟上进度有多容易。

不是要夺走 Vue 的任何东西，但现在，它不是我的茶。

你可以在网上找到很多关于 React 和 Angular 的教程。如果您打算使用 React，我建议您也学习 Next.js。

还有很多其他的荣誉提名，还有一个身材苗条的新成员，但是到目前为止，统治市场的是 3 个不同的框架/库。

**关键的一点是永远不要成为粉丝这些都是技术，会随风而变；)**

快乐编码，保持好奇心…
