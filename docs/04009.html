作为一名全栈工程师，我为什么喜欢React和Angular

原文：https://medium.com/geekculture/why-i-love-both-react-and-angular-f711d2c0494c?source=collection_archive---------18-----------------------#2021-06-19

我喜欢与React和Angular一起工作的原因，以及为什么Vue不是一个选项。

JavaScript框架和我使用React的经验

与其他领域和语言不同，Javascript框架并不是垄断性的，它们大量存在，并且以不同的形式存在于不同的用例中。

但是主要的三个是反应，角度和Vue。我的职业生涯始于React Dev，尽管jsx对一些人来说很难，但我喜欢它的简单性。我曾经使用过几乎所有不同类型的模块，这些模块可能是webapp的一部分，我发现React足以满足我的需求。

有时我不得不使用不同的第三方模块，这对我来说很好，给了我使用不同模块的经验，以及如何为应用程序选择正确的模块。

每当我遇到困难时，我喜欢的事情是做出反应，并在网上找到一个实际上对我有用的解决方案，因为该库提供了API稳定性。

几年前推出的钩子重新点燃了人们的兴趣，去掉了许多不必要的样板文件，直接带你进入商业逻辑。

你也可以将你的定制实现提取到一个单独的文件中。

使用React for me时必须有库React Redux for large apps React Router Dom和React-hook forms。

使用角度

使用Angular是完全不同的，因为它生成的文件数量太多了，引导一个模块可能会让一些人感到不知所措，但是一旦你明白你并不需要所有的东西，但是当你需要它的时候，它就在那里。

虽然Rxjs是一个很难掌握的库，但我喜欢的Angular、内置HTTP客户端、管道、AuthGuards、与observables一起工作的反应式表单的特性感觉非常强大和直观。

Angular提供的功能非常好用，如果你知道自己在做什么，它会给你很大的力量。

这里有一个警告，如果你几年前在网上找到类似Angular 2的解决方案，你不确定这些突破性的变化今年会不会对你有用

为什么我不喜欢Vue，尽管它很特别

Vue真的很喜欢开发社区，并受到全球许多开发人员的赞赏，但与它一起工作，我感觉它是Angular和React组合的缩小版，这可能是一些人正在寻找的，但对我来说，它感觉不完整。这样一来，我所在的地区在机会数量上就显得相当乏善可陈。对我来说，要么你使用React，它几乎囊括了所有不同层次的应用程序，要么你就全力以赴。除此之外，反应提供的SSR和SSG框架的数量，如Next.js和Gatsby，在Vue中也有，但没有达到标准，或者如果足够好，也没有社区支持它。

当我从事某项工作时，主要的障碍是发现该领域和社区中的机会数量，我寻求的是作为一名技术主管，让更多的人加入我的团队并让他们尽快跟上进度有多容易。

不是要夺走Vue的任何东西，但现在，它不是我的茶。

你可以在网上找到很多关于React和Angular的教程。如果您打算使用React，我建议您也学习Next.js。

还有很多其他的荣誉提名，还有一个身材苗条的新成员，但是到目前为止，统治市场的是3个不同的框架/库。

关键的一点是永远不要成为粉丝这些都是技术，会随风而变；)

快乐编码，保持好奇心…
