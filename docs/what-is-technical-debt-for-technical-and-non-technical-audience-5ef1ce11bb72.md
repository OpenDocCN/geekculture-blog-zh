“技术债务”是软件开发中使用的一个术语，用来描述由质量和速度之间的最初权衡所导致的延迟维护成本。

这是工程团队中常见的[比喻](https://resources.sei.cmu.edu/asset_files/Article/2012_101_001_424872.pdf),因为它使得技术决策有时不透明的成本在整个团队中变得容易理解。技术债务的功能就像任何其他类型的债务一样:先方便，后付款加利息。
