在过去的几年中，TypeScript 现在越来越受重视。它被评为 2020 年最受期待的五大语言之一。Angular 使用 TypeScript，这是最流行的前端框架之一。

大约 60%的 JS 开发人员使用 TypeScript，剩下的 22%有兴趣尝试一下。为什么？

传统上，JavaScript 一直是网页编码的主要语言，包括 web 技术。考虑到 Node.js 和 Deno 等框架，在前端和后端都使用 JavaScript 是可行的。

但是 JavaScript 是被设计来构建类似于最新网络上看到的大型复杂系统的吗？

嗯，答案是否定的。

在这篇文章中，我们将向您解释这个问题的解决方案——TypeScript——并让您开始将类型集成到现有 JavaScript 代码的旅程。

# 什么是 TypeScript？

简而言之，TypeScript 确实是带有动态类型的 JavaScript 的超集，与普通 JavaScript 进行校对。换句话说，TypeScript 就是在你需要的时候用静态编码的 JavaScript。

使用静态编码的优点是减少错误，简化代码修改的麻烦而不改变整个代码。它有助于简化复杂的编码，尤其是在大规模系统的情况下。

# 是什么让 TypeScript 优于 JavaScript？

事实上，根据一项研究，TypeScript 可以发现 15%的 JavaScript 错误。

动态类型的灵活性经常导致错误，这降低了开发人员的生产力；然而，由于合并新的编码部分的费用不断增加，它甚至会使进度慢如蜗牛。

因此，JavaScript 缺乏类型和编译时错误检查等特性，这使得它不适合企业和大型代码库的服务器端编程。

# 学习 TypeScript 的前提条件是什么？

TypeScript 的核心是一个 JS linter。或者，JS 带有编译器理解的指令。

因此，与 Coffee Script(提供语法糖)或 Pure Script(不像 JavaScript)等其他语言不同，您不需要学习太多就可以开始创建类型脚本代码。

对于 TypeScript，类型是不必要的，任何 JS 文件都是功能性的 TypeScript 文件。如果您的初始文件中有类型错误，编译器会发出警告，但这应该会提供一个 JavaScript 文件，其功能与前面一样。无论你在哪里，TypeScript 都会和你打招呼，稳步提升自己的能力很简单。

# TypeScript 适合前端和后端开发吗？

JavaScript 是从 TypeScript 生成的。因此，TS 可以用在任何可以使用 JS 的地方，包括前端和后端。

JavaScript 可能是前端平台和网站使用最广泛的脚本技术。类似地，TypeScript 可以用于类似的目的；然而，它主要在大型企业应用程序的服务器端表现出色。

# 比较 TypeScript 和 JavaScript

*   TypeScript 是一种面向对象的脚本语言，与 JavaScript 相同
*   TypeScript 是一种编译成 JavaScript 的依赖语言，而 JavaScript 是一种独立的语言，可以被解释和执行。
*   TypeScript 是一种编译语言，不能在浏览器中直接执行，而 JavaScript 则不是这样。
*   TypeScript 最适合执行大型项目，而 JavaScript 不是简单的小型项目。

# 选择 TypeScript 而不是 JavaScript 的三大理由

> **高度可靠**

与 JavaScript 代码相比，TypeScript 代码更值得信赖，重构起来也更简单。这使得程序员避免错误和修改代码变得相当简单。

类型消除了 JavaScript 开发中可能出现的大多数简单错误。它们提供了及时的反馈机制，以解决开发新代码和调试时的所有小问题。

> **高度露骨**

保持特定类型集中于我们的生态系统是如何设计的，以及它的不同组件是如何相互通信的。

巨大的结构必须包含身体的功能，同时保持帐户中的框架。类型赋予我们这样做的权力。

> **最适合与 JavaScript 交互**

由于 JavaScript 是 TypeScript 的子集，任何人都可以利用现有 TypeScript 项目中的任何 JavaScript 库或功能。

2020 年，最广泛使用的 JavaScript 库将会以类型为特色——Typed 是众多 JavaScript 框架的类型库，你可以用它来改进你的操作，特别是类型安全。

上述内容意味着您可以逐步将 TypeScript 合并到您的 JavaScript 开发中，首先将类型引入特定的部分，然后用您的代码世界扩展它。

# 打字稿的缺点

您不能简单地将 JavaScript 工作组或 JavaScript 源转换成传统的 TypeScript。似乎需要进行一些考虑，以及大量的资源支出。

虽然从长远来看，我们可能会争论清楚类型的好处，但这确实需要更多的努力来在短期内将事情合并。这似乎不是什么大事；然而，它提供了一个支持 JavaScript 的论点。

因此，由于您的使用，您可能不希望为小规模应用程序(包括原型)实现 TypeScript。

# 最后的话

最后，主要是如果您没有最大限度地使用 TypeScript，它确实是您的工具箱中拥有的一个优秀工具。从基础开始，逐步扩展，随着您的接近，发现和实现功能是很简单的。没有理由感到害怕，因为 TypeScript 是现实的，对新来者是友好的。
