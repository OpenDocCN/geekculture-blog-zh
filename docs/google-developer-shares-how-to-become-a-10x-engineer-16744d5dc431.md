# 主要亮点

以下是这篇文章将涉及的关键细节-

1.  **10 倍工程师 vs 尝试新事物-**10 倍工程师可以取得很大成就，因为他们最终会使用他们已经掌握的工具和技术。然而，这也意味着不要在新的框架/工具/语言上花费时间，有时你最终会使用与你的组织不兼容的工具。这是一个你必须解决的矛盾。我将分享我平衡这种权衡的经验。
2.  **做更多的编码=/=成为一个更好的编码者-** 作为工程师，试图把一切都当作工程问题来处理是很诱人的。遇到问题？尝试编写解决方案。这个领域也为这样的人进行自我选择。然而，重要的是有时后退一步，而不是匆忙。深呼吸，花点时间想想利益相关者。和很多人交谈。这肯定会有回报的。
3.  背景转换不利于创造性工作- 谷歌的一项研究表明，为了达到最佳表现，你需要避免持续的会议和干扰。为深度工作留出一段时间，尽量减少会议，每周只开一天会，这些都被证明是非常有效的。
4.  **生产力要求 F . r . e . e .**——专注、以结果为导向、效率和同情心。这是一个提高生产力的惊人框架。不要错过这个。

# 做得更多与做得不同

我发现的第一个非常有说服力的想法是关于超高生产力和收集测试新事物的想法。这是我们必须做出的权衡，平衡两者是职业发展的关键。

Chris 提到有时 10x 工程师与技术团队的需求不一致。他们可能是一个领域/技术堆栈的主人，并使用它来构建解决方案。然而，从长远来看，这些工具和技术的使用可能并不总是对项目最好的。

作为一名工程师，你在旅途中也会面临这种困境。当你坚持一个领域并掌握一系列技术时，你将能够完成更多的工作。然而，从长远来看，这可能会阻止你探索其他领域并获得更多。但是这个探索的过程是需要投入的，你尝试的大部分东西都不会很适合你。

熟悉人工智能的人会知道这种两难境地，即探索与开发的权衡。为了找到一个真正好的解决方案，你应该建立一个人工智能来满足一个好的解决方案，还是你的代理应该探索许多领域(最糟糕的)？后面我会分享我对如何平衡两者的想法。

# 如何平衡勘探和开采

开始时，你绝对应该检查不同的领域和框架。测试概念和想法，看看哪个领域最让你感兴趣。我学 Java 是因为在印度他们是这么教我们的。我选择了 Android，因为那也是我儿子 Noni 喜欢的。我碰巧在看一些关于决策理论的讲座，这是我进入 ML 的原因。为我的气候变化建模工作学习 JS。诸如此类…

跳来跳去让我接触到不同的想法，并教会我如何快速学习新的框架。它还教会了我如何识别什么是该问的问题。最重要的是，它帮助我确定我不喜欢什么，以及我会做什么。Android 很有趣，但设置却是一场噩梦。我真的不喜欢网页设计中的 CSS/设计元素。机器学习让我将我对数学的热爱和我的编码能力结合在一起。

一旦你找到了与你的技能和爱好相匹配的东西，你就可以投入大量的时间来学习你的专业领域。比如我这几天做的大部分研究都和机器学习有关。我不申请非 ML 角色。ML 是我决定追求精通的领域。

# 少即是多

> *对拿着锤子的人来说，一切看起来都像钉子。*

这是我最喜欢的名言之一。这也适用于软件工程师。有时我们需要放下锤子，拿出其他工具。或者直接跟钉子说话。我什么意思？

作为工程师，尝试为所有事情编写解决方案是很诱人的。这是一个致命的错误。有时，最好的解决方案是与利益相关者交谈，并基于此进行筛选。正如前面提到的 rockstar 工程师所做的那样，有时解决方案是简单地识别痛点，并要求它们不要那么痛苦。

# 提高您的生产力

Chris 提到了一些组织/工作方式的改变，这些改变可以让你变得更有效率。这些是最有趣的-

1.  **留出一段时间进行深度工作** -他举了一个例子，为整个组织每天留出 1-4 点时间进行深度工作。这使得人们更有效率，因为他们周围的整个环境在这个时候都在工作。
2.  **尽量减少会议**——会议可以帮助你分享挑战，并向他人学习和更新。但是太多会降低你的生产力和工作效率。
3.  **每周有一天不开会-** 克里斯举了一个周三不开会的例子。

这些都是有效地将你的注意力转移到工作上的方法。你的注意力是一种强大的货币。一些最大的公司都在为你的观点而战，这不是巧合。我们很快会谈到注意力经济。现在，只要记住你的注意力是至关重要的。生产力要求你全神贯注于手头的任务。

关于生产力，Chris 分享了他的高水平生产力的要素。我们最后会谈到这一点。

# 生产力是免费的。

一旦我们仔细阅读每一封信，你会发现我们表达的所有想法是如何联系在一起的-

1.  专注- 这意味着专注于正确的事情。1 个巨大的附加值比 10 个普通的要好得多。[记住，技术是以规模运作的](https://codinginterviewsmadesimple.substack.com/p/the-mathematics-of-scale-math-mondays)。慢慢挑选最有影响力的战斗。
2.  **结果导向-** 你做的每一件事都必须有一个目标。当涉及到你的工作时，确保你所做的一切都有助于最终目标。如果没有，那就不值得做。然而，确保你在不工作的时候通过探索其他兴趣来平衡这一点。我们已经讨论过这将如何帮助你的工作(并使生活更有趣)。
3.  **高效-** 看到一个你反复做的任务？自动化它。如果它不够重要(并且难以自动化)，就把责任委托给其他人。构建您的管道，以确保最大限度地减少浪费的时间/精力。
